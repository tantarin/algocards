22. Generate Parentheses
Дано число n — количество пар скобок. Необходимо сгенерировать все допустимые комбинации из n пар скобок.
Допустимая комбинация означает:
- Каждой открывающей скобке ( соответствует закрывающая )
- При чтении слева направо количество закрывающих никогда не превышает количество открывающих
Поиск в глубину с отсечением ветвей (DFS + pruning).
Отслеживаем open (открывающие) и close (закрывающие).
- Не добавляем больше n каждого типа
- close никогда не превышает open
- Когда open равен close и равен n — допустимая комбинация найдена
class Solution {
    private List<String> result = new ArrayList<>();
    private int pairCount;
    public List<String> generateParenthesis(int n) {
        this.pairCount = n;
        backtrack(0, 0, "");
        return result;
    }
    private void backtrack(int openCount, int closeCount, String current) {
        // нарушен инвариант — ветка невалидна
        if (openCount > pairCount || closeCount > pairCount || openCount < closeCount) {
            return;
        }
        // использовали все n пар — готовая комбинация
        if (openCount == pairCount && closeCount == pairCount) {
            result.add(current);
            return;
        }
        backtrack(openCount + 1, closeCount, current + "(");
        backtrack(openCount, closeCount + 1, current + ")");
    }
}
Время: O(4^n / √n), Память: O(n)
Рекурсивный DFS с двумя ветвлениями строит все скобочные последовательности; их число — число Каталана O(4^n/√n). Глубина рекурсии и длина строки — O(n) памяти.
Декартово произведение
Дано N массивов. Вернуть декартово произведение — все комбинации, где из каждого массива выбран ровно один элемент.
На каждом уровне рекурсии выбираем элемент из соответствующего массива. После возврата удаляем последний элемент (backtrack). Когда idx == arrays.size() — комбинация готова.
class Solution {
    private List<List<Integer>> result = new ArrayList<>();
    public List<List<Integer>> cartesianProduct(List<List<Integer>> arrays) {
        backtrack(arrays, 0, new ArrayList<>());
        return result;
    }
    private void backtrack(List<List<Integer>> arrays, int idx, List<Integer> current) {
        if (idx == arrays.size()) {
            result.add(new ArrayList<>(current));
            return;
        }
        for (int val : arrays.get(idx)) {
            current.add(val);
            backtrack(arrays, idx + 1, current);
            current.remove(current.size() - 1);
        }
    }
}
Время: O(∏ |arrays[i]|), Память: O(k)
На каждом уровне перебираем элементы массива, рекурсия углубляется — всего перебирается произведение размеров. Глубина рекурсии = число массивов k — O(k) памяти.
Перебор IP-адресов
Дана строка, содержащая только цифры. Вернуть все возможные валидные IP-адреса.
Валидный IP: 4 октета, каждый 0–255, без ведущих нулей.
Разбиваем строку на 4 октета. Для каждого пробуем 1–3 цифры. Проверяем валидность: 0–255 без ведущих нулей. Когда 4 части и строка кончилась — IP найден.
class Solution {
    private List<String> result = new ArrayList<>();
    public List<String> restoreIpAddresses(String s) {
        backtrack(s, 0, new ArrayList<>());
        return result;
    }
    private void backtrack(String s, int start, List<String> parts) {
        // Набрали 4 части — проверяем, что использовали всю строку
        if (parts.size() == 4) {
            if (start == s.length()) {
                result.add(String.join(".", parts));
            }
            return;
        }
        // Пробуем длину октета 1, 2 или 3 символа
        for (int len = 1; len <= 3; len++) {
            if (start + len > s.length()) break;
            String segment = s.substring(start, start + len);
            // Запрещаем ведущие нули (например "01")
            if (segment.length() > 1 && segment.startsWith("0")) break;
            // Проверяем диапазон 0–255
            int val = Integer.parseInt(segment);
            if (val > 255) break;
            // Рекурсивный перебор
            parts.add(segment);
            backtrack(s, start + len, parts);
            parts.remove(parts.size() - 1); // откат
        }
    }
}
Время: O(1) (3^4 = 81 ветвлений), Память: O(1)
Четыре октета, на каждом не более трёх длин — конечное дерево ветвлений (константа). Рекурсия глубины 4, список из 4 строк — O(1) памяти.
Remove Invalid Parentheses
Дана строка, содержащая скобки и другие символы. Удалить минимальное количество невалидных скобок, чтобы строка стала валидной. Вернуть все уникальные результаты.
Считаем лишние открывающие и закрывающие скобки. Backtrack: пробуем удалить каждую, пропуская дубликаты (если s[i]s[i-1]). Когда обе счётчика  0, проверяем валидность строки.
class Solution {
    private Set<String> result = new HashSet<>();
    public List<String> removeInvalidParentheses(String s) {
        int openToRemove = 0, closeToRemove = 0;
        for (char c : s.toCharArray()) {
            if (c == '(') openToRemove++;
            else if (c == ')') {
                if (openToRemove > 0) openToRemove--;
                else closeToRemove++;
            }
        }
        backtrack(s, 0, openToRemove, closeToRemove);
        return new ArrayList<>(result);
    }
    private void backtrack(String s, int start,
                           int openRem, int closeRem) {
        if (openRem == 0 && closeRem == 0) {
            if (isValid(s)) result.add(s);
            return;
        }
        for (int i = start; i < s.length(); i++) {
            if (i > start && s.charAt(i) == s.charAt(i - 1))
                continue;
            char c = s.charAt(i);
            String next = s.substring(0, i) + s.substring(i + 1);
            if (c == '(' && openRem > 0)
                backtrack(next, i, openRem - 1, closeRem);
            if (c == ')' && closeRem > 0)
                backtrack(next, i, openRem, closeRem - 1);
        }
    }
    private boolean isValid(String s) {
        int count = 0;
        for (char c : s.toCharArray()) {
            if (c == '(') count++;
            else if (c == ')') count--;
            if (count < 0) return false;
        }
        return count == 0;
    }
}
Время: O(2^n · n), Память: O(n)
В худшем случае backtrack перебирает O(2^n) подпоследовательностей. Для каждой из них при openRem0 && closeRem0 вызывается isValid — линейный проход O(n). Итого O(2^n · n). Рекурсия глубины n, строки длиной n — O(n) памяти на уровень стека.
Letter Combinations of a Phone Number
Дана строка s, содержащая цифры от 2 до 9. Нужно вернуть все возможные комбинации букв, которые могли получиться при нажатии на эти цифры на телефоне, в порядке, соответствующем цифрам в строке s. Ответ можно вернуть в любом порядке.
Это дерево перебора: каждая цифра задаёт следующий слой, а её буквы — ветви. На уровне idx выбираем одну букву для s[idx], добавляем её в текущую строку и рекурсивно строим комбинации для оставшихся цифр. Когда дошли до конца строки, текущая комбинация готова.
class Solution {
    private static final String[] PHONE = {
        "", "", "abc", "def", "ghi", "jkl",
        "mno", "pqrs", "tuv", "wxyz"
    };
    public List<String> letterCombinations(String s) {
        List<String> result = new ArrayList<>();
        if (s == null || s.isEmpty()) return result;
        backtrack(s, 0, new StringBuilder(), result);
        return result;
    }
    private void backtrack(String s, int idx,
                           StringBuilder current,
                           List<String> result) {
        if (idx == s.length()) {
            result.add(current.toString());
            return;
        }
        String letters = PHONE[s.charAt(idx) - '0'];
        for (int i = 0; i < letters.length(); i++) {
            current.append(letters.charAt(i));
            backtrack(s, idx + 1, current, result);
            current.deleteCharAt(current.length() - 1);
        }
    }
}
Время: O(4^n · n), Память: O(n)
Для каждой позиции перебираем 3 или 4 буквы; в худшем случае все цифры дают по 4 варианта, значит число комбинаций O(4^n). Построение строки результата длины n для каждой готовой комбинации даёт множитель n. Глубина рекурсии и текущий StringBuilder — O(n) памяти, не считая списка ответа.
LC 33 — Search in Rotated Sorted Array
Дан целочисленный массив nums, изначально отсортированный по возрастанию, содержащий уникальные значения. Массив мог быть повернут влево в некоторой неизвестной точке k.
Найти заданное target значение и вернуть его индекс. Если не найдено — вернуть -1.
Если идти в лоб, получится обычный линейный поиск. Но после поворота массив не стал полностью хаотичным: в любой момент одна из половин всё равно остаётся отсортированной. Значит, на каждом шаге можно понять, в какой половине target точно не лежит, и выбросить её. Отсюда и получается модифицированный бинарный поиск, а не проход по всем элементам.
class Solution {
    public int search(int[] nums, int target) {
        int left = 0;
        int right = nums.length - 1;
        while (left <= right) {
            int mid = left + (right - left) / 2;
            if (nums[mid] == target) {
                return mid;
            }
            // левая половина [left..mid] отсортирована
            if (nums[left] <= nums[mid]) {
                // target входит в отсортированный диапазон [left..mid)
                if (nums[left] <= target && target < nums[mid]) {
                    right = mid - 1;
                } else {
                    // target вне левой половины — ищем справа
                    left = mid + 1;
                }
            } else {
                // правая половина [mid..right] отсортирована
                // target входит в отсортированный диапазон (mid..right]
                if (nums[mid] < target && target <= nums[right]) {
                    left = mid + 1;
                } else {
                    // target вне правой половины — ищем слева
                    right = mid - 1;
                }
            }
        }
        return -1;
    }
}
Время: O(log n) сравнений (каждое O(1)), Память: O(1) доп. (три индекса left, right, mid — фиксированное число переменных, не от n)
Интервал [left, right] сокращается примерно вдвое за шаг, шагов O(log n). Дополнительно храним только границы и mid.
 LC 34 — Find First and Last Position of Element in Sorted Array
Дан отсортированный массив целых чисел nums и значение target. Найти первую и последнюю позиции target.
Для левой границы: при nums[mid]==target идём влево (hi=mid-1). Для правой: идём вправо (lo=mid+1). Оба поиска O(log n).
class Solution {
    public int[] searchRange(int[] nums, int target) {
        int left = lowerBound(nums, target);
        int right = upperBound(nums, target) - 1;
        if (left == nums.length || nums[left] != target) {
            return new int[]{-1, -1};
        }
        return new int[]{left, right};
    }
    // Первый индекс i, такой что nums[i] >= target
    private int lowerBound(int[] nums, int target) {
        int lo = 0, hi = nums.length;
        while (lo < hi) {
            int mid = lo + (hi - lo) / 2;
            if (nums[mid] >= target) {
                hi = mid;
            } else {
                lo = mid + 1;
            }
        }
        return lo;
    }
    // Первый индекс i, такой что nums[i] > target
    private int upperBound(int[] nums, int target) {
        int lo = 0, hi = nums.length;
        while (lo < hi) {
            int mid = lo + (hi - lo) / 2;
            if (nums[mid] > target) {
                hi = mid;
            } else {
                lo = mid + 1;
            }
        }
        return lo;
    }
}
Время: O(log n), Память: O(1)
Два бинарных поиска (findLeft + findRight) — суммарно O(log n). Константные переменные — O(1) памяти.
LeetCode 704. Binary Search
Дан отсортированный (по возрастанию) массив целых чисел nums и целое число target. Напишите функцию для поиска target в nums. Если target существует, верните его индекс, иначе верните -1.
O(log n) время, O(1) память. На каждом шаге пространство поиска сокращается вдвое.
class Solution {
    public int search(int[] nums, int target) {
        int left = 0;
        int right = nums.length - 1;
        while (left <= right) {
            int mid = left + (right - left) / 2;  // защита от переполнения
            if (nums[mid] == target) {
                return mid;
            } else if (nums[mid] < target) {
                left = mid + 1;   // ищем в правой половине
            } else {
                right = mid - 1;  // ищем в левой половине
            }
        }
        return -1;  // не найдено
    }
}
Время: O(log n), Память: O(1)
Классический бинарный поиск: интервал сокращается вдвое — O(log n). Только lo, hi, mid — O(1) памяти.
Validate BST
Дан корень бинарного дерева. Проверить, является ли оно валидным (правильным) деревом поиска (BST).
Свойства BST:
- Все значения в левом поддереве строго меньше значения узла
- Все значения в правом поддереве строго больше значения узла
- Оба поддерева также должны быть валидными BST
Рекурсия с границами. Для левого поддерева: max = node.val. Для правого: min = node.val. Используем long чтобы обработать граничные значения Integer.
class Solution {
    public boolean isValidBST(TreeNode root) {
        return isValid(root, Long.MIN_VALUE, Long.MAX_VALUE);
    }
    private boolean isValid(TreeNode node, long low, long high) {
        if (node == null) return true;
        if (node.val <= low || node.val >= high) {
            return false;
        }
        return isValid(node.left, low, node.val) && isValid(node.right, node.val, high);
    }
}
Время: O(n), Память: O(h)
Рекурсия isValid посещает каждый узел один раз — O(n). Стек вызовов по высоте дерева — O(h) памяти.
LeetCode 356: Line Reflection
Дан массив точек (x, y). Определить, существует ли вертикальная прямая, относительно которой все точки симметричны.
Сначала хочется как-то угадывать ось симметрии, но код опирается на более жёсткое наблюдение: если симметрия есть, то крайние точки по x уже фиксируют эту ось как середину между minX и maxX. После этого задача сводится к простой проверке пар: для каждой точки (x, y) должна существовать зеркальная (minX + maxX - x, y). Поэтому точки складываются в HashSet, чтобы каждую такую проверку делать быстро.
class Solution {
    public boolean isSymmetric(int[][] points) {
        int minX = Integer.MAX_VALUE;
        int maxX = Integer.MIN_VALUE;
        Set<String> pointSet = new HashSet<>();
        for (int[] p : points) {
            minX = Math.min(minX, p[0]);
            maxX = Math.max(maxX, p[0]);
            pointSet.add(p[0] + "," + p[1]);
        }
        // axis = (minX + maxX) / 2, но умножаем на 2 чтобы не работать с double
        // mirrorX = 2*axis - x = (minX + maxX) - x
        int axisDouble = minX + maxX;
        for (int[] p : points) {
            int mirrorX = axisDouble - p[0];
            if (!pointSet.contains(mirrorX + "," + p[1])) {
                return false;
            }
        }
        return true;
    }
}
Время: O(n) (два прохода по точкам), Память: O(n) (HashSet строк-ключей «x,y», до одной на точку)
Первый проход: min/max и вставка ключей. Второй: contains за O(1) в среднем на ключ фиксированной длины (координаты). Память — множество из ≤ n ключей.
Строгая симметрия по оси Y
Дан массив точек points. Нужно вернуть true, если существует такая прямая, параллельная оси Y, которая симметрично отражает все данные точки и false, если такой прямой нет.
ВАЖНО: При этом каждая точка должна иметь симметричную ей точку в массиве с таким же числом вхождений. Если же точка имеет координату X равную оси симметрии, то она симметрична сама себе в любом случае.
Строгая биекция: количество совпадений точки и её зеркала должно совпадать. Точки на оси допустимы. O(n) время и память.
class Solution {
    public boolean isStrictlySymmetric(int[][] points) {
        int minX = Integer.MAX_VALUE;
        int maxX = Integer.MIN_VALUE;
        Map<String, Integer> countMap = new HashMap<>();
        for (int[] p : points) {
            minX = Math.min(minX, p[0]);
            maxX = Math.max(maxX, p[0]);
            String key = p[0] + "," + p[1];
            countMap.merge(key, 1, Integer::sum);
        }
        // Ось симметрии
        int axisSum = minX + maxX;  // 2 * axis, всегда целое число
        for (int[] p : points) {
            int mirrorX = axisSum - p[0];  // зеркальная координата X
            String key = p[0] + "," + p[1];
            String mirrorKey = mirrorX + "," + p[1];
            // Количество точек должно совпадать с количеством их зеркальных отражений
            if (!countMap.getOrDefault(key, 0).equals(countMap.getOrDefault(mirrorKey, 0))) {
                return false;
            }
        }
        return true;
    }
}
Время: O(n), Память: O(n)
Два линейных прохода: HashMap кратностей, затем проверка зеркальных пар — O(n). Карта до n ключей — O(n) памяти.
LeetCode 356 — Line Reflection
Дан набор точек на плоскости. Определить, существует ли прямая, параллельная оси Y, которая отражает все точки.
Логика та же: сначала находим крайние x и тем самым фиксируем возможную ось симметрии. Дальше для каждой точки нужно проверить, есть ли у неё зеркальная относительно этой оси. Код специально хранит не саму ось через деление, а сумму minX + maxX, чтобы обойтись без double и считать зеркальный x как sumAxis - x. После этого остаётся только быстрая проверка через HashSet.
class Solution {
    public boolean isReflected(int[][] points) {
        int minX = Integer.MAX_VALUE;
        int maxX = Integer.MIN_VALUE;
        Set<String> set = new HashSet<>();
        for (int[] p : points) {
            minX = Math.min(minX, p[0]);
            maxX = Math.max(maxX, p[0]);
            set.add(p[0] + "," + p[1]);
        }
        int sumAxis = minX + maxX;
        for (int[] p : points) {
            int mirrorX = sumAxis - p[0];
            if (!set.contains(mirrorX + "," + p[1])) {
                return false;
            }
        }
        return true;
    }
}
Время: O(n) (два прохода), Память: O(n) (HashSet ключей «x,y», до n записей)
Первый проход строит множество точек; второй проверяет для каждой точки зеркальный ключ. Операции с множеством — O(1) в среднем на короткий ключ.
Кратчайший путь в графе
Дан невзвешенный неориентированный граф в виде списка рёбер edges и две вершины start и end. Расстояния между вершинами равны 1. Найти кратчайший путь от start до end. Вернуть список вершин пути или пустой список, если путь не существует.
BFS гарантирует кратчайший путь в невзвешенном графе. Очередь FIFO обрабатывает сначала ближайших соседей. Восстановление пути через карту родителей с обратным обходом от end к start.
public class Solution {
    public List<Integer> findShortestPath(
            List<List<Integer>> edges,
            Integer start, Integer end) {
        Map<Integer, List<Integer>> graph = new HashMap<>();
        for (List<Integer> edge : edges) {
            int u = edge.get(0), v = edge.get(1);
            graph.putIfAbsent(u, new ArrayList<>());
            graph.putIfAbsent(v, new ArrayList<>());
            graph.get(u).add(v);
            graph.get(v).add(u);
        }
        Queue<Integer> queue = new LinkedList<>();
        queue.add(start);
        Map<Integer, Integer> visited = new HashMap<>();
        visited.put(start, null);
        while (!queue.isEmpty()) {
            int node = queue.poll();
            if (node == end) break;
            for (int nb : graph.getOrDefault(node, new ArrayList<>())) {
                if (!visited.containsKey(nb)) {
                    visited.put(nb, node);
                    queue.add(nb);
                }
            }
        }
        if (!visited.containsKey(end)) return new ArrayList<>();
        List<Integer> path = new ArrayList<>();
        Integer cur = end;
        while (cur != null) {
            path.add(cur);
            cur = visited.get(cur);
        }
        Collections.reverse(path);
        return path;
    }
}
Время: O(V + E), Память: O(V + E)

Запутанный маршрут туриста
Дан набор пар городов tickets, между которыми турист совершил прямые перелёты. Информация о направлении перелётов утеряна — tickets[i] = [A, B] означает перелёт между A и B в любую сторону. Также утерян порядок перелётов.
Известно, что все указанные перелёты относятся к одному путешествию. Каждый следующий перелёт турист начинал из того города, в котором закончил предыдущий. Никакой город не был посещён туристом дважды. Город начала путешествия отличается от конечной точки.
Необходимо восстановить маршрут следования и вернуть города в порядке маршрута. Существует два возможных ответа — подойдёт любой.
Совокупность рёбру без повторов вершин образует простой путь в неориентированном графе. У концов пути степень 1, внутри — 2, поэтому лист (степень 1) — один из концов. Идя от него, на каждом внутреннем шаге из двух соседей ровно один ещё не посещён — переход к нему однозначен. После n шагов посещены все n+1 вершин — маршрут совпал с оригиналом (прямым или обратным).
import java.util.*;
public class Solution {
    public List<String> route(List<List<String>> tickets) {
        // для каждого города храним список городов, с которыми он связан
        //без графа мы не знаем соседей
        Map<String, List<String>> graph = new HashMap<>();
        for (List<String> ticket : tickets) {
            //у серединных городов по два соседа
            graph.computeIfAbsent(ticket.get(0), k -> new ArrayList<>()).add(ticket.get(1));
            graph.computeIfAbsent(ticket.get(1), k -> new ArrayList<>()).add(ticket.get(0));
        }
        // начальный город — тот, у которого ровно одна связь (край маршрута)
        String start = "";
        for (Map.Entry<String, List<String>> entry : graph.entrySet()) {
            if (entry.getValue().size() == 1) {
                start = entry.getKey();
                break;
            }
        }
        // восстанавливаем маршрут, отмечая посещённые города
        List<String> result = new ArrayList<>();
        result.add(start);
        //Если середина — у города 2 соседа, но один уже посещён, второй — нет
        Set<String> visited = new HashSet<>();
        visited.add(start);
        for (int i = 0; i < tickets.size(); i++) {
            String current = result.get(result.size() - 1);
            for (String neighbor : graph.get(current)) {
                if (!visited.contains(neighbor)) {
                    visited.add(neighbor);
                    result.add(neighbor);
                    break;
                }
            }
        }
        return result;
    }
}
Время: O(n) (n — число рёбер; построение списков смежности + обход), Память: O(n) (граф, посещения, маршрут)
Суммарная длина списков смежности 2n — O(n) построение. На каждом из n шагов смотрим не более константы соседей у вершины степени 2, у конца — 1. HashMap и Set — O(кол-во вершин) ≤ O(n+1).
Циклические зависимости
Дан список зависимостей вида [A зависит от B]. Определить, есть ли циклическая зависимость.
Алгоритм Кана: считаем in-degree для каждой вершины, добавляем в очередь вершины с 0 входящих. Удаляем их, уменьшая in-degree соседей. Если не все вершины обработаны — граф содержит цикл. O(V+E).
class Solution {
    // Алгоритм Кана: топологическая сортировка для обнаружения цикла
    // O(V+E) по времени, O(V+E) по памяти
    public boolean hasCycle(List<List<String>> deps) {
        // Граф: вершина -> список соседей (куда идут ребра)
        Map<String, List<String>> graph = new HashMap<>();
        // Счетчик входящих ребер для каждой вершины
        Map<String, Integer> indegree = new HashMap<>();
        // 1. Строим граф и считаем in-degree
        for (List<String> dep : deps) {
            String from = dep.get(0), to = dep.get(1);
            graph.putIfAbsent(from, new ArrayList<>());
            graph.putIfAbsent(to, new ArrayList<>());
            indegree.putIfAbsent(from, 0);
            indegree.putIfAbsent(to, 0);
            graph.get(from).add(to);      // ребро from -> to
            indegree.merge(to, 1, Integer::sum); // увеличиваем in-degree для to
        }
        // 2. Находим все вершины без входящих ребер (стартовые)
        Queue<String> queue = new LinkedList<>();
        for (var entry : indegree.entrySet()) {
            if (entry.getValue() == 0)
                queue.add(entry.getKey());
        }
        // 3. Последовательно удаляем вершины с in-degree = 0
        int processed = 0;
        while (!queue.isEmpty()) {
            String node = queue.poll();
            processed++;
            // Уменьшаем in-degree у всех соседей
            for (String nb : graph.getOrDefault(node, new ArrayList<>())) {
                indegree.merge(nb, -1, Integer::sum); // уменьшаем на 1
                if (indegree.get(nb) == 0)
                    queue.add(nb); // новый кандидат на удаление
            }
        }
        // 4. Если обработали не все вершины -> есть цикл
        return processed != indegree.size();
    }
}
Время: O(V + E), Память: O(V + E)
Kahn: каждое ребро снимается один раз при уменьшении indegree — O(V+E). Граф, очередь и счётчики — O(V+E) памяти.
LC 207 — Course Schedule
Всего необходимо пройти numCourses курсов, обозначенных от 0 до numCourses - 1. Дан массив prerequisites, где prerequisites[i] = [ai, bi] означает: чтобы пройти курс ai, сначала нужно пройти курс bi.
Верните true, если можно закончить все курсы. Иначе — false.
Задача сводится к поиску цикла в ориентированном графе. Алгоритм Кана: начинаем с курсов без prerequisites (indegree = 0), поочерёдно «снимаем» их и уменьшаем indegree соседей. Если удалось обработать все n курсов — цикла нет → true. Иначе — в графе есть цикл → false.
class Solution {
    // Алгоритм Кана: можно ли выполнить все курсы (нет цикла в зависимостях)
    public boolean canFinish(int n, int[][] prerequisites) {
        List<Integer>[] adj = new List[n];
        int[] indegree = new int[n];
        // 1. Строим граф
        for (int[] pair : prerequisites) {
            int course = pair[0];
            int prereq = pair[1];
            if (adj[prereq] == null) adj[prereq] = new ArrayList<>();
            adj[prereq].add(course);
            indegree[course]++;
        }
        // 2. Находим стартовые вершины (нет prerequisite)
        Queue<Integer> queue = new LinkedList<>();
        for (int i = 0; i < n; i++) {
            if (indegree[i] == 0) queue.offer(i);
        }
        // 3. Последовательно удаляем вершины с indegree = 0
        int processed = 0;
        while (!queue.isEmpty()) {
            int cur = queue.poll();
            processed++;
            if (adj[cur] != null) {
                for (int next : adj[cur]) {
                    indegree[next]--;
                    if (indegree[next] == 0) queue.offer(next);
                }
            }
        }
        // 4. Если обработали все курсы — цикла нет
        return processed == n;
    }
}
Время: O(V + E), Память: O(V + E)
Каждая вершина и каждое ребро обрабатываются один раз. Граф, очередь и indegree — O(V+E) памяти.
 LC 802 — Find Eventual Safe States.
Дан ориентированный граф из n узлов (graph[i] — список соседей узла i).
- Терминальный узел — нет исходящих рёбер.
- Безопасный узел — каждый путь из него ведёт в терминальный (или другой безопасный) узел.
Вернуть все безопасные узлы, отсортированные по возрастанию.
Безопасные узлы = не участвующие в цикле и не ведущие в цикл. Алгоритм Кана на обратном графе: терминальные (outdegree=0) безопасны; убираем их рёбра, уменьшаем outdegree родителей — все достигшие 0 тоже безопасны. Альтернатива — DFS с 3 состояниями: VISITING при повторном заходе означает цикл.
class Solution {
    public List<Integer> eventualSafeNodes(int[][] graph) {
        int n = graph.length;
        List<List<Integer>> reverse = new ArrayList<>();
        for (int i = 0; i < n; i++)
            reverse.add(new ArrayList<>());
        int[] outdegree = new int[n];
        // 1. Обратный граф + outdegree
        for (int u = 0; u < n; u++) {
            outdegree[u] = graph[u].length;
            for (int v : graph[u])
                reverse.get(v).add(u);
        }
        // 2. Терминальные вершины в очередь
        Queue<Integer> q = new ArrayDeque<>();
        for (int i = 0; i < n; i++)
            if (outdegree[i] == 0) q.offer(i);
        boolean[] safe = new boolean[n];
        // 3. BFS (Kahn на обратном графе)
        while (!q.isEmpty()) {
            int node = q.poll();
            safe[node] = true;
            for (int parent : reverse.get(node))
                if (--outdegree[parent] == 0)
                    q.offer(parent);
        }
        // 4. Собираем ответ
        List<Integer> res = new ArrayList<>();
        for (int i = 0; i < n; i++)
            if (safe[i]) res.add(i);
        return res;
    }
}
Время: O(V + E), Память: O(V + E)
Оба подхода обходят каждую вершину и ребро один раз — O(V+E). Обратный граф и массивы — O(V+E) памяти.

763 Partition Labels
Разбить строку на максимальное количество частей так, чтобы каждая буква встречалась не более чем в одной части. Вернуть размеры частей.
Заметим, что сами части могут быть мелкими, главное чтоб их было как можно больше. 
Для каждого символа запоминаем его последнюю позицию. 
Для каждого пройденного символа есть его максимальная позиция справа. Вот через Math.max мы ее узнаем. И когда текущая позиция равна этой максимальной - значит для всех пройденных символов дальше они не попадутся и это максимальная длина подоторезка
class Solution {
    public List<Integer> partitionLabels(String s) {
        int[] lastOccurrence = new int[26];
        for (int i = 0; i < s.length(); i++) {
            lastOccurrence[s.charAt(i) - 'a'] = i;
        }
        List<Integer> result = new ArrayList<>();
        //partitionStart — где начался текущий кусок
        int partitionStart = 0;
        int maxReach = 0;
        for (int i = 0; i < s.length(); i++) {
            maxReach = Math.max(maxReach, lastOccurrence[s.charAt(i) - 'a']);
            if (i == maxReach) { //все символы, которые мы встретили в текущем куске, больше не встречаются дальше
                result.add(i - partitionStart + 1);
                partitionStart = i + 1;
            }
        }
        return result;
    }
}
Время: O(n), Память: O(1)
Два прохода: заполнение lastOccurrence[26] и проход по строке — O(n). Массив из 26 целых — O(1) памяти.
Разбиение на 3 части
Дан массив. Разбить его на 3 непустые части. Вернуть минимальную сумму первых элементов каждой части.
Первый элемент всегда начинает первую часть. Из позиций 1..n-1 выбираем два минимальных — это начала второй и третьей частей. O(n).
class Solution {
    public int minSumOfFirstElements(int[] arr) {
        int first = arr[0];
        int min1 = Integer.MAX_VALUE;
        int min2 = Integer.MAX_VALUE;
        for (int i = 1; i < arr.length; i++) {
            if (arr[i] < min1) {
                min2 = min1;
                min1 = arr[i];
            } else if (arr[i] < min2) {
                min2 = arr[i];
            }
        }
        return first + min1 + min2;
    }
}
Время: O(n), Память: O(1)
Один проход с обновлением двух минимумов — O(n). Только переменные — O(1) памяти.
Подотрезок с минимумом X
Найти самый длинный подотрезок массива, в котором минимум равен x. Вернуть длину подотрезка.
Все элементы в окне должны быть >= x, и хотя бы один == x. Элементы < x разрывают окно. Сдвигаем left и сбрасываем hasTarget. O(n).
class Solution {
    public int longestSubarrayWithMinX(int[] arr, int x) {
        int result = 0;
        int left = 0;
        boolean hasTarget = false;
        for (int right = 0; right < arr.length; right++) {
            if (arr[right] < x) {
                left = right + 1;
                hasTarget = false;
                continue;
            }
            if (arr[right] == x) hasTarget = true;
            if (hasTarget) {
                result = Math.max(result, right - left + 1);
            }
        }
        return result;
    }
}
Время: O(n), Память: O(1)
Один цикл со скользящим окном, каждый индекс обрабатывается константное число раз — O(n). Переменные — O(1).
LC 3105 Longest Strictly Increasing or Strictly Decreasing Subarray
Найти самый длинный строго монотонный (возрастающий или убывающий) подотрезок. Вернуть [start, end].
Ведём два счётчика: incLen и decLen. При смене направления или равных элементах сбрасываем. Запоминаем лучший результат. O(n).
import java.util.*;
class Solution {
    public static List<Integer> searchMonoton(List<Integer> nums) {
        int maxLen = 1, incLen = 1, decLen = 1;
        List<Integer> result = Arrays.asList(0, 0);
        for (int idx = 1; idx < nums.size(); idx++) {
            if (nums.get(idx - 1) < nums.get(idx)) {
                incLen++;
                decLen = 1;
            } else if (nums.get(idx - 1) > nums.get(idx)) {
                decLen++;
                incLen = 1;
            } else {
                incLen = 1;
                decLen = 1;
            }
            int currLen = Math.max(incLen, decLen);
            if (currLen > maxLen) {
                result = Arrays.asList(idx - currLen + 1, idx);
                maxLen = currLen;
            }
        }
        return result;
    }
}
Время: O(n), Память: O(1)
Один проход с обновлением счётчиков incLen/decLen — O(n). Несколько целых — O(1) памяти.
Расстояние между X и Y
Дана строка из символов X, Y и O. Найти кратчайшее расстояние между X и Y.
Минимальное расстояние всегда достигается парой соседних X и Y по позиции. Один проход, обновляем lastX/lastY и result. O(n).
class Solution {
    public int minDistanceXY(String s) {
        int lastX = -1, lastY = -1;
        int result = Integer.MAX_VALUE;
        for (int i = 0; i < s.length(); i++) {
            char c = s.charAt(i);
            if (c == 'X') {
                lastX = i;
                if (lastY != -1)
                    result = Math.min(result, i - lastY);
            } else if (c == 'Y') {
                lastY = i;
                if (lastX != -1)
                    result = Math.min(result, i - lastX);
            }
        }
        return result == Integer.MAX_VALUE ? -1 : result;
    }
}
Время: O(n), Память: O(1)
Один цикл по строке с обновлением lastX/lastY — O(n). Несколько индексов — O(1) памяти.
Половины с разницей ≤ k
Дан массив целых чисел nums чётной длины 2*n и неотрицательное число k. Нужно переставить элементы массива так, чтобы в обеих половинах массива для всех пар индексов i, j выполнялось неравенство |nums[i] - nums[j]| <= k (попарная разница элементов в обеих половинах массива была не больше чем k). Если подходящих перестановок несколько, можно вернуть любую. Если перестановка невозможна, вернуть пустой массив.
Или, что то же самое: max(половины) - min(половины) ≤ k.
Сортируем массив. Распределяем по группам: близкие к min, близкие к max, и промежуточные. Каждая половина отсортированного массива автоматически удовлетворяет условию разницы ≤ k.
class Solution {
    public int[] rearrangeHalves(int[] arr, int k) {
        Arrays.sort(arr);
        int n = arr.length;
        int half = n / 2;
        List<Integer> lowGroup = new ArrayList<>();
        List<Integer> highGroup = new ArrayList<>();
        List<Integer> midGroup = new ArrayList<>();
        int minVal = arr[0];
        int maxVal = arr[n - 1];
        for (int val : arr) {
            if (val <= minVal + k) {
                lowGroup.add(val);
            } else if (val >= maxVal - k) {
                highGroup.add(val);
            } else {
                midGroup.add(val);
            }
        }
        int[] result = new int[n];
        int idx = 0;
        for (int v : lowGroup) result[idx++] = v;
        for (int v : midGroup) result[idx++] = v;
        for (int v : highGroup) result[idx++] = v;
        return result;
    }
}
Время: O(n log n), Память: O(n)
Arrays.sort — O(n log n), затем линейное распределение по группам. Три списка суммарно n элементов — O(n) памяти.
Оптимизация маршрута
Дан путь из команд U/D/L/R (вверх/вниз/лево/право). Удалить все петли (когда возвращаемся в уже посещённую точку).
Отслеживаем посещённые точки. При повторном посещении разматываем путь (удаляем координаты и команды) до точки совпадения — это и есть петля. O(n) амортизировано.
class Solution {
    public List<String> optimizeRoute(String[] moves) {
        List<String> result = new ArrayList<>();
        List<int[]> coords = new ArrayList<>();
        Set<String> visited = new HashSet<>();
        int x = 0, y = 0;
        coords.add(new int[]{x, y});
        visited.add(x + "," + y);
        for (String move : moves) {
            switch (move) {
                case "U": y++; break;
                case "D": y--; break;
                case "L": x--; break;
                case "R": x++; break;
            }
            String key = x + "," + y;
            if (visited.contains(key)) {
                while (true) {
                    int[] last = coords.get(coords.size() - 1);
                    String lastKey = last[0] + "," + last[1];
                    if (lastKey.equals(key)) break;
                    visited.remove(lastKey);
                    coords.remove(coords.size() - 1);
                    result.remove(result.size() - 1);
                }
            } else {
                visited.add(key);
                coords.add(new int[]{x, y});
                result.add(move);
            }
        }
        return result;
    }
}
Время: O(n), Память: O(n)
Каждый добавленный шаг может быть удалён не более одного раза — амортизированно O(n). coords, visited, result — O(n) памяти.
Maximize Distance to Closest Person
Дан массив seats из 0 и 1, где 1 — занятое место. Найти максимальное расстояние до ближайшего занятого места, если сесть на оптимальное свободное.
O(n) один проход. Три случая: начальные нули (расстояние = позиция первой 1), промежуток между двумя 1 (расстояние = gap/2), конечные нули (расстояние = n-1-lastOne).
class Solution {
    public int maxDistToClosest(int[] seats) {
        int n = seats.length;
        int prev = -1;
        int best = 0;
        for (int i = 0; i < n; i++) {
            if (seats[i] == 1) {
                if (prev == -1) {
                    best = i;
                } else {
                    best = Math.max(best, (i - prev) / 2);
                }
                prev = i;
            }
        }
        best = Math.max(best, n - 1 - prev);
        return best;
    }
}
Время: O(n), Память: O(1)
Один линейный проход по seats — O(n). Константные переменные — O(1) памяти.
LC 49. Group Anagrams
Дан массив строк strs. Сгруппировать все анаграммы вместе.
Анаграммы — слова с одинаковыми символами в разном порядке.
Если сортировать каждое слово, тоже можно группировать анаграммы, но код идёт от более прямой идеи. У анаграмм одинаков не порядок символов, а их количества. Значит, для каждого слова можно построить канонический ключ по частотам букв и уже по нему складывать строки в HashMap. Тогда все слова с одинаковым набором букв автоматически оказываются в одной группе.
class Solution {
    public List<List<String>> groupAnagrams(String[] strs) {
        Map<String, List<String>> map = new HashMap<>();
        for (String s : strs) {
            int[] cnt = new int[26];
            for (int i = 0; i < s.length(); i++) {
                cnt[s.charAt(i) - 'a']++;
            }
            String key = buildKey(cnt);
            map.computeIfAbsent(key, k -> new ArrayList<>()).add(s);
        }
        return new ArrayList<>(map.values());
    }
    private String buildKey(int[] cnt) {
        StringBuilder sb = new StringBuilder(26 * 2);
        for (int x : cnt) {
            sb.append('#').append(x);
        }
        return sb.toString();
    }
}
Время: O(n·L), Память: O(n·L); L — макс. длина слова; на слово: массив cnt[26] (|Σ|=26 латинских букв — константа алфавита) + сборка ключа O(26)
Суммарная длина всех строк O(n·L); на каждое слово линейный проход по символам. Память: карта групп хранит все строки входа и ключи.
Слово-анаграмма
Проверить, является ли строка t анаграммой строки s.
Частотный подсчёт. Если все счётчики равны нулю — строки являются анаграммами. O(n) время, O(1) память.
class Solution {
    public boolean isAnagram(String s, String t) {
        if (s.length() != t.length()) return false;
        int[] count = new int[26];
        for (int i = 0; i < s.length(); i++) {
            count[s.charAt(i) - 'a']++;
            count[t.charAt(i) - 'a']--;
        }
        for (int c : count) {
            if (c != 0) return false;
        }
        return true;
    }
}
Время: O(n), Память: O(1)
Один проход по символам с инкрементами в count[26] — O(n). Массив из 26 элементов — O(1) памяти.
205 Isomorphic Strings
Проверить, являются ли строки s и t изоморфными. Каждый символ s можно заменить на символ t с сохранением порядка. Биекция: разные символы не могут отображаться в один.
Первая мысль — изоморфизм это биекция. Каждый символ из s должен однозначно отображаться в символ из t, и обратно. Значит нужно хранить два маппинга: s→t и t→s.
Наивно — две HashMap. Но символы это просто числа от 0 до 255, поэтому HashMap избыточен — заменяем на два массива размера 256. Это O(1) память по факту, константа.
Теперь вопрос как хранить. Первая идея — хранить сам символ в который отображаемся, заполнить массивы -1 как sentinel. Но тут я подумал: а зачем хранить символ? Достаточно хранить когда последний раз видели каждый символ. Если s[i] и t[i] оба встречались впервые — окей, запоминаем. Если оба встречались раньше — они должны были встретиться на одном и том же шаге, иначе маппинг конфликтует.
Это одна проверка вместо двух, и Arrays.fill не нужен — массивы инициализируются нулями, а мы пишем i+1 чтобы отличить "не видели" от "видели на шаге 0".
Итого: один проход O(n), два массива O(1), никаких вспомогательных структур.
class Solution {
    public boolean isIsomorphic(String s, String t) {
        int[] lastSeenS = new int[256];
        int[] lastSeenT = new int[256];
        for (int i = 0; i < s.length(); i++) {
            if (lastSeenS[s.charAt(i)] != lastSeenT[t.charAt(i)]) {
                return false;
            }
            lastSeenS[s.charAt(i)] = i + 1;  // +1 чтобы 0 означал "не видели"
            lastSeenT[t.charAt(i)] = i + 1;
        }
        return true;
    }
}
Время: O(n), Память: O(1)
Один цикл по позициям строки с проверкой пар в массивах — O(n). Два массива на 256 целых — O(1) памяти.
Удаление лишних дубликатов
Дан массив и число n. Оставить не более n вхождений каждого элемента.
Хотим оставить не более n копий каждого элемента. Один проход по arr: смотрим count[val] — сколько раз этот элемент уже добавлен. Если count < n — добавляем в результат и увеличиваем счётчик. Иначе — пропускаем. O(n) время и память.
class Solution {
    public List<Integer> removeDuplicates(int[] arr, int n) {
        Map<Integer, Integer> count = new HashMap<>();
        List<Integer> result = new ArrayList<>();
        for (int val : arr) {
            int cnt = count.getOrDefault(val, 0);
            if (cnt < n) {
                result.add(val);
                count.put(val, cnt + 1);
            }
        }
        return result;
    }
}
Время: O(n), Память: O(u)
Один проход по массиву с O(1) операциями HashMap на элемент — O(n). Карта до u различных ключей — O(u) памяти.
Перестановка букв и палиндром
Можно ли переставить символы строки так, чтобы получился палиндром?
Цикл по строке считает частоту каждого символа (массив count[26]). Затем считаем количество символов с нечётной частотой (oddCount). Палиндром допускает максимум один такой символ (центр строки нечётной длины) → если oddCount ≤ 1, вернуть true. O(n) время, O(1) память.
class Solution {
    public boolean canFormPalindrome(String s) {
        int[] count = new int[26];
        for (char c : s.toCharArray()) {
            count[c - 'a']++;
        }
        int oddCount = 0;
        for (int c : count) {
            if (c % 2 != 0) oddCount++;
        }
        return oddCount <= 1;
    }
}
Время: O(n), Память: O(1)
Цикл по строке считает частоты за O(n), затем 26 проверок. Массив count[26] — O(1) памяти.
Маршрут туриста
Даны авиабилеты в виде пар [откуда, куда]. Восстановить маршрут.
По коду видно, что здесь не нужен ни DFS, ни перебор вариантов: каждый билет задаёт единственный переход from -> to, то есть весь маршрут на самом деле является обычной цепочкой. Тогда естественно хранить HashMap "откуда -> куда" и просто идти по ней. Осталось понять старт: это единственный город, который встречается как from, но ни разу не встречается как destination. После этого маршрут восстанавливается простым проходом по цепочке.
class Solution {
    public List<String> reconstructRoute(String[][] tickets) {
        // Шаг 1: хранилище всех пунктов назначения
        Set<String> destinations = new HashSet<>();
        // Шаг 2: маппинг "откуда → куда"
        Map<String, String> mapping = new HashMap<>();
        // Заполняем структуры
        for (String[] ticket : tickets) {
            mapping.put(ticket[0], ticket[1]);  // откуда → куда
            destinations.add(ticket[1]);        // запоминаем все пункты назначения
        }
        // Шаг 3: находим стартовую точку
        // Ищем город, который есть в "откуда", но НЕТ в "куда"
        String start = null;
        for (String[] ticket : tickets) {
            if (!destinations.contains(ticket[0])) {
                start = ticket[0];
                break;
            }
        }
        // Шаг 4: восстанавливаем маршрут
        List<String> route = new ArrayList<>();
        String current = start;
        while (current != null) {
            route.add(current);
            current = mapping.get(current);  // переходим к следующему городу
        }
        return route;
    }
}
Время: O(n) (n — число билетов; построение карт + проход цепочки), Память: O(n) (mapping, множество назначений, маршрут)
Два прохода по билетам + цикл по цепочке — O(n). HashMap, HashSet и список маршрута — O(n) памяти.
Билеты по маршруту
Задача: Восстановить цепочку маршрута по перепутанным билетам.
Входные данные:
tickets — неотсортированный список билетов. Каждый билет представлен парой [Откуда, Куда].
start — название города, из которого начинается маршрут.
Гарантии (свойства маршрута):
Маршрут складывается в одну идеальную линию: от start до конечной точки.
Нет развилок (из любого города есть максимум один вылет).
Нет петель (вы никогда не возвращаетесь в город, где уже были).
Что нужно сделать:
Вернуть список билетов, выстроенный в правильном порядке следования.
Здесь код исходит из того, что branching нет: из каждого города есть не набор вариантов, а максимум один следующий билет. Поэтому вместо графового обхода достаточно собрать карту byFrom и дальше идти от заданного start по цепочке перелётов. На каждом шаге берём билет из текущего города, добавляем его в ответ и переходим в следующий город. Получается простой линейный проход по уже заданному маршруту.
class Solution {
    public List<List<String>> orderTickets(List<List<String>> tickets, String start) {
        Map<String, List<String>> byFrom = new HashMap<>();
        for (List<String> t : tickets) {
            byFrom.put(t.get(0), t);
        }
        List<List<String>> ordered = new ArrayList<>();
        String city = start;
        while (byFrom.containsKey(city)) {
            List<String> t = byFrom.get(city);
            ordered.add(t);
            city = t.get(1);
        }
        return ordered;
    }
}
Время: O(n) (n — число билетов; построение карты from→билет + проход цепочки ≤ n шагов), Память: O(n) (HashMap — до n пар «город → ссылка на тот же List из tickets»; список ordered — n ссылок на те же внутренние списки, новых списков-билетов не создаём)
Каждый билет один раз кладём в map. Цикл по цепочке не длиннее n. Доп. память: карта и список указателей на исходные List<String>.
LeetCode 347 Top K Frequent Elements
Дан массив целых чисел nums и число k. Вернуть k наиболее часто встречающихся элементов. Гарантируется, что ответ уникален. Ответ можно вернуть в любом порядке.
Шаг 1: один проход по nums — считаем частоты в HashMap. Шаг 2: для каждой пары (элемент, частота) добавляем в min-heap (отсортирован по частоте). Если размер кучи > k — удаляем минимальный элемент. В итоге в куче остаются ровно k самых частых. O(n log k) время, O(n + k) память.
class Solution {
    public int[] topKFrequent(int[] nums, int k) {
        Map<Integer, Integer> frequencyMap = new HashMap<>();
        for (int num : nums) {
            frequencyMap.merge(num, 1, Integer::sum);
        }
        PriorityQueue<Map.Entry<Integer, Integer>> minHeap = new PriorityQueue<>(
                Comparator.comparingInt(Map.Entry::getValue));
        for (Map.Entry<Integer, Integer> entry : frequencyMap.entrySet()) {
            minHeap.offer(entry);
            if (minHeap.size() > k) {
                minHeap.poll();
            }
        }
       int[] result = new int[k];
       for (int i = k - 1; i >= 0; i--) {
           result[i] = minHeap.poll().getKey();
       }
       return result;
    }
}
Время: O(n log k), Память: O(n + k)
Подсчёт частот O(n), затем для каждого элемента push/pop в min-heap размера k — O(log k). Итого O(n log k). Карта O(n), куча O(k).
Top K Frequent Words
Дан массив строк words и число k. Найти k самых частых слов. При равной частоте — лексикографический порядок.
Шаг 1: один проход по words — считаем частоты в HashMap. Шаг 2: min-heap с компаратором: наверху — наименее частое слово, при равной частоте — лексикографически большее (чтобы первыми вытеснять «худшие»). Если размер > k — poll(). При сборе результата переворачиваем список. O(n log k) время.
class Solution {
    public List<String> topKFrequent(String[] words, int k) {
        Map<String, Integer> freqMap = new HashMap<>();
        for (String w : words) {
            freqMap.merge(w, 1, Integer::sum);
        }
        PriorityQueue<Map.Entry<String, Integer>> minHeap =
            new PriorityQueue<>((a, b) -> {
                if (!a.getValue().equals(b.getValue())) {
                    return a.getValue() - b.getValue();
                }
                return b.getKey().compareTo(a.getKey());
            });
        for (var entry : freqMap.entrySet()) {
            minHeap.offer(entry);
            if (minHeap.size() > k) {
                minHeap.poll();
            }
        }
        List<String> result = new ArrayList<>();
        while (!minHeap.isEmpty()) {
            result.add(minHeap.poll().getKey());
        }
        Collections.reverse(result);
        return result;
    }
}
Время: O(n log k), Память: O(n + k)
Частоты слов O(n), затем операции с кучей размера k — O(n log k). freqMap O(n), куча O(k).
LC 215 · Kth Largest Element in an Array
Дан массив nums и число k. Найти k-й по величине элемент (не k-й уникальный).
k-й по величине — это минимум из k наибольших элементов. Инвариант: min-heap всегда хранит ровно k наибольших из просмотренных. Для каждого num: добавляем в кучу, если size > k — poll() (выталкиваем наименьший). В конце peek() — это и есть k-й по величине. O(n log k) время, O(k) память.
class Solution {
    public int findKthLargest(int[] nums, int k) {
        PriorityQueue<Integer> minHeap = new PriorityQueue<>();
        for (int num : nums) {
            minHeap.offer(num);
            if (minHeap.size() > k) {
                minHeap.poll();
            }
        }
        return minHeap.peek();
    }
}
Время: O(n log k), Память: O(k)
Для каждого элемента делаем вставку в кучу и иногда удаление минимума, обе операции O(log k). Размер кучи ограничен k.
Merge Intervals
Дан список отрезков [начало, конец]. Необходимо объединить все пересекающиеся отрезки.
Ключевая мысль здесь не в самом merge, а в том, что без сортировки вообще непонятно, с чем сравнивать текущий интервал. Как только интервалы упорядочены по началу, достаточно держать один "текущий" отрезок. Если следующий начинается уже после его конца, пересечения нет и текущий можно зафиксировать в ответе. Если пересечение есть, мы просто расширяем правую границу текущего интервала.
class Solution {
    public int[][] merge(int[][] intervals) {
        Arrays.sort(intervals, Comparator.comparingInt(a -> a[0]));
        List<int[]> merged = new ArrayList<>();
        int currentStart = intervals[0][0];
        int currentEnd   = intervals[0][1];
        for (int i = 1; i < intervals.length; i++) {
            int nextStart = intervals[i][0];
            int nextEnd   = intervals[i][1];
            if (currentEnd < nextStart) { //нет пересечения
                merged.add(new int[]{currentStart, currentEnd});
                currentStart = nextStart;
                currentEnd   = nextEnd;
            } else {
                currentEnd = Math.max(currentEnd, nextEnd);
            }
        }
        merged.add(new int[]{currentStart, currentEnd});
        return merged.toArray(new int[merged.size()][]);
    }
}
Время: O(n log n) доминирует сортировка по left + O(n) слияние, Память: O(n) (список merged; стек сортировки in-place O(log n), обычно не указывают отдельно)
Arrays.sort по началу интервала — O(n log n). Один проход слияния — O(n). Результат до n интервалов.
Пересекающиеся отрезки
Найти все отрезки, которые пересекаются хотя бы с одним другим.
Задача про пересечение отрезков естественно решается после СОРТИРОВКИ по началу.
После сортировки проверяем: если текущий start ≤ предыдущий bestEnd, оба отрезка пересекаются. Помечаем флагами.
import java.util.*;
public class Solution {
    private static boolean isOverlapping(List<Integer> a, List<Integer> b) {
        // Проверяет, пересекаются ли два отрезка
        return Math.max(a.get(0), b.get(0)) <= Math.min(a.get(1), b.get(1));
    }
    public static List<List<Integer>> findOverlappings(List<List<Integer>> segments) {
        segments.sort(Comparator.comparingInt((List<Integer> x) -> x.get(0)).thenComparingInt(x -> x.get(1)));
        Set<List<Integer>> result = new HashSet<>();
        int maxEndIndex = 0;
        for (int i = 1; i < segments.size(); i++) {
            if (isOverlapping(segments.get(i), segments.get(maxEndIndex))) {
                result.add(segments.get(i));
                result.add(segments.get(maxEndIndex));
            }
            if (segments.get(i).get(1) > segments.get(maxEndIndex).get(1)) {
                maxEndIndex = i;
            }
        }
        return new ArrayList<>(result);
    }
}
Время: O(n log n), Память: O(n)
Сортировка индексов — O(n log n), линейный проход с массивом флагов — O(n). Массив idx и boolean[] — O(n) памяти.
Car Pooling
Автомобиль с вместимостью capacity. Список поездок [passengers, from, to]. Можно ли выполнить все поездки?
Вместо того чтобы следить за каждой поездкой отдельно, используем разностный массив: на остановке from люди садятся (+passengers), на to — выходят (-passengers). Затем один проход — префиксная сумма: текущая загрузка = накопленная сумма изменений. Если в любой точке > capacity → false. O(n + maxLocation) время, O(1) доп. память.
class Solution {
    public boolean carPooling(int[][] trips,
                              int capacity) {
        int[] passengerChanges = new int[1001];
        for (int[] trip : trips) {
            int passengers = trip[0];
            int from = trip[1];
            int to = trip[2];
            passengerChanges[from] += passengers;
            passengerChanges[to] -= passengers;
        }
        int current = 0;
        for (int change : passengerChanges) {
            current += change;
            if (current > capacity) return false;
        }
        return true;
    }
}
Время: O(n + U), Память: O(U)
Один проход по trips для массива разностей длины U — O(n+U). Массив passengerChanges — O(U) памяти.
Meeting Rooms II
Дан список интервалов совещаний. Найти минимальное количество переговорных комнат.
Ключевой вопрос: нужна ли новая комната или можно занять освободившуюся? Нам не важно КАКАЯ встреча заканчивается — важно лишь: закончилась ли хоть одна к моменту starts[i]? Сортируем starts и ends по отдельности. Два указателя: если starts[i] >= ends[j] — одна встреча уже закончилась, комнату переиспользуем (rooms--, j++). Иначе — нужна новая (rooms++). Запоминаем максимум. O(n log n).
public int minMeetingRooms(int[][] intervals) {
    int n = intervals.length;
    if (n == 0) return 0;
    int[] starts = new int[n];
    int[] ends = new int[n];
    for (int i = 0; i < n; i++) {
        starts[i] = intervals[i][0];
        ends[i] = intervals[i][1];
    }
    Arrays.sort(starts);
    Arrays.sort(ends);
    int rooms = 0;
    int maxRooms = 0;
    int endIdx = 0;  // указатель на ends
    for (int startIdx = 0; startIdx < n; startIdx++) {
        // СНАЧАЛА освобождаем ВСЕ комнаты, закончившиеся ДО starts[startIdx]
        while (endIdx < n && starts[startIdx] >= ends[endIdx]) {
            rooms--;
            endIdx++;
        }
        // ПОТОМ выделяем новую для этой встречи
        rooms++;
        maxRooms = Math.max(maxRooms, rooms);
    }
    return maxRooms;
}
Время: O(n log n), Память: O(n)
Сортировка двух массивов starts/ends — O(n log n). Два указателя — O(n). Два массива — O(n) памяти.
LC 206 · Reverse Linked List
Дан head односвязного списка. Перевернуть список и вернуть его новый head.
Три указателя: prev = null, curr = head. Цикл: сохраняем next = curr.next, разворачиваем curr.next = prev, затем сдвигаем: prev = curr, curr = next. Когда curr == null — prev указывает на новый head. O(n) время, O(1) память.
class Solution {
    public ListNode reverseList(ListNode head) {
        ListNode prev = null;
        ListNode curr = head;
        while (curr != null) {
            ListNode tmp = curr.next;  // 1. сохраняем следующий
            curr.next = prev;          // 2. разворачиваем стрелку
            prev = curr;               // 3. сдвигаем prev
            curr = tmp;                // 4. сдвигаем curr
        }
        return prev;  // новая голова
    }
}
Время: O(n), Память: O(1)
Один проход по списку с перестановкой next — O(n). Три указателя — O(1) памяти.
Удалите N-й узел с конца
Дан связный список. Удалить n-й узел с конца. Вернуть head.
Если смотреть на задачу правильно, удалять нужно не сам N-й узел с конца, а иметь доступ к узлу перед ним, чтобы перекинуть ссылку. Поэтому код разводит fast и slow так, чтобы к моменту, когда fast дойдёт до конца списка, slow оказался ровно перед удаляемым элементом. Для этого fast стартует с dummy и уходит вперёд на n+1 шагов. Dummy здесь важен отдельно: он делает удаление головы обычным случаем, без специальной ветки в коде.
class Solution {
    public ListNode removeNthFromEnd(ListNode head, int n) {
        ListNode dummy = new ListNode(0);
        dummy.next = head;
        ListNode fast = dummy;
        ListNode slow = dummy;
        // Шаг 1: Создаем отставание в n+1 шагов
        for (int i = 0; i <= n; i++) {
            fast = fast.next;
        }
        while (fast != null) {
            fast = fast.next;
            slow = slow.next;
        }
        slow.next = slow.next.next;
        return dummy.next;
    }
}
Время: O(n) (один проход списка), Память: O(1) доп. (указатели fast, slow и dummy-узел — фиксированное число ссылок)
fast уходит на n+1 шагов, затем оба идут до конца — O(n). Константные указатели — O(1) памяти.
Умножение длинного числа
Число хранится как массив цифр в обратном порядке. Умножить его на цифру n (1-9). Вернуть результат в том же формате.
Умножение столбиком: цифра × n + carry. Остаток от 10 — текущая цифра, целая часть — перенос. O(n) время.
class Solution {
    public List<Integer> multiplyByDigit(List<Integer> num, int n) {
        List<Integer> result = new ArrayList<>();
        int carry = 0;
        for (int digit : num) {
            int product = digit * n + carry;
            result.add(product % 10);
            carry = product / 10;
        }
        if (carry > 0) {
            result.add(carry);
        }
        return result;
    }
}
Время: O(m), Память: O(m)
Один цикл по m цифрам с carry — O(m). Список result до m+1 цифр — O(m) памяти.
Сумма hex чисел
Сложить два шестнадцатеричных числа, представленных строками. Вернуть результат как hex-строку.
Аналогично десятичному сложению, но в системе 16. Справа налево: складываем цифры + carry. Остаток от 16 — цифра, целая часть — перенос. O(max(a,b)).
public String addHex(String a, String b) {
    // Обработка null и пустых строк
    if (a == null) a = "";
    if (b == null) b = "";
    int i = a.length() - 1;
    int j = b.length() - 1;
    int carry = 0;
    StringBuilder result = new StringBuilder();
    while (i >= 0 || j >= 0 || carry > 0) {
        int sum = carry 
        + (i >= 0 ? hexToInt(a.charAt(i--)) : 0)
        + (j >= 0 ? hexToInt(b.charAt(j--)) : 0);
        result.append(intToHex(sum % 16));
        carry = sum / 16;
    }
    // Результат получился в обратном порядке, переворачиваем
    return result.reverse().toString();
}
private int hexToInt(char c) {
    if (c >= '0' && c <= '9') return c - '0';
    if (c >= 'a' && c <= 'f') return c - 'a' + 10;
    if (c >= 'A' && c <= 'F') return c - 'A' + 10;
    throw new IllegalArgumentException("Invalid hex char: " + c);
}
private char intToHex(int n) {
    if (n >= 0 && n <= 9) return (char) (n + '0');
    if (n >= 10 && n <= 15) return (char) (n - 10 + 'a');
    throw new IllegalArgumentException("Invalid hex digit: " + n);
}
Время: O(max(|a|, |b|)), Память: O(max(|a|, |b|))
Один проход с конца строк с carry — O(max(|a|,|b|)). StringBuilder длины результата — O(max(|a|,|b|)) памяти.
Минимальное произведение
Найти минимальное произведение двух элементов массива.
Минимальное произведение пары — один из трёх кандидатов: min1*min2 (два наименьших, оба отрицательных → положительный), min1*max1 (наименьшее × наибольшее → наибольшее по модулю отрицательное), max1*max2 (два наибольших положительных). За один проход находим min1, min2 и max1, max2. Ответ = min(candidate1, candidate2, candidate3).
class Solution {
    public long minProduct(int[] arr) {
        int min1 = Integer.MAX_VALUE;
        int min2 = Integer.MAX_VALUE;
        int max1 = Integer.MIN_VALUE;
        int max2 = Integer.MIN_VALUE;
        for (int val : arr) {
            if (val < min1) {
                min2 = min1;
                min1 = val;
            } else if (val < min2) {
                min2 = val;
            }
            if (val > max1) {
                max2 = max1;
                max1 = val;
            } else if (val > max2) {
                max2 = val;
            }
        }
        long candidate1 = (long) min1 * min2;
        long candidate2 = (long) min1 * max1;
        long candidate3 = (long) max1 * max2;
        return Math.min(candidate1,
            Math.min(candidate2, candidate3));
    }
}
Время: O(n), Память: O(1)
Один проход с обновлением двух min и двух max — O(n). Только переменные — O(1) памяти.
Индекс Хирша
Дан массив цитирований. Найти h-индекс: максимальное h, при котором h статей имеют ≥ h цитирований.
O(n) с bucket sort. Создаём buckets[0..n], где buckets[i] = количество статей с i цитированиями (или >= n). Суммируем справа налево: когда count >= i, нашли h-индекс.
class Solution {
    public int hIndex(int[] citations) {
        int n = citations.length;
        int[] buckets = new int[n + 1];
        for (int c : citations) {
            if (c >= n) {
                buckets[n]++;
            } else {
                buckets[c]++;
            }
        }
        int count = 0;
        //Идем с конца к началу, накапливая количество статей
        for (int i = n; i >= 0; i--) {
            count += buckets[i];
            if (count >= i) {
                return i;
            }
        }
        return 0;
    }
}
Время: O(n), Память: O(n)
Один проход для buckets размера n+1, второй — накопление справа — O(n). Массив buckets — O(n) памяти.
LC 560 Subarray Sum Equals K
Дан массив целых чисел nums и число k. Найти общее количество непрерывных подмассивов, сумма элементов которых равна k.
Наблюдение: сумма nums[i+1..j] = prefix[j] - prefix[i] = k ↔ prefix[i] = currentSum - k. Для каждого j проверяем: сколько раз значение (currentSum - k) уже встречалось — столько подмассивов оканчивается здесь с нужной суммой. map.put(0, 1) в начале — чтобы поймать подмассивы с самого нулевого индекса. Текущий префикс добавляем в карту ПОСЛЕ проверки. O(n) время, O(n) память.
class Solution {
    public int subarraySum(int[] nums, int k) {
        Map<Integer, Integer> prefixSumCount = new HashMap<>();
        prefixSumCount.put(0, 1);
        int count = 0;
        int currentSum = 0;
        for (int num : nums) {
            currentSum += num;
            count += prefixSumCount.getOrDefault(currentSum - k, 0);
            prefixSumCount.merge(currentSum, 1, Integer::sum);
        }
        return count;
    }
}
Время: O(n), Память: O(n)
Один проход: обновляем currentSum и проверяем HashMap за O(1) в среднем — O(n). Карта до n различных сумм — O(n) памяти.
Последовательность с суммой K
Дан неотсортированный массив nums целых чисел, необходимо определить, существует ли такая непрерывная последовательность, сумма элементов которой равна k (target).
Нужно вернуть индекс последнего элемента первой встретившейся последовательности, иначе -1.
Та же идея что ps1: сумма nums[i+1..j] = k ↔ prefix[i] = currentSum - k. HashSet хранит все предыдущие префиксы. prefixSums.add(0) в начале — для подмассивов с индекса 0. Если (currentSum - k) есть в сете — нашли правую границу, возвращаем i. Текущий префикс добавляем в сет ПОСЛЕ проверки. O(n) время и память.
class Solution {
    public int findSubarrayEnd(int[] nums, int k) {
        Set<Integer> prefixSums = new HashSet<>();
        prefixSums.add(0);
        int currentSum = 0;
        for (int i = 0; i < nums.length; i++) {
            currentSum += nums[i];
            if (prefixSums.contains(currentSum - k)) {
                return i;  
            }
            prefixSums.add(currentSum);
        }
        return -1;
    }
}
Время: O(n), Память: O(n)
Линейный проход с currentSum и проверкой HashMap — O(n). Карта «сумма → индекс» до O(n) записей.
Последовательность с суммой K — диапазон индексов
Дан неотсортированный массив nums целых чисел и число k. Необходимо найти непрерывный подмассив, сумма элементов которого равна k.
Нужно вернуть список из двух элементов — индексы начала и конца любого подходящего подмассива (включительно). Если такого подмассива не существует, вернуть [−1, −1].
Сумма nums[l..r] = prefix[r] − prefix[l−1]. Ищем в карте prefix[r]−k = префикс до l−1. Индекс начала: сохранённый индекс + 1. putIfAbsent даёт самое раннее начало при нескольких вариантах. O(n) время и память.
import java.util.*;
class Solution {
    public List<Integer> subsequenceSumK(List<Integer> nums, int k) {
        Map<Integer, Integer> prefixIndex = new HashMap<>();
        prefixIndex.put(0, -1);
        int prefixSum = 0;
        for (int idx = 0; idx < nums.size(); idx++) {
            prefixSum += nums.get(idx);
            int diff = prefixSum - k;
            if (prefixIndex.containsKey(diff)) {
                return Arrays.asList(prefixIndex.get(diff) + 1, idx);
            }
            prefixIndex.putIfAbsent(prefixSum, idx);
        }
        return Arrays.asList(-1, -1);
    }
}
Время: O(n), Память: O(n)
Один проход с обновлением prefixSum и проверкой HashMap — O(n) в среднем. Карта хранит до O(n) различных префиксных сумм — O(n) памяти.
LC 523. Continuous Subarray Sum
Дан массив целых чисел nums и целое число k. Вернуть true, если существует непрерывный подмассив длиной не менее 2, сумма которого кратна k.
Если у нас есть два префикса с ОДИНАКОВЫМ остатком от деления на k, то сумма элементов между ними КРАТНА k.
Три важных нюанса:
1) map.put(0, -1) — позволяет поймать случай, когда префикс с индекса 0 уже кратен k.
2) i - map.get(prefix) >= 2 — длина подмассива должна быть минимум 2.
3) map.put(prefix, i) только в ветке else — сохраняем первое вхождение остатка, чтобы не терять максимально ранний индекс.
class Solution {
    public boolean checkSubarraySum(int[] nums, int k) {
        // Если у нас есть два префикса с ПРОСТО ОДИНАКОВЫМ остатком от деления на k,
        // то сумма элементов между ними КРАТНА k.
        Map<Integer, Integer> map = new HashMap<>();
        // Нюанс 1: кладём фиктивный префикс с остатком 0 на индексе -1
        // Это позволяет поймать случай, когда сам подмассив от 0 до i кратен k
        map.put(0, -1);
        int prefix = 0;  // текущий остаток от деления суммы префикса на k
        for (int i = 0; i < nums.length; i++) {
            // Накопливаем сумму и берём остаток (для больших чисел и отрицательного k)
            prefix = (prefix + nums[i]) % k;
            // Если такой остаток уже встречался...
            if (map.containsKey(prefix)) {
                if (i - map.get(prefix) >= 2) {
                    return true;
                }
                // Если длина < 2, ничего не делаем — продолжаем искать дальше
            } else {
                // сохраняем ТОЛЬКО первое вхождение остатка, так как НАМ ТОЛЬКО ВАЖНО ТРУ ИЛИ ФОЛС
                // Потому что чем раньше индекс, тем больше шансов получить длину >=2
                map.put(prefix, i);
            }
        }
        return false;  // ничего не нашли
    }
}
Время: O(n), Память: O(k)
Один проход по nums — O(n). В HashMap храним первое вхождение остатка prefix % k; число разных остатков ограничено k (и не больше n), поэтому O(k).
Пересечение множеств префиксов двух массивов
Даны два массива одинаковой длины N. Для каждого индекса i от 0 до N−1 нужно найти размер пересечения множеств значений в префиксах A[0..i] и B[0..i] (каждый префикс рассматриваем как множество **уникальных** чисел, без кратности).
Инвариант: common = |inA ∩ inB|. При добавлении нового значения в одно множество, если оно уже было в другом, размер пересечения увеличивается на 1. Дубликаты внутри одного массива не увеличивают множество повторно.
import java.util.*;
class Solution {
    public int[] prefixIntersectionSizes(int[] a, int[] b) {
        int n = a.length;
        Set<Integer> inA = new HashSet<>();
        Set<Integer> inB = new HashSet<>();
        int[] ans = new int[n];
        int common = 0;
        for (int i = 0; i < n; i++) {
            if (!inA.contains(a[i]) && inB.contains(a[i])) {
                common++;
            }
            inA.add(a[i]);
            if (!inB.contains(b[i]) && inA.contains(b[i])) {
                common++;
            }
            inB.add(b[i]);
            ans[i] = common;
        }
        return ans;
    }
}
Время: O(n) в среднем, Память: O(n)
На каждом из n шагов константное число операций с HashSet — O(n) амортизированно. В худшем случае в множествах до O(n) различных значений — O(n) памяти.
Пересечение двух массивов с учётом повторений
Найти пересечение двух массивов с учётом повторений (intersection with duplicates).
Если элемент встречается в первом массиве 2 раза и во втором 2 раза — в ответе он тоже 2 раза. **Порядок элементов в ответе не важен.**
Для каждого значения v в пересечении берём min(частота в nums1, частота в nums2) вхождений. Реализация через уменьшение счётчика при совпадении из второго массива даёт ровно эту минимальную кратность.
import java.util.*;
class Solution {
    public int[] intersectWithDuplicates(int[] nums1, int[] nums2) {
        // Всегда хешируем меньший массив
        if (nums1.length > nums2.length) {
            return intersectWithDuplicates(nums2, nums1);
        }
        Map<Integer, Integer> freq = new HashMap<>();
        for (int x : nums1) {
            freq.merge(x, 1, Integer::sum);
        }
        List<Integer> out = new ArrayList<>();
        for (int x : nums2) {
            int c = freq.getOrDefault(x, 0);
            if (c > 0) {
                out.add(x);
                freq.put(x, c - 1);
            }
        }
        return out.stream().mapToInt(i -> i).toArray();
    }
}
Время: O(n + m), Память: O(min(n, m))
Подсчёт частот первого массива — O(n), проход по второму — O(m). В карте не больше уникальных значений меньшего массива — O(min(n,m)) памяти.
LC 238 · Product of Array Except Self
Дан массив nums. Вернуть массив, где каждый элемент — произведение всех кроме текущего. Без деления.
Первый проход слева направо: result[i] = произведение всех элементов левее i. Второй проход справа налево: накапливаем suffix = произведение всех правее i, умножаем result[i] *= suffix. Итог: result[i] = prefix * suffix = всё кроме i. O(n) время, O(1) доп. память.
class Solution {
    public int[] productExceptSelf(int[] nums) {
        int n = nums.length;
        int[] result = new int[n];
        // первый проход: result[i] = произведение всех элементов левее i
        // для i=0 левее ничего нет — нейтральный элемент умножения
        result[0] = 1;
        for (int i = 1; i < n; i++) {
            result[i] = result[i - 1] * nums[i - 1];
        }
        // второй проход: suffix накапливает произведение всех элементов правее i
        // умножаем на уже лежащий prefix — получаем произведение всех кроме i
        int suffix = 1;
        for (int i = n - 2; i >= 0; i--) {
            suffix *= nums[i + 1];
            result[i] *= suffix;
        }
        return result;
    }
}
Время: O(n), Память: O(1) доп.
Два прохода: слева префиксные произведения, справа суффикс в переменной — O(n). Кроме массива ответа — O(1) доп. памяти.
 LC 724 — Find Pivot Index
Найти индекс, где сумма слева равна сумме справа.
O(n) один проход (после подсчёта суммы). rightSum = totalSum - leftSum - nums[i]. Когда leftSum == rightSum — нашли pivot.
class Solution {
    public int pivotIndex(int[] nums) {
        int totalSum = 0;
        for (int num : nums) totalSum += num;
        int leftSum = 0;
        for (int i = 0; i < nums.length; i++) {
            int rightSum = totalSum - leftSum - nums[i];
            if (leftSum == rightSum) return i;
            leftSum += nums[i];
        }
        return -1;
    }
}
Время: O(n), Память: O(1)
Сумма всех элементов O(n), затем один проход с leftSum — O(n). Пара переменных — O(1) памяти.
Minimum Window Substring
Даны две строки s и t. Найти минимальную по длине подстроку, которая содержит все символы строки t (включая дубликаты). Если такой подстроки нет — вернуть пустую строку "".
В лоб хотелось бы проверить все подстроки, но это слишком дорого. Здесь условие окна можно поддерживать на лету: freq показывает, каких символов нам ещё не хватает, а needCount хранит, сколько обязательных символов всё ещё не покрыто. Поэтому правую границу мы расширяем, пока не соберём валидное окно, а потом двигаем левую, чтобы выкинуть всё лишнее и оставить минимальный вариант. Такой инвариант и делает sliding window естественным решением.
class Solution {
    public String minWindow(String s, String t) {
        if (s == null || t == null || s.length() < t.length()) return "";
        // freq[c] > 0 — нужен, = 0 — покрыт, < 0 — лишний
        int[] freq = new int[128];
        for (char c : t.toCharArray()) freq[c]++;
        int needCount = t.length();  // сколько символов ещё нужно
        int bestStart = 0;
        int minLen = Integer.MAX_VALUE;
        int left = 0;
        for (int right = 0; right < s.length(); right++) {
            char rightChar = s.charAt(right);
            // Добавляем правый символ
            if (freq[rightChar] > 0) needCount--;
            freq[rightChar]--;
            // Пока всё покрыто — сжимаем окно слева
            while (needCount == 0) {
                int windowSize = right - left + 1;
                if (windowSize < minLen) {
                    minLen = windowSize;
                    bestStart = left;
                }
                // Удаляем левый символ
                char leftChar = s.charAt(left);
                freq[leftChar]++;
                if (freq[leftChar] > 0) needCount++;
                left++;
            }
        }
        return minLen == Integer.MAX_VALUE ? "" : s.substring(bestStart, bestStart + minLen);
    }
}
Время: O(|s|) (каждый индекс входит/выходит из окна ≤1 раз), Память: O(1) доп. (массив freq[128] под ASCII + целые left/right/best*; размер 128 не растёт с n)
Левый и правый указатели двигаются только вправо — O(|s|) амортизированно. Массив частот фиксированной длины (алфавит ASCII в условии) — O(1) памяти.
Longest Repeating Char Replacement
Дана строка s и число k. Можно заменить до k символов. Найти длину самой длинной подстроки из одинаковых символов.
Хотим найти окно, где заменой k символов можно получить строку из одинаковых. Количество нужных замен = длина окна − частота самого частого символа (windowLength − maxFreq). Если это значение > k — замен не хватает, сжимаем слева.
maxFreq специально не уменьшаем при сжатии: нас интересует только нахождение окна лучше текущего максимума. Если maxFreq недостижим при текущем left — окно просто не вырастет, но и не уменьшится. Результат берётся из window size, который монотонно растёт.
class Solution {
    public int characterReplacement(String s, int k) {
        int[] freq = new int[26];
        int maxFreq = 0;
        int left = 0;
        int result = 0;
        for (int right = 0; right < s.length(); right++) {
            // Расширяем окно вправо
            char c = s.charAt(right);
            freq[c - 'A']++;
            maxFreq = Math.max(maxFreq, freq[c - 'A']);
            // Проверяем, нужно ли сужать окно
            int windowLength = right - left + 1;
            if (windowLength - maxFreq > k) {
                // Сужаем окно слева
                freq[s.charAt(left) - 'A']--;
                left++;
                // maxFreq может уменьшиться, но мы его не обновляем вниз
                // Это оптимизация: если maxFreq был достигнут, он остаётся
                // Актуальное значение не нужно, так как мы проверяем только windowLength - maxFreq > k
            }
            // Обновляем результат
            result = Math.max(result, right - left + 1);
        }
        return result;
    }
}
Время: O(n), Память: O(1)
right проходит строку один раз, left тоже не более n раз — O(n). Массив частот на 26 букв — O(1) памяти.
Поиск анаграмм
Найти все стартовые индексы анаграмм строки t в строке s.
Шаг 1: проходим по t и заполняем freq[] — сколько каждого символа нужно, needCount = |t|. Шаг 2: фиксированное окно размера |t| скользит по s: правый край добавляет символ (если freq[c] > 0 → needCount--; freq[c]--), левый убирает (freq[c]++; если freq[c] > 0 → needCount++). Когда needCount == 0 — анаграмма, записываем индекс. O(n).
class Solution {
    public List<Integer> findAnagrams(String s, String t) {
        List<Integer> result = new ArrayList<>();
        if (s.length() < t.length()) return result;
        // freq[c] > 0 — нужен, = 0 — покрыт, < 0 — лишний
        int[] freq = new int[128];
        for (char c : t.toCharArray()) freq[c]++;
        int needCount = t.length();  // сколько символов ещё нужно
        int windowSize = t.length(); // фиксированный размер окна
        for (int right = 0; right < s.length(); right++) {
            char rightChar = s.charAt(right);
            // Добавляем правый символ
            if (freq[rightChar] > 0) needCount--;
            freq[rightChar]--;
            // Удаляем левый символ (когда окно полное)
            if (right >= windowSize) {
                char leftChar = s.charAt(right - windowSize);
                freq[leftChar]++;
                if (freq[leftChar] > 0) needCount++;
            }
            //ПРОВЕРЯЕМ текущее окно [right-windowSize+1, right]
            if (needCount == 0) {
                result.add(right - windowSize + 1);
            }
        }
        return result;
    }
}
Время: O(|s|), Память: O(1)
Один проход с окном длины |t|, на каждом шаге обновляем freq и needCount — O(|s|). Массив freq[128] — O(1) памяти.
LC 674 Longest Continuous Increasing Subsequence
Найти длину самого длинного непрерывного строго возрастающего подмассива.
Один проход O(n). Если текущий элемент > предыдущего — увеличиваем длину. Иначе — сбрасываем до 1. Запоминаем максимум.
class Solution {
    public int longestIncreasing(int[] arr) {
        if (arr.length == 0) return 0;
        int currLength = 1;
        int maxLength = 1;
        for (int i = 1; i < arr.length; i++) {
            if (arr[i] > arr[i - 1]) {
                currLength++;
                maxLength = Math.max(maxLength, currLength);
            } else {
                currLength = 1;
            }
        }
        return maxLength;
    }
}
Время: O(n), Память: O(1)
Один цикл от 1 до n−1, сравнение соседей — O(n). Несколько счётчиков — O(1) памяти.
Наглый подставной отчет
Дан бинарный массив. Можно заменить до k нулей на единицы. Найти максимальную длину подмассива из единиц.
Скользящее окно с двумя указателями. Расширяем окно вправо (right), считаем нули внутри. Когда нулей становится > k — сжимаем слева: сдвигаем left, уменьшая zerosCount если nums[left] == 0. На каждом шаге обновляем максимум длины окна (right - left + 1). O(n) время, O(1) память.
class Solution {
    public int longestOnes(int[] nums, int k) {
        int left = 0;
        int zerosCount = 0;
        int result = 0;
        for (int right = 0; right < nums.length; right++) {
            if (nums[right] == 0) {
                zerosCount++;
            }
            while (zerosCount > k) {
                if (nums[left] == 0) zerosCount--;
                left++;
            }
            result = Math.max(result, right - left + 1);
        }
        return result;
    }
}
Время: O(n), Память: O(1)
Два указателя: right проходит массив, left только увеличивается — O(n). Счётчики — O(1) памяти.
LeetCode 1493 Longest Subarray of 1's After Deleting One Element.
Дан массив из нулей и единиц. Нужно определить, какой максимальный по длине подинтервал
единиц можно получить, удалив ровно один элемент массива. Вернуть 0, если такого подинтервала не существует. Удаление элемента обязательно.
Задача сводится к: найти максимальное окно с не более чем одним нулём, удалить из него ровно один элемент. Скользящее окно: расширяем right, считаем нули. Когда нулей > 1 — сжимаем left.
Ключевой момент: длина подмассива единиц = right − left, а не right − left + 1. Потому что один элемент нужно обязательно удалить — это или ноль внутри окна, или любая единица если нулей нет. В обоих случаях убираем один символ из окна длины (right − left + 1), получаем right − left.
class Solution {
    public int longestSubarray(int[] nums) {
        int left = 0;
        int zeroCount = 0;
        int maxLen = 0;
        for (int right = 0; right < nums.length; right++) {
            // Добавляем правый элемент
            if (nums[right] == 0) zeroCount++;
            // Если нулей больше 1, убираем слева
            while (zeroCount > 1) {
                if (nums[left] == 0) zeroCount--;
                left++;
            }
            // Длина подмассива после удаления одного элемента
            maxLen = Math.max(maxLen, right - left);
        }
        return maxLen;
    }
}
Время: O(n), Память: O(1)
Один проход с обновлением prev и curr при нулях — O(n). Константные переменные — O(1) памяти.
Подставной отчет
Дан бинарный массив. Можно перевернуть или ЗАМЕНИТЬ ОДИН ноль в единицу. Найти максимальную длину подмассива из единиц.
Замена обязательная.
prev — единицы перед последним нулём, count — единицы после. При нуле: prev + 1 + count (1 — перевёрнутый ноль). Ограничиваем длиной массива. O(n).
class Solution {
    public int longestOnesAfterFlip(int[] nums) {
        int left = 0;
        int zeroCount = 0;
        int maxLen = 0;
        boolean hasZero = false;
        for (int right = 0; right < nums.length; right++) {
            if (nums[right] == 0) {
                zeroCount++;
                hasZero = true;
            }
            while (zeroCount > 1) {
                if (nums[left] == 0) {
                    zeroCount--;
                }
                left++;
            }
            maxLen = Math.max(maxLen, right - left + 1);
        }
        // если нулей не было — обязаны "сломать" одну единицу
        if (!hasZero) {
            return nums.length - 1;
        }
        return maxLen;
    }
}
Время: O(n), Память: O(1)
Линейный проход с подсчётом блоков единиц — O(n). Несколько целых — O(1) памяти.
Подстроки с полным алфавитом
Подсчитать количество подстрок, содержащих все уникальные символы s.
Шаг 1: считаем количество уникальных символов (unique). Шаг 2: расширяем right — при freq[char]  1 → covered++. Шаг 3: когда covered  unique — все уникальные в окне; все дальнейшие расширения вправо (s.length() - right) тоже валидны, сразу прибавляем к ответу. Шаг 4: сжимаем left — при freq[char] == 0 → covered--. O(n).
class Solution {
    public int countComplete(String s) {
        // target — сколько уникальных символов нужно собрать
        int target = (int) s.chars().distinct().count();
        // freq[c] — сколько раз символ в текущем окне
        int[] freq = new int[128];
        int haveCount = 0;  // сколько уникальных уже в окне
        int left = 0;
        int count = 0;
        for (int right = 0; right < s.length(); right++) {
            char rightChar = s.charAt(right);
            freq[rightChar]++;
            if (freq[rightChar] == 1) haveCount++;
            // Пока окно "полное" — сжимаем слева
            while (haveCount == target) {
                // Все суффиксы от right до конца тоже валидны
                count += s.length() - right;
                // Удаляем левый символ
                char leftChar = s.charAt(left);
                freq[leftChar]--;
                if (freq[leftChar] == 0) haveCount--;
                left++;
            }
        }
        return count;
    }
}
Время: O(n), Память: O(1)
right идёт один раз, left сдвигается при валидном окне — O(n). Массив freq[128] — O(1) памяти.
Поиск мутирующего вируса
Проверить, содержит ли строка gene какую-либо перестановку строки virus.
Та же техника, что и поиск анаграмм, но возвращаем boolean. Сначала заполняем freq по virus, затем ведём фиксированное окно по gene и поддерживаем needCount. Итого O(|gene| + |virus|), память O(1).
public boolean containsMutation(String gene, String virus) {
    if (gene.length() < virus.length()) return false;
    int[] freq = new int[128];
    for (char c : virus.toCharArray()) freq[c]++;
    int need = virus.length();
    int left = 0;
    for (int right = 0; right < gene.length(); right++) {
        char rc = gene.charAt(right);
        if (freq[rc] > 0) need--;
        freq[rc]--;
        if (right - left + 1 > virus.length()) {
            char lc = gene.charAt(left);
            freq[lc]++;
            if (freq[lc] > 0) need++;
            left++;
        }
        if (need == 0) return true;
    }
    return false;
}
Время: O(|gene| + |virus|), Память: O(1)
Сначала инициализируем freq по строке virus за O(|virus|), затем один раз проводим фиксированное окно по gene с O(1) работой на каждый сдвиг — O(|gene|). Итого O(|gene| + |virus|). Массив freq[128] — O(1) памяти.
Инвестор в стране дураков
В стране дураков вместо сложения используют умножение и богатый инвестор хочет этим воспользоваться, чтобы заработать и выкупить k подряд идущих домов.
Дан массив price и число k. Найдите произведения всех подряд идущих участков длины k. Массив price — цены домов, если price[i] ≤ 0, это должник — он покроет долг и доплатит при необходимости.
O(n). Отслеживаем нули отдельно (zeroCount). Для ненулевых поддерживаем произведение, делим/умножаем при сдвиге окна. Если есть нули — результат = 0.
class Solution {
    public int[] slidingProduct(int[] arr, int k) {
        int n = arr.length;
        int[] result = new int[n - k + 1];
        int zeroCount = 0;
        long prod = 1;
        // Первое окно
        for (int i = 0; i < k; i++) {
            if (arr[i] == 0) {
                zeroCount++;
            } else {
                prod *= arr[i];
            }
        }
        result[0] = zeroCount > 0 ? 0 : (int) prod;
        // Сдвигаем окно - O(n)
        for (int i = k; i < n; i++) {
            // Удаляем левый элемент
            if (arr[i - k] == 0) {
                zeroCount--;
            } else {
                prod /= arr[i - k];
            }
            // Добавляем правый элемент
            if (arr[i] == 0) {
                zeroCount++;
            } else {
                prod *= arr[i];
            }
            result[i - k + 1] = zeroCount > 0 ? 0 : (int) prod;
        }
        return result;
    }
}
Время: O(n), Память: O(1) доп.
Инициализация первого окна O(k), затем O(n−k) сдвигов — O(n). Массив результата O(n−k+1), дополнительно O(1).
LC 20 · Valid Parentheses
Дана строка s, содержащая только символы '(', ')', '{', '}', '[', ']'. Определить, является ли строка допустимой.
Строка допустима если:
- Каждая открывающая скобка закрыта соответствующей закрывающей
- Скобки закрываются в правильном порядке
- Каждой закрывающей скобке соответствует открывающая того же типа
O(n) время, O(n) память в худшем случае (все открывающие). Стек хранит незакрытые скобки. При закрывающей — проверяем пару с вершиной.
class Solution {
    public boolean isValid(String s) {
        // стек хранит незакрытые открывающие скобки
        Deque<Character> stack = new ArrayDeque<>();
        for (char c : s.toCharArray()) {
            if (c == '(' || c == '{' || c == '[') {
                // открывающая — кладём на стек, ждём пару
                stack.push(c);
            } else if (stack.isEmpty() || !isMatchingPair(stack.pop(), c)) {
                // закрывающая, но стек пуст — нет пары
                // или вершина стека не совпадает с текущей закрывающей
                return false;
            }
        }
        // если стек пуст — все открывающие закрыты
        // если нет — остались незакрытые скобки
        return stack.isEmpty();
    }
    private boolean isMatchingPair(char open, char close) {
        // проверяем что открывающая и закрывающая — одного типа
        return (open == '(' && close == ')')
            || (open == '{' && close == '}')
            || (open == '[' && close == ']');
    }
}
Время: O(n), Память: O(n)
Один проход: каждый символ — push или pop стека — O(n). Стек до n элементов — O(n) памяти.
LeetCode 739 Daily Temperatures
Дан массив temperature, где temperature[i] — температура в i-й день. Нужно вернуть массив result, где result[i] — количество дней, через которое температура станет выше, чем в i-й день. Если потепления не будет, result[i] = 0.
Дни с одинаковой температурой не считаются потеплением.
Монотонный убывающий стек хранит индексы. Каждый индекс push/pop ровно один раз → O(n). При нахождении более тёплого дня записываем разницу.
import java.util.ArrayDeque;
import java.util.Deque;
class Solution {
    public int[] dailyTemperatures(int[] temps) {
        int n = temps.length;
        int[] ans = new int[n];
        // Храним индексы дней.
        // Температуры по этим индексам в стеке идут по убыванию.
        Deque<Integer> stack = new ArrayDeque<>();
        for (int i = 0; i < n; i++) {
            // Пока текущий день теплее дня на вершине стека,
            // мы нашли ответ для этого старого дня.
            while (!stack.isEmpty() && temps[i] > temps[stack.peek()]) {
                int prevDay = stack.pop();
                ans[prevDay] = i - prevDay;
            }
            // Текущий день пока тоже ждёт свой более тёплый день справа.
            stack.push(i);
        }
        return ans;
    }
}
Время: O(n), Память: O(n)
Каждый индекс кладётся и извлекается из стека не более одного раза — O(n). Стек до n элементов — O(n) памяти.
RPN-калькулятор
Вычислить выражение в обратной польской нотации (RPN).
Стек для RPN-вычислений. Числа — в стек. Оператор — извлекаем два операнда, считаем, результат обратно. Порядок важен для - и /. O(n).
class Solution {
    public long evalRPN(String[] tokens) {
        Deque<Long> stack = new ArrayDeque<>();
        for (String token : tokens) {
            switch (token) {
                case "+": {
                    long b = stack.pop();
                    long a = stack.pop();
                    stack.push(a + b);
                    break;
                }
                case "-": {
                    long b = stack.pop();
                    long a = stack.pop();
                    stack.push(a - b);
                    break;
                }
                case "*": {
                    long b = stack.pop();
                    long a = stack.pop();
                    stack.push(a * b);
                    break;
                }
                case "/": {
                    long b = stack.pop();
                    long a = stack.pop();
                    stack.push(a / b);
                    break;
                }
                default:
                    stack.push(Long.parseLong(token));
            }
        }
        return stack.pop();
    }
}
Время: O(n), Память: O(n)
Один проход по токенам, на каждом операторе два pop и push — O(n). Стек до O(n) операндов — O(n) памяти.
Простой калькулятор
Дан массив `s` с положительными числами и знаками `*` и `+`. Нужно вернуть результат вычислений и важно, чтобы решение было за `O(1)` по дополнительной памяти.
Выражение — сумма произведений. prevMultiply накапливает текущее произведение. При + — добавляем его к result и начинаем новое. O(n) время, O(1) память.
class Solution {
    public int calculate(String[] s) {
        int result = 0;
        int currentProduct = 1;
        for (int i = 0; i < s.length; i++) {
            String token = s[i];
            if (token.equals("*")) {
                // просто пропускаем, умножение обработаем на числе
                continue;
            } else if (token.equals("+")) {
                // закончили блок умножений
                result += currentProduct;
                currentProduct = 1;
            } else {
                // это число
                int num = Integer.parseInt(token);
                currentProduct *= num;
            }
        }
        // добавить последний блок
        result += currentProduct;
        return result;
    }
}
Время: O(n), Память: O(1)
Один цикл по токенам — O(n). Только result и prevMultiply — O(1) памяти.
LC 227 · Базовый калькулятор II
Дана строка s, представляющая арифметическое выражение. Нужно вычислить и вернуть результат.
Допустимые операторы: +, -, *, /.
Все числа неотрицательные, в строке могут быть пробелы.
Важно:
- Деление целочисленное, с усечением к нулю
- Нельзя использовать встроенные функции вычисления выражений (например, eval)
Здесь важно заметить, что операции + и - можно отложить, а * и / нужно схлопывать сразу, иначе потеряется приоритет. Поэтому код хранит две сущности: sum для уже окончательно посчитанной части и last для последнего терма, который ещё может измениться из-за умножения или деления. Когда встречаем новый + или -, старый last можно безопасно перенести в sum и начать собирать следующий терм. Так приоритет соблюдается даже без стека.
class Solution {
    public int calculate(String s) {
        int num = 0;
        char operator = '+';
        int last = 0, sum = 0;
        for (int i = 0; i < s.length(); i++) {
            char c = s.charAt(i);
            if (Character.isDigit(c)) {
                num = num * 10 + (c - '0');
            }
            if (isOperator(c) || i == s.length() - 1) {
                if (operator == '+') {
                    sum += last;
                    last = num;
                } else if (operator == '-') {
                    sum += last;
                    last = -num;
                } else if (operator == '*') {
                    last *= num;
                } else if (operator == '/') {
                    last /= num;
                }
                num = 0;
                operator = c;
            }
        }
        return sum + last;
    }
    private boolean isOperator(char c) {
        return c == '+' || c == '-' || c == '*' || c == '/';
    }
}
Время: O(|s|), Память: O(1) доп. (целые num, last, sum, текущий оператор — константное число слотов)
Один проход по строке длины |s|; на каждый символ O(1) действий. Доп. память не зависит от длины входа.
Разворот слов
Развернуть порядок слов, сохраняя пробелы на своих местах.
Разделяем строку на блоки слов и пробелов. Слова собираем в deque в обратном порядке. При реконструкции сохраняем пробельные блоки и подставляем слова из deque. O(n).
class Solution {
    public String reverseWords(String s) {
        List<String> parts = new ArrayList<>();
        Deque<String> words = new ArrayDeque<>();
        int i = 0;
        while (i < s.length()) {
            if (s.charAt(i) == ' ') {
                int start = i;
                while (i < s.length() && s.charAt(i) == ' ') i++;
                parts.add(s.substring(start, i));
            } else {
                int start = i;
                while (i < s.length() && s.charAt(i) != ' ') i++;
                String word = s.substring(start, i);
                parts.add(word);
                words.addFirst(word);
            }
        }
        StringBuilder sb = new StringBuilder();
        for (String part : parts) {
            if (part.charAt(0) == ' ') {
                sb.append(part);
            } else {
                sb.append(words.pollFirst());
            }
        }
        return sb.toString();
    }
}
Время: O(n), Память: O(n)
Два прохода по строке — O(n). Список частей и deque слов — O(n) памяти.
Number of Islands
Дана карта в виде двумерного массива grid из '1' (суша) и '0' (вода). Определить количество островов. Остров — группа клеток с сушей, соединённых по горизонтали или вертикали. По диагонали клетки не считаются соединёнными. Карта окружена водой со всех сторон.
DFS flood fill. При нахождении 1 — затапливаем весь остров (ставим 0), +1 к счётчику. Каждая клетка посещается один раз. O(m×n) время и память.
public class Solution {
    // Проверка выхода за границы сетки
    private static boolean inBound(int i, int j, List<List<Integer>> grid) {
        return i >= 0 && i < grid.size() && j >= 0 && j < grid.get(0).size();
    }
    // DFS: затапливаем весь остров (1 → 0)
    private static void dfs(int i, int j, List<List<Integer>> grid) {
        if (!inBound(i, j, grid) || grid.get(i).get(j) == 0) return;
        grid.get(i).set(j, 0);  // затапливаем текущую клетку
        // Рекурсивно затапливаем соседей (вверх, вниз, влево, вправо)
        int[][] dirs = {{1,0},{-1,0},{0,1},{0,-1}};
        for (int[] d : dirs) {
            dfs(i + d[0], j + d[1], grid);
        }
    }
    public static int numIslands(List<List<Integer>> grid) {
        if (grid.isEmpty()) return 0;
        int count = 0;
        for (int i = 0; i < grid.size(); i++) {
            for (int j = 0; j < grid.get(0).size(); j++) {
                if (grid.get(i).get(j) == 1) {  // нашли новый остров
                    dfs(i, j, grid);            // затапливаем его целиком
                    count++;                    // увеличиваем счетчик
                }
            }
        }
        return count;
    }
}
Время: O(m·n), Память: O(m·n)
Двойной цикл по клеткам; DFS посещает каждую '1' один раз — O(m·n). Глубина рекурсии в худшем O(m·n) — O(m·n) памяти.
LC 101 Symmetric Tree
Дан корень бинарного дерева. Проверить, является ли дерево симметричным (зеркальным отражением самого себя).
Рекурсивное зеркальное сравнение: left.left ↔ right.right и left.right ↔ right.left. O(n) время, O(h) стек рекурсии.
class Solution {
    public boolean isSymmetric(TreeNode root) {
        if (root == null) return true;
        return isMirror(root.left, root.right);
    }
    private boolean isMirror(TreeNode left, TreeNode right) {
        if (left == null && right == null) return true;
        if (left == null || right == null) return false;
        if (left.val != right.val) return false;
        return isMirror(left.left, right.right) && isMirror(left.right, right.left);
    }
}
Время: O(n), Память: O(h)
Рекурсия isMirror посещает каждый узел один раз — O(n). Стек вызовов глубины h — O(h) памяти.
LC 98 · Validate Binary Search Tree
Дан корень бинарного дерева. Проверить, является ли оно валидным деревом поиска (BST).
Свойство BST: все значения в левом поддереве строго меньше текущего узла, все значения в правом поддереве строго больше.
Рекурсия с границами: для левого поддерева max = node.val, для правого min = node.val. Используем long для обработки граничных значений Integer. O(n).
class Solution {
    public boolean isValidBST(TreeNode root) {
        return isValid(root, Long.MIN_VALUE, Long.MAX_VALUE);
    }
    private boolean isValid(TreeNode node, long low, long high) {
        if (node == null) return true;
        if (node.val <= low || node.val >= high) {
            return false;
        }
        return isValid(node.left, low, node.val) && isValid(node.right, node.val, high);
    }
}
Время: O(n), Память: O(h)
isValid обходит каждый узел один раз с границами — O(n). Глубина рекурсии — O(h) памяти.
Поиск k-ого наименьшего
Дан корень BST и число k. Найти k-й наименьший элемент в дереве (1-indexed).
Inorder-обход BST выдаёт элементы в отсортированном порядке. Считаем посещённые узлы, при count == k — нашли ответ. O(H + k) время.
class Solution {
    private int count = 0;
    private int result = 0;
    public int kthSmallest(TreeNode root, int k) {
        inorder(root, k);
        return result;
    }
    private void inorder(TreeNode node, int k) {
        if (node == null) return;
        inorder(node.left, k);
        count++;
        if (count == k) {
            result = node.val;
            return;
        }
        inorder(node.right, k);
    }
}
Время: O(h + k), Память: O(h)
Inorder-обход: спуск на глубину h + k шагов до k-го элемента — O(h+k). Рекурсивный стек — O(h) памяти.
Container With Most Water
Дан массив heights, heights[i] — высота линии.
Найти максимальную площадь воды между двумя линиями.
Площадь = min(heights[i], heights[j]) × (j - i)
O(n) время, O(1) память. Два указателя с краёв. Сдвигаем меньший — сдвиг большего уменьшит ширину без возможности увеличить высоту.
public class Solution {
    public int maxArea(int[] height) {
        int l = 0;
        int r = height.length - 1;
        int resultArea = 0;
        while (l < r) {
            int currArea = Math.min(height[l], height[r]) * (r - l);
            resultArea = Math.max(resultArea, currArea);
            if (height[l] < height[r]) {
                l++;
            } else {
                r--;
            }
        }
        return resultArea;
    }
}
Время: O(n), Память: O(1)
Цикл while двигает l или r на каждом шаге — O(n) итераций. Несколько переменных — O(1) памяти.
LC 125. Valid Palindrome
Проверить, является ли строка палиндромом, игнорируя неалфавитные символы и регистр.
Два указателя к центру. Пропускаем пунктуацию/пробелы. Сравниваем в нижнем регистре. O(n) время, O(1) память.
class Solution {
    public boolean isPalindrome(String s) {
        // два указателя с двух концов, сходятся к центру
        int l = 0;
        int r = s.length() - 1;
        while (l < r) {
            // пропускаем всё что не буква и не цифра слева
            while (l < r && !Character.isLetterOrDigit(s.charAt(l))) l++;
            // пропускаем всё что не буква и не цифра справа
            while (l < r && !Character.isLetterOrDigit(s.charAt(r))) r--;
            // сравниваем символы без учёта регистра
            if (Character.toLowerCase(s.charAt(l)) != Character.toLowerCase(s.charAt(r))) {
                return false;
            }
            // сдвигаем оба указателя к центру
            l++;
            r--;
        }
        // все пары совпали — палиндром
        return true;
    }
}
Время: O(n), Память: O(1)
Указатели l и r сходятся к центру — O(n). Только индексы — O(1) памяти.
LC 283. Move Zeroes
Дан массив целых чисел nums. Переместить все нули в конец, сохраняя относительный порядок ненулевых элементов. Изменять массив нужно in-place, не создавая копию.
Если пытаться менять местами элементы по дороге, легко получить лишние действия. Код делает проще: сначала одним проходом компактно переносит все ненулевые числа в начало массива, сохраняя их порядок. Указатель write всегда показывает, куда писать следующий ненулевой элемент. Когда все полезные значения уже сдвинуты влево, остаток массива можно спокойно заполнить нулями.
class Solution {
    public void moveZeroes(int[] nums) {
        int write = 0;
        for (int read = 0; read < nums.length; read++) {
            if (nums[read] != 0) {
                // ненулевой элемент — записываем на позицию write
                nums[write] = nums[read];
                write++;
            }
            // нулевой — read идёт дальше, write стоит на месте
        }
        // хвост от write до конца заполняем нулями
        while (write < nums.length) {
            nums[write] = 0;
            write++;
        }
    }
}
Время: O(n), Память: O(1) доп. (два индекса read/write; массив nums меняем на месте)
Один проход сдвигает ненули, затем заполняем хвост нулями — O(n). In-place — O(1) памяти.
Сжатие пробелов
Дан массив символов arr. Заменить каждую последовательность из одного или более пробелов ровно одним пробелом. Изменять массив нужно in-place. Вернуть новую длину массива.
Указатель записи w. Пробел записываем только при !prevSpace. Возвращаем новую длину. O(n) время, O(1) память.
class Solution {
    public int compressSpaces(char[] arr) {
        int write = 0;
        boolean prevSpace = false;
        for (int read = 0; read < arr.length; read++) {
            if (arr[read] == ' ') {
                //если до этого был не пробел
                if (!prevSpace) {
                    arr[write++] = ' ';
                    prevSpace = true;
                }
            } else {
                arr[write++] = arr[read];
                prevSpace = false;
            }
        }
        return write;
    }
}
Время: O(n), Память: O(1)
Один проход по arr с указателем записи w — O(n). In-place — O(1) памяти.
LC 392. Is Subsequence
Даны две строки s и t. Необходимо определить, можно ли получить строку s, удаляя некоторые (возможно, ни одного) символы из строки t, не изменяя порядок оставшихся символов.
Два указателя: p1 по s, p2 по t. При совпадении продвигаем оба, иначе только p2. Если p1 дошёл до конца — s является подпоследовательностью t. O(|t|).
class Solution {
    public boolean isSubsequence(String s, String t) {
        int p1 = 0, p2 = 0;
        while (p1 < s.length() && p2 < t.length()) {
            if (s.charAt(p1) == t.charAt(p2)) {
                p1++;
            }
            p2++;
        }
        return p1 == s.length();
    }
}
Время: O(|t|), Память: O(1)
Указатель по t проходит строку один раз — O(|t|). Два индекса — O(1) памяти.
LC 167. Two Sum II - Input Array Is Sorted
Найти два элемента в отсортированном массиве, дающих в сумме target. Вернуть их индексы (1-based). O(1) доп. памяти.
Массив отсортирован → два указателя. Сумма < target → увеличиваем (l++). Сумма > target → уменьшаем (r--). O(n) время, O(1) память.
class Solution {
    public int[] twoSum(int[] nums, int target) {
        int l = 0;
        int r = nums.length - 1;
        while (l < r) {
            int currSum = nums[l] + nums[r];
            if (currSum == target) {
                return new int[]{l + 1, r + 1};
            } else if (currSum < target) {
                l++;
            } else {
                r--;
            }
        }
        return new int[]{-1, -1};
    }
}
Время: O(n), Память: O(1)
Два указателя с концов: на каждом шаге сдвигается один — O(n). Константная память.
LC 350. Intersection of Two Arrays II
Найти общие элементы двух отсортированных массивов (с учётом дубликатов).
Merge-подобная техника на отсортированных массивах. При совпадении — добавляем и двигаем оба указателя. Иначе — двигаем указатель на меньший элемент. O(n+m).
class Solution {
    public List<Integer> commonElements(int[] a, int[] b) {
        List<Integer> result = new ArrayList<>();
        int p1 = 0, p2 = 0;
        while (p1 < a.length && p2 < b.length) {
            if (a[p1] == b[p2]) {
                result.add(a[p1]);
                p1++;
                p2++;
            } else if (a[p1] < b[p2]) {
                p1++;
            } else {
                p2++;
            }
        }
        return result;
    }
}
Время: O(n + m), Память: O(min(n, m))
Указатели по двум массивам, каждый элемент один раз — O(n+m). Список совпадений — O(min(n,m)) памяти.
LC 228. Свертка в диапазоны (Summary Ranges)
Дан список неотрицательных целых чисел nums, повторов нет. Нужно преобразовать его в строку, сворачивая соседние по числовому ряду числа в диапазоны.
Если числа идут подряд (разница 1), записать как "x-y". Иначе одно число ("x"). Диапазоны через запятую.
Сортируем — тогда числа, идущие подряд по числовому ряду, окажутся рядом в массиве. Далее один проход: запоминаем start = nums[i], затем двигаем i вперёд пока nums[i+1]  nums[i] + 1. Когда последовательность прерывается — оформляем диапазон: если start  nums[i] — одиночное число, иначе "start->nums[i]". Добавляем в список и переходим к следующему.
import java.util.*;
class Solution {
    public String summaryRanges(int[] nums) {
        if (nums == null || nums.length == 0) {
            return "";
        }
        Arrays.sort(nums);
        StringBuilder result = new StringBuilder();
        for (int i = 0; i < nums.length; i++) {
            int start = nums[i];
            while (i + 1 < nums.length && nums[i] + 1 == nums[i + 1])
                i++;
            // Добавляем разделитель (кроме первого элемента)
            if (result.length() > 0) {
                result.append(",");
            }
            if (start != nums[i]) {
                result.append(start).append("->").append(nums[i]);
            } else {
                result.append(start);
            }
        }
        return result.toString();
    }
}
Время: O(n log n), Память: O(n)
Сортировка O(n log n), один проход группировки O(n). Список строк результата — O(n) памяти.
Удаление смайликов
Удалить из массива символов все вхождения смайликов :-) и :-(. После двоеточия, дефиса и первой скобки (')' или '(') нужно выкинуть и все подряд идущие такие же скобки того же типа (например ":-)))" целиком). Необходимо вернуть целое число w — новую логическую длину массива (сколько символов осталось). Если w = 0, подходящих символов нет.
Что такое in-place
Работать нужно в том же массиве arr, без выделения второго массива под ответ.
Обнаруживаем паттерн :- после чего ) или (. Пропускаем смайлик и повторы скобки. Два указателя read/write сжимают массив в начало; возвращаем write. O(n).
class Solution {
    public int removeSmileys(char[] arr) {
        int write = 0;
        int read = 0;
        int n = arr.length;
        while (read < n) {
            if (read + 2 < n 
                      && arr[read] == ':' && arr[read + 1] == '-' 
                      && (arr[read + 2] == ')' || arr[read + 2] == '(')) {
                char bracket = arr[read + 2];
                read += 3;
                while (read < n && arr[read] == bracket) {
                    read++;
                }
            } else {
                arr[write++] = arr[read++];
            }
        }
        return write;
    }
}
Время: O(n), Память: O(1)
Один проход с пропуском смайликов — O(n). Несколько счётчиков — O(1) памяти.
LC 443. String Compression
Run-length encode массива символов. Последовательные одинаковые символы заменить на символ + количество (если > 1).
Группируем последовательные одинаковые символы. Записываем символ, затем цифры количества (если > 1). Для многозначных чисел записываем каждую цифру отдельно. O(n).
class Solution {
    public int encode(char[] arr) {
        int w = 0;
        int i = 0;
        while (i < arr.length) { //так как i должен затем прыгнуть на конец группы j
            char c = arr[i];
            int j = i;
            while (j < arr.length && arr[j] == c) {
                j++;
            }
            arr[w++] = c;
            int count = j - i;
            if (count > 1) {
                String countStr = String.valueOf(count);
                for (char d : countStr.toCharArray()) {
                    arr[w++] = d;
                }
            }
            i = j;
        }
        return w;
    }
}
Время: O(n), Память: O(1)
Цикл по блокам одинаковых символов — O(n). Запись in-place — O(1) памяти.
LC 680. Valid Palindrome II
Можно ли удалить не более одного символа, чтобы строка стала палиндромом?
Два указателя с краёв. При первом несовпадении проверяем оба варианта: пропустить левый или правый. Если хотя бы один даёт палиндром — true.
Поэтому используем знак ||.
В отдельном методе делаем обычную стандартную проверку на палиндром.
class Solution {
    public boolean validPalindrome(String s) {
        int l = 0, r = s.length() - 1;
        while (l < r) {
            if (s.charAt(l) != s.charAt(r)) {
                return isPalin(s, l + 1, r) || isPalin(s, l, r - 1);
            }
            l++;
            r--;
        }
        return true;
    }
    private boolean isPalin(String s, int l, int r) {
        while (l < r) {
            if (s.charAt(l) != s.charAt(r)) {
                return false;
            }
            l++;
            r--;
        }
        return true;
    }
}
Время: O(n), Память: O(1)
Основной проход двумя указателями делает не более n/2 сравнений. При первом несовпадении метод сразу завершает основной цикл и максимум один раз запускает две дополнительные проверки, каждая тоже до n/2 сравнений. Итого не более n/2 + n/2 + n/2 = 3n/2 сравнений, а это O(n). Память O(1): только несколько индексов.
LC 977. Squares of a Sorted Array
Дан отсортированный массив. Вернуть массив квадратов в отсортированном порядке.
Наибольшие квадраты — на краях массива (отрицательные или положительные). Два указателя сравнивают абсолютные значения, заполняем результат справа налево. O(n).
class Solution {
    public int[] sortedSquares(int[] nums) {
        int n = nums.length;
        int[] result = new int[n];
        int l = 0, r = n - 1;
        int w = n - 1;
        while (l <= r) {
            int leftSq = nums[l] * nums[l];
            int rightSq = nums[r] * nums[r];
            if (leftSq > rightSq) {
                result[w--] = leftSq;
                l++;
            } else {
                result[w--] = rightSq;
                r--;
            }
        }
        return result;
    }
}
Время: O(n), Память: O(n)
Один проход двумя указателями — O(n). Массив result — O(n) памяти.
Объединение графиков
Дан массив графиков, где каждый график — список пар [время, значение], отсортированный по времени. Объединить все графики в один: [время, сумма значений].
Min-heap обрабатывает события по времени. Все события одного момента — за раз. Поддерживаем текущие значения каждого графика и их сумму. O(N log k) где N — суммарное число точек.
public class Solution {
    public List<List<Integer>> merge(List<List<List<Integer>>> graphics) {
        int k = graphics.size();
        int[] currentValues = new int[k];
        long currentSum = 0;
        //Хранит массивы int[] из 3 элементов: [время, индекс_графика, индекс_точки]
        PriorityQueue<int[]> heap = new PriorityQueue<>(Comparator.comparingInt(a -> a[0]));
        for (int i = 0; i < k; i++) {
            var g = graphics.get(i);
            if (g != null && !g.isEmpty()) {
                //Добавляет первую точку каждого графика в кучу
                heap.offer(new int[]{g.get(0).get(0), i, 0});
            }
        }
        List<List<Integer>> result = new ArrayList<>();
        while (!heap.isEmpty()) {
            //Смотрим на вершину кучи (самое раннее время)
            int curTime = heap.peek()[0];
            while (!heap.isEmpty() && heap.peek()[0] == curTime) {
                int[] ev = heap.poll();              // извлекаем [время, индекс_графика, индекс_точки]
                int gi = ev[1], pi = ev[2];         // gi - номер графика, pi - номер точки
                var g = graphics.get(gi);           // получаем график
                int newVal = g.get(pi).get(1);      // новое значение в этой точке
                int oldVal = currentValues[gi];     // текущее значение этого графика
                if (newVal != oldVal) {
                    currentValues[gi] = newVal;
                    currentSum += (long) newVal - oldVal;
                }
                //Если у этого графика есть следующая точка, добавляем ее
                if (pi + 1 < g.size()) {
                    heap.offer(new int[]{g.get(pi + 1).get(0), gi, pi + 1});
                }
            }
            result.add(Arrays.asList(curTime, (int) currentSum));
        }
        return result;
    }
}
Время: O(N log k), Память: O(k)
В куче O(k) элементов, всего N извлечений по O(log k) — O(N log k). Куча и currentValues — O(k) памяти.
Объединение нескольких графиков
Дан массив графиков [время, значение]. Объединить в один суммарный.
Идентичный подход: min-heap обрабатывает события хронологически. Поддерживаем текущую сумму, обновляя при каждом событии. O(N log k).
public class Solution {
    public List<int[]> mergeCharts(
            List<List<int[]>> charts) {
        int k = charts.size();
        int[] vals = new int[k];
        long sum = 0;
        PriorityQueue<int[]> pq =
            new PriorityQueue<>(
                Comparator.comparingInt(a -> a[0]));
        for (int i = 0; i < k; i++) {
            if (!charts.get(i).isEmpty()) {
                int[] first = charts.get(i).get(0);
                pq.offer(new int[]{first[0], i, 0});
            }
        }
        List<int[]> result = new ArrayList<>();
        while (!pq.isEmpty()) {
            int time = pq.peek()[0];
            while (!pq.isEmpty()
                   && pq.peek()[0] == time) {
                int[] ev = pq.poll();
                int ci = ev[1], pi = ev[2];
                var chart = charts.get(ci);
                int newVal = chart.get(pi)[1];
                sum += (long) newVal - vals[ci];
                vals[ci] = newVal;
                if (pi + 1 < chart.size()) {
                    int[] next = chart.get(pi + 1);
                    pq.offer(new int[]{
                        next[0], ci, pi + 1});
                }
            }
            result.add(new int[]{time, (int) sum});
        }
        return result;
    }
}
Время: O(N log k), Память: O(k)
Тот же паттерн: N операций с кучей размера k — O(N log k). Куча и vals — O(k) памяти.
LC 14 Longest Common Prefix
Найти самый длинный общий префикс массива строк.
Вертикальное сканирование: внешний цикл по позиции символа, внутренний — по всем строкам. Останавливаемся при первом несовпадении или конце строки. O(S) где S — суммарная длина строк.
class Solution {
    public String longestCommonPrefix(String[] strs) {
        if (strs == null || strs.length == 0) return "";
        for (int i = 0; i < strs[0].length(); i++) {
            char c = strs[0].charAt(i);
            for (int j = 1; j < strs.length; j++) {
                if (i >= strs[j].length() || strs[j].charAt(i) != c) {
                    return strs[0].substring(0, i);
                }
            }
        }
        return strs[0];
    }
}
Время: O(S), Память: O(1)
Внешний цикл по позициям, внутренний по строкам — до суммарной длины S. Только индексы — O(1) памяти.
LC 1438 Longest Continuous Subarray With Absolute Diff Less Than or Equal to Limit.
Дан массив целых чисел nums и целое число limit. Вернуть длину наидлиннейшего непрерывного подмассива такого, что абсолютная разность между любыми двумя элементами этого подмассива не превышает limit.
Два монотонных дека: maxD хранит индексы в убывающем порядке значений (голова = максимум окна), minD — в возрастающем (голова = минимум). Когда max - min > limit, сжимаем окно слева. Каждый элемент push/pop один раз → O(n).
class Solution {
    public int longestSubarray(int[] nums, int limit) {
        Deque<Integer> maxD = new ArrayDeque<>();
        Deque<Integer> minD = new ArrayDeque<>();
        int left = 0;
        int result = 0;
        for (int right = 0; right < nums.length; right++) {
            while (!maxD.isEmpty() && nums[maxD.peekLast()] <= nums[right]) {
                maxD.pollLast();
            }
            maxD.addLast(right);
            while (!minD.isEmpty() && nums[minD.peekLast()] >= nums[right]) {
                minD.pollLast();
            }
            minD.addLast(right);
            while (nums[maxD.peekFirst()] - nums[minD.peekFirst()] > limit) {
                left++;
                if (maxD.peekFirst() < left) {
                    maxD.pollFirst();
                }
                if (minD.peekFirst() < left) {
                    minD.pollFirst();
                }
            }
            result = Math.max(result, right - left + 1);
        }
        return result;
    }
}
Время: O(n), Память: O(n)
Каждый индекс входит/выходит из деков один раз — O(n) амортизированно. Деки до O(n) элементов — O(n) памяти.
Longest Palindromic Substring
Дана строка s. Найти самую длинную подстроку-палиндром.
Алгоритм Манакера. Строим расширенную строку с разделителями '#'. Для каждой позиции используем зеркальное свойство: если i внутри правой границы текущего палиндрома, начальный радиус берём из зеркала. Затем расширяем. Итого O(n) за счёт того, что правая граница сдвигается только вправо.
class Solution {
    public String longestPalindrome(String s) {
        // Добавляем разделители '#' между символами, а также граничные маркеры '^' и '$'
        // Это позволяет единообразно обрабатывать чётные и нечётные палиндромы
        // Пример: "bob" -> "^#b#o#b#$" 
        char[] t = new char[2 * s.length() + 3];  // +3 для '^', '#', '$'
        t[0] = '^';        // Левая граница (никогда не совпадёт с '#')
        t[1] = '#';        // Начальный разделитель
        for (int i = 0; i < s.length(); i++) {
            t[2 * i + 2] = s.charAt(i);   // Оригинальный символ на чётной позиции
            t[2 * i + 3] = '#';           // Разделитель на нечётной позиции
        }
        t[t.length - 1] = '$';  // Правая граница (никогда не совпадёт)
        // p[i] = максимальный радиус палиндрома с центром в i
        // Радиус включает сам центр, т.е. полная длина палиндрома = 2*p[i]+1
        int[] p = new int[t.length];
        // center - центр текущего палиндрома, right - его правая граница
        int center = 0, right = 0;
        // Пропускаем граничные маркеры (индексы 0 и t.length-1)
        for (int i = 1; i < t.length - 1; i++) { 
            // Ключевая оптимизация: используем зеркальное свойство
            // Зеркальная позиция относительно текущего центра
            int mirror = 2 * center - i;
            // Если i находится внутри границы текущего палиндрома,
            // то начальный радиус можно взять из зеркальной позиции
            if (i < right) {
                // Не можем выйти за правую границу, поэтому берём минимум
                p[i] = Math.min(right - i, p[mirror]);
            }
            // Если i вне границы, начинаем с радиуса 0 (значение по умолчанию)
            // Расширяем палиндром от центра i в обе стороны
            // Сравниваем символы, не выходя за границы t
            while (t[i + p[i] + 1] == t[i - p[i] - 1]) {
                p[i]++;  // Увеличиваем радиус, пока символы совпадают
            }
            // Обновляем текущий палиндром, если нашли более правый
            if (i + p[i] > right) {
                center = i;          // Новый центр
                right = i + p[i];    // Новая правая граница
                // КЛЮЧЕВОЕ СВОЙСТВО: right только увеличивается,
                // поэтому алгоритм работает за O(n)
            }
        }
        // ШАГ 4: отдельный проход для поиска максимума среди уже вычисленных значений.
        // Ключевое понимание: в расширенной строке радиус палиндрома равен длине исходного палиндрома в оригинальной строке.
        int maxLen = 0;      // Максимальная длина (в символах расширенной строки)
        int bestCenter = 0;  // Центр максимального палиндрома
        for (int i = 1; i < t.length - 1; i++) {
            if (p[i] > maxLen) {
                maxLen = p[i];      // Радиус = длина в оригинальной строке
                bestCenter = i;     // Запоминаем центр
            }
        }
        // ШАГ 5: ИЗВЛЕЧЕНИЕ ПАЛИНДРОМА ИЗ ОРИГИНАЛЬНОЙ СТРОКИ
        // Формула преобразования индекса из расширенной строки в исходную:
        // start = (bestCenter - maxLen) / 2
        // 
        // Почему? В расширенной строке позиция bestCenter соответствует
        // символу в исходной строке с индексом (bestCenter-1)/2
        // Вычитая радиус, получаем начало палиндрома в расширенной строке,
        // деление на 2 даёт индекс в исходной строке
        int start = (bestCenter - maxLen) / 2;
        return s.substring(start, start + maxLen);
        // Временная сложность: O(n), где n = длина исходной строки
        // Пространственная сложность: O(n) для расширенной строки и массива p
    }
}
Время: O(n), Память: O(n)
Алгоритм Манакера: один проход, правая граница только растёт — O(n). Массив радиусов — O(n) памяти.
Разложение числа на минимальное количество квадратов
Дано натуральное число N. Нужно вернуть список из минимального количества элементов, сумма квадратов которых равна N.
Жадный выбор здесь не работает: самый большой квадрат не всегда ведёт к оптимальному разложению. Значит, нужно для каждой суммы sum честно спросить, какой последний квадрат выгоднее взять, если оптимум для меньшей суммы уже известен. Именно поэтому код строит dp снизу вверх по формуле dp[sum] = min(dp[sum - x^2] + 1). А массив chosen нужен затем, чтобы восстановить не только количество, но и сами числа, из квадратов которых получился ответ.
class Solution {
    public List<Integer> minSquareDecomposition(int n) {
        int[] dp = new int[n + 1];
        int[] chosen = new int[n + 1]; // chosen[i] = число x, квадрат x^2 выбран последним для суммы i
        Arrays.fill(dp, Integer.MAX_VALUE / 2);
        dp[0] = 0;
        for (int sum = 1; sum <= n; sum++) {
            // Идём от больших квадратов к меньшим, чтобы в типовых кейсах получать "крупные" числа раньше
            for (int x = (int) Math.sqrt(sum); x >= 1; x--) {
                int sq = x * x;
                if (dp[sum - sq] + 1 < dp[sum]) {
                    dp[sum] = dp[sum - sq] + 1;
                    chosen[sum] = x;
                }
            }
        }
        List<Integer> result = new ArrayList<>();
        int cur = n;
        while (cur > 0) {
            int x = chosen[cur];
            result.add(x);
            cur -= x * x;
        }
        return result;
    }
}
Время: O(N·√N) (для каждой суммы 1..N — до √N кандидатов x), Память: O(N) (массивы dp и chosen длины N+1)
Для каждой суммы i перебираем все квадратные числа ≤ i (их порядка sqrt(i)). Итого O(N * sqrt(N)). Массивы dp и chosen длиной N+1 дают O(N) памяти.
Скобочная грамматика
(term)[n] — повторить term n раз. Вложенность возможна.
Стек строк хранит накопленные prefix-ы. При открывающей скобке — push текущего контекста и начинаем новую строку. При закрытии ] достаём prefix и повторяем текущую строку n раз. O(n × суммарная длина результата).
class Solution {
    public String decode(String s) {
        Deque<StringBuilder> strStack = new ArrayDeque<>();
        StringBuilder current = new StringBuilder();
        int i = 0;
        while (i < s.length()) {
            char c = s.charAt(i);
            if (c == '(') {
                // Начинаем новую группу
                strStack.push(current);
                current = new StringBuilder();
                i++;
            } else if (c == ')') {
                i++; // переходим после ')'
                // Ожидаем '['
                if (i < s.length() && s.charAt(i) == '[') {
                    i++; // пропускаем '['
                }
                // Парсим число повторений
                int repeat = 0;
                while (i < s.length() && Character.isDigit(s.charAt(i))) {
                    repeat = repeat * 10 + (s.charAt(i) - '0');
                    i++;
                }
                // Пропускаем ']'
                if (i < s.length() && s.charAt(i) == ']') {
                    i++;
                }
                // Повторяем текущую строку repeat раз
                String repeated = current.toString();
                StringBuilder prev = strStack.pop();
                for (int r = 0; r < repeat; r++) {
                    prev.append(repeated);
                }
                current = prev;
            } else {
                // Обычный символ
                current.append(c);
                i++;
            }
        }
        return current.toString();
    }
}
Время: O(|результат|), Память: O(глубина вложенности)
Каждый символ обрабатывается константное число раз — O(размер выхода). Стеки глубины вложенности — O(глубина) памяти.
LC 341. Flatten Nested List Iterator
Дан вложенный список nestedList. Каждый элемент — либо целое число, либо список, элементы которого тоже могут быть числами или списками.
Реализовать итератор, который возвращает все целые числа в порядке слева направо.
Кладём стартовый список в стек в обратном порядке, чтобы сохранить левый-to-правый порядок при pop.
Вся "тяжёлая" логика в hasNext(): пока на вершине список, раскрываем его и кладём элементы обратно в обратном порядке. Когда на вершине число, hasNext() возвращает true, а next() просто забирает его.
Такой дизайн ленивый: не нужно флаттенить весь ввод заранее, если итератор используют частично.
public class NestedIterator implements Iterator<Integer> {
    private Deque<NestedInteger> stack = new ArrayDeque<>();
    //инициализация
    public NestedIterator(List<NestedInteger> nestedList) {
        // Кладём всё в стек в обратном порядке (чтобы сохранить порядок при pop)
        for (int i = nestedList.size() - 1; i >= 0; i--) {
            stack.push(nestedList.get(i));
        }
    }
    public Integer next() {
        return stack.pop().getInteger(); // Берём число с вершины
    }
    public boolean hasNext() {
        // Пока на вершине не число - раскрываем список
        while (!stack.isEmpty() && !stack.peek().isInteger()) {
            NestedInteger top = stack.pop();           // Достаём список
            List<NestedInteger> list = top.getList();
            // Кладём его элементы в обратном порядке
            for (int i = list.size() - 1; i >= 0; i--) {
                stack.push(list.get(i));
            }
        }
        return !stack.isEmpty();
    }
}
Время: O(n) суммарно, Память: O(n)
Каждый NestedInteger попадает в стек и извлекается из стека не более одного раза, поэтому суммарно по всем вызовам next/hasNext получаем O(n). Стек может хранить до O(n) элементов.
FilteringIterator<T> — итератор по предикату
Реализовать FilteringIterator<T> — итератор-обёртку, который принимает другой Iterator<T> и предикат Predicate<T>, и возвращает только те элементы, для которых предикат возвращает true.
Пример (идея):
Ввод: итератор по [1, 2, 3, 4], предикат x -> x % 2 == 0
Вывод при обходе: 2, 4
Ключевая идея — lookahead: advance() двигает внутренний итератор до первого подходящего значения и запоминает его. hasNext() только читает флаг; next() отдаёт сохранённое значение и снова вызывает advance().
Контракт Iterator: hasNext() можно вызывать многократно без побочных эффектов; next() — только когда есть следующий элемент.
import java.util.Iterator;
import java.util.NoSuchElementException;
import java.util.function.Predicate;
public class FilteringIterator<T> implements Iterator<T> {
    private final Predicate<T> predicate;
    private final Iterator<T> iterator;
    private T next;
    private boolean hasNext;
    public FilteringIterator(Predicate<T> predicate, Iterator<T> iterator) {
        this.predicate = predicate;
        this.iterator = iterator;
        advance();
    }
    private void advance() {
        // Ищем следующий элемент, удовлетворяющий предикату
        while (iterator.hasNext()) {
            T candidate = iterator.next();
            if (predicate.test(candidate)) {
                // Нашли — кешируем и выходим
                next = candidate;
                hasNext = true;
                return;
            }
         }
        // Подходящих элементов больше нет
        hasNext = false;
        next = null;
    }
    @Override
    public boolean hasNext() {
        return hasNext;
    }
    @Override
    public T next() {
        if (!hasNext) throw new NoSuchElementException();
        T result = next;
        advance(); // next теперь указывает на следующий
        return result; // возвращаем текущий, а не следующий
    }
}
Время: O(1) амортизированно на вызов next/hasNext, Память: O(1)
Каждый элемент базового итератора рассматривается не более одного раза — суммарно O(n) на весь обход. Дополнительно храним только ссылку на следующий элемент и флаг — O(1).
Поиск Person по имени (Optional)
Реализовать функцию поиска Person по значению name. Функция принимает список людей и имя, возвращает Optional<Person>.
Предполагается, что у Person есть поле name (или геттер с тем же смыслом).
findFirst() останавливается на первом подходящем элементе (для последовательного stream — это линейный поиск слева направо).
Если нужна устойчивость к null name, вместо p.name.equals(name) обычно пишут Objects.equals(p.name, name) или проверяют аргументы.
import java.util.*;
class Person {
    String name;
    Person(String name) { this.name = name; }
}
class Solution {
    Optional<Person> findPersonByName(List<Person> persons, String name) {

                .filter(p -> p.name.equals(name))
                .findFirst();
    }
}

LC 1229. Meeting Scheduler
Два человека имеют списки свободных интервалов (отсортированных). Найти первый общий слот длительностью >= duration.
Считаем пересечение: min(end1,end2) - max(start1,start2). Если >= duration — нашли слот. Двигаем указатель, чей интервал заканчивается раньше. O(n+m).
class Solution {
    public int[] meetingTime(int[][] a, int[][] b, int duration) {
        int i = 0, j = 0;
        while (i < a.length && j < b.length) {
            int start = Math.max(a[i][0], b[j][0]);
            int end = Math.min(a[i][1], b[j][1]);
            if (end - start >= duration) {
                return new int[]{start, start + duration};
            }
            if (a[i][1] < b[j][1]) {
                i++;
            } else {
                j++;
            }
        }
        return new int[]{};
    }
}
Время: O(n + m), Память: O(1)
Два указателя по спискам интервалов — O(n+m). Константная память.
Общий префикс без учёта кратности
Даны два массива целых чисел nums1 и nums2 одинаковой длины. Для каждого префикса нужно посчитать число общих элементов без учета кратности.
"Без учета кратности" - это значит, что присутствие одного и того же числа в префиксах считается только один раз независимо от количества повторов.
Массив count[v] хранит, сколько раз число v встречено (в A, в B, или в обоих). Когда count == 2 — число есть в обоих массивах. Не учитываем кратность (число входит максимум 1 раз в каждый массив). O(n).
class Solution {
    public int[] findCommonPrefix(int[] A, int[] B) {
        int n = A.length;
        boolean[] seenInA = new boolean[n + 1];  // встречалось ли число в A
        boolean[] seenInB = new boolean[n + 1];  // встречалось ли число в B
        int[] result = new int[n];
        int common = 0;
        for (int i = 0; i < n; i++) {
            // Обрабатываем A[i]
            if (!seenInA[A[i]]) {
                seenInA[A[i]] = true;
                if (seenInB[A[i]]) {
                    common++;
                }
            }
            // Обрабатываем B[i]
            if (!seenInB[B[i]]) {
                seenInB[B[i]] = true;
                if (seenInA[B[i]]) {
                    common++;
                }
            }
            result[i] = common;
        }
        return result;
    }
}
Время: O(n), Память: O(n)
Один цикл с двумя инкрементами в count — O(n). Массив count[n+1] — O(n) памяти.
Общий префикс с учётом кратности
Даны два массива целых чисел nums1 и nums2 одинаковой длины. Для каждого префикса нужно посчитать число общих элементов c учетом кратности.
"С учетом кратности" - это значит, что присутствие одного и того же числа в префиксах считается с учетом наличия дубликатов.
При добавлении A[i]: если countA[v] <= countB[v] (т.е. после инкремента новый min увеличился), то common++. Аналогично для B[i]. O(n) при HashMap.
class Solution {
    public int[] commonPrefixWithMultiplicity(int[] A, int[] B) {
        int n = A.length;
        Map<Integer, Integer> countA = new HashMap<>();
        Map<Integer, Integer> countB = new HashMap<>();
        int[] result = new int[n];
        int common = 0;
        for (int i = 0; i < n; i++) {
            int ca = countA.merge(A[i], 1, Integer::sum);
            int cb = countB.getOrDefault(A[i], 0);
            if (ca <= cb) {
                common++;
            }
            int cb2 = countB.merge(B[i], 1, Integer::sum);
            int ca2 = countA.getOrDefault(B[i], 0);
            if (cb2 <= ca2) {
                common++;
            }
            result[i] = common;
        }
        return result;
    }
}
Время: O(n), Память: O(n)
Константное число операций HashMap на шаг — O(n). Две карты с O(n) ключей — O(n) памяти.
Prefix Common Array
Даны две перестановки A и B длины n (числа от 1 до n). Найти массив C, где C[i] = количество чисел, присутствующих в обоих префиксах A[0..i] и B[0..i].
Так как A и B — перестановки, каждое число встречается ровно по 1 разу в каждом. freq[v] считает сколько раз v встретилось суммарно. Когда freq == 2 — число есть и в A[0..i], и в B[0..i]. O(n) время, O(n) память.
class Solution {
    public int[] findThePrefixCommonArray(int[] A, int[] B) {
        int n = A.length;
        int[] freq = new int[n + 1];
        int[] result = new int[n];
        int common = 0;
        for (int i = 0; i < n; i++) {
            freq[A[i]]++;
            if (freq[A[i]] == 2) common++;
            freq[B[i]]++;
            if (freq[B[i]] == 2) common++;
            result[i] = common;
        }
        return result;
    }
}
Время: O(n), Память: O(n)
Один проход с обновлением freq — O(n). Массив freq[n+1] — O(n) памяти.
LC 1331. Rank Transform of an Array
Дан массив целых чисел arr (не отсортированный). Заменить каждый элемент его рангом. Ранг — это позиция элемента среди уникальных значений отсортированного массива, начиная с 1.
Сортируем копию массива. Назначаем ранги уникальным значениям (1, 2, 3, ...). Затем заменяем каждый элемент его рангом. O(n log n).
class Solution {
    public int[] arrayRankTransform(int[] arr) {
        // копируем массив чтобы не испортить оригинал при сортировке
        int[] sorted = new int[arr.length];
        for (int i = 0; i < arr.length; i++) {
            sorted[i] = arr[i];
        }
        // сортируем копию — теперь можем определить порядок значений
        Arrays.sort(sorted);
        // назначаем ранги уникальным значениям по порядку
        Map<Integer, Integer> rank = new HashMap<>();
        int nextRank = 1;
        for (int i = 0; i < arr.length; i++) {
            if (!rank.containsKey(sorted[i])) {
                rank.put(sorted[i], nextRank++);
            }
        }
        // заменяем каждый элемент его рангом
        for (int j = 0; j < arr.length; j++) {
            arr[j] = rank.get(arr[j]);
        }
        return arr;
    }
}
Время: O(n log n), Память: O(n)
Сортировка копии O(n log n), построение рангов + второй проход O(n). Копия, карта, результат — O(n) памяти.
Скалярное произведение векторов
Даны два сжатых вектора vec1 и vec2 одинаковой развёрнутой длины в виде списков пар [value, count]. Например, [3, 3, 3, 7] — это [[3, 3], [7, 1]]. Посчитать скалярное произведение развёрнутых векторов.
Разворачивать векторы не нужно. На каждом шаге активны val1, val2 и оставшиеся длины cnt1, cnt2; пересечение блоков даёт min(cnt1,cnt2) слагаемых val1*val2. Сумма по шагам — скалярное произведение.
import java.util.*;
public class Solution {
    public long dotProduct(List<List<Integer>> vec1, List<List<Integer>> vec2) {
        long result = 0;
        int i = 0, j = 0;
        int cnt1 = 0, cnt2 = 0;
        int val1 = 0, val2 = 0;
        while (i < vec1.size() || j < vec2.size()) {
            if (cnt1 == 0) {
                val1 = vec1.get(i).get(0);
                cnt1 = vec1.get(i).get(1);
                i++;
            }
            if (cnt2 == 0) {
                val2 = vec2.get(j).get(0);
                cnt2 = vec2.get(j).get(1);
                j++;
            }
            int minVal = Math.min(cnt1, cnt2);
            result += (long) val1 * val2 * minVal;
            cnt1 -= minVal;
            cnt2 -= minVal;
        }
        return result;
    }
}
Время: O(m + k), Память: O(1)
Каждый блок обрабатываем за O(1), указатели только двигаются вперёд — O(m+k) для m и k сегментов. Константное число переменных — O(1) доп. памяти.
Подсчёт пар индексов
Дан массив nums и целое число k. Посчитать количество пар (i, j), где i < j и nums[i] + nums[j] >= k.
Сортируем массив. Если nums[l] + nums[r] >= k, то все элементы между l и r в паре с r тоже подойдут (массив отсортирован) → count += (r-l). O(n log n).
class Solution {
    public long countPairs(int[] nums, int k) {
        Arrays.sort(nums);
        long count = 0;
        int l = 0, r = nums.length - 1;
        while (l < r) {
            if (nums[l] + nums[r] >= k) {
                count += (r - l);
                r--;
            } else {
                l++;
            }
        }
        return count;
    }
}
Время: O(n log n), Память: O(1)
Сортировка O(n log n), два указателя O(n) — доминирует сортировка. In-place — O(1) доп. памяти.
LC 658. Find K Closest Elements
Дан отсортированный массив arr и два числа k и x. Найти k ближайших к x элементов. Вернуть в отсортированном порядке.
После сортировки ответ здесь всегда является непрерывным окном длины k. Значит, искать нужно не сами элементы по отдельности, а правильное начало такого окна. Код бинарно ищет эту левую границу и сравнивает два соседних кандидата: окно, начинающееся в mid, и окно, начинающееся в mid + 1. Если правое окно ближе к x, старт надо двигать вправо; иначе лучше оставить текущее или идти влево.
class Solution {
    //Бинарный поиск начала окна размером k
    public List<Integer> findClosestElements(int[] arr, int k, int x) {
        // left и right — возможные позиции НАЧАЛА окна длиной k
        // Окно может начинаться от 0 до arr.length - k (чтобы поместиться)
        int left = 0;
        int right = arr.length - k;
        // Бинарный поиск левой границы оптимального окна, остановка когда left==right
        // Сравниваем: выгоднее сдвинуть окно вправо или оставить
        while (left < right) {
            int mid = left + (right - left) / 2;
            // Сравниваем расстояния до x:
            // - arr[mid] — левый край ТЕКУЩЕГО окна
            // - arr[mid + k] — левый край СЛЕДУЮЩЕГО окна (сдвиг на 1 вправо)
            // 
            // Если левый край текущего окна ДАЛЬШЕ от x, чем левый край следующего,
            // значит окно нужно сдвинуть вправо (left = mid + 1)
            // Иначе оставляем окно левее (right = mid)
            if (x - arr[mid] > arr[mid + k] - x) {
                left = mid + 1;   // Сдвигаем окно вправо
            } else {
                right = mid;      // Оставляем окно левее
            }
        }
        // Собираем результат: k элементов начиная с left
        List<Integer> result = new ArrayList<>();
        for (int i = left; i < left + k; i++) {
            result.add(arr[i]);
        }
        return result;
    }
}
Время: O(log(n−k) + k) (бинпоиск начала окна длины k на отсортированном arr + формирование ответа), Память: O(k) (список результата; вход отсортирован заранее)
Бинарный поиск по [0, n−k] — O(log(n−k)), затем копируем k элементов. Список из k элементов — O(k) памяти.
К ближайших чисел
Дан отсортированный массив и число target. Найти k ближайших к target чисел. Если расстояния равны — меньший элемент предпочтительнее.
Бинарный поиск позиции ближайшей к target. Два указателя расходятся от этой позиции: выбираем ближайший из arr[left] и arr[right]. O(log n + k).
class Solution {
    public List<Integer> kClosest(int[] arr, int target, int k) {
        int pos = binarySearch(arr, target);
        int left = pos - 1;
        int right = pos;
        List<Integer> result = new ArrayList<>();
        while (result.size() < k) {
            if (left < 0) {
                result.add(arr[right++]);
            } else if (right >= arr.length) {
                result.add(arr[left--]);
            } else if (Math.abs(arr[left] - target) <= Math.abs(arr[right] - target)) {
                result.add(arr[left--]);
            } else {
                result.add(arr[right++]);
            }
        }
        return result;
    }
    private int binarySearch(int[] arr, int target) {
        int lo = 0, hi = arr.length;
        while (lo < hi) {
            int mid = lo + (hi - lo) / 2;
            if (arr[mid] < target) lo = mid + 1;
            else hi = mid;
        }
        return lo;
    }
}
Время: O(log n + k), Память: O(k)
binarySearch O(log n), затем k шагов выбора ближайшего — O(k). Список из k чисел — O(k) памяти.
Минимальная разность
Даны два отсортированных массива. Найти минимальную абсолютную разность между элементами из разных массивов.
Два указателя на отсортированных массивах. Двигаем указатель на меньший элемент — это приближает значения друг к другу. O(n+m) время, O(1) память.
class Solution {
    public int minDifference(int[] a, int[] b) {
        int i = 0, j = 0;
        int minDiff = Integer.MAX_VALUE;
        while (i < a.length && j < b.length) {
            int diff = Math.abs(a[i] - b[j]);
            minDiff = Math.min(minDiff, diff);
            if (a[i] < b[j]) {
                i++;
            } else {
                j++;
            }
        }
        return minDiff;
    }
}
Время: O(n + m), Память: O(1)
Два указателя по отсортированным массивам — O(n+m). Только переменные — O(1) памяти.
Группировка чисел-анаграмм
Сгруппировать числа, чьи цифры являются анаграммами друг друга.
Для каждого числа сортируем его цифры — это ключ в HashMap. Числа с одинаковым ключом — анаграммы. O(n × d log d), где d — кол-во цифр.
class Solution {
    public List<List<Integer>> groupAnagramNumbers(int[] nums) {
        Map<String, List<Integer>> map = new HashMap<>();
        for (int num : nums) {
            char[] digits = String.valueOf(num).toCharArray();
            Arrays.sort(digits);
            String key = new String(digits);
            map.computeIfAbsent(key, k -> new ArrayList<>()).add(num);
        }
        return new ArrayList<>(map.values());
    }
}
Время: O(n · d log d), Память: O(n)
Для каждого числа сортируем d цифр O(d log d) и кладём в HashMap — O(n·d log d). Карта и списки — O(n) памяти.
Односторонняя разница
Дан массив и число k >= 0. Посчитать количество пар (i, j), где i < j и nums[j] - nums[i] == k.
Ведём HashMap частот и идём слева направо. Для каждого num проверяем, сколько раз (num − k) уже встречалось в map — столько новых пар с нужной разностью. После подсчёта добавляем num в map. Это корректно: i < j гарантируется порядком обхода. Работает и при k == 0 (ищем дубликаты num — 0 = num). O(n) время, O(n) память.
class Solution {
    public int findPairs(int[] nums, int k) {
        if (k < 0) {
          return 0;
        }
        Map<Integer, Integer> freq = new HashMap<>();
        int count = 0;
        for (int num : nums) {
            // сколько раз num-k уже встречался — столько новых пар
            count += freq.getOrDefault(num - k, 0);
            freq.put(num, freq.getOrDefault(num, 0) + 1);
        }
        return count;
    }
}
Время: O(n), Память: O(n)
Один проход по массиву — O(n). HashMap занимает O(n) памяти.
Подсчёт пар с разницей >= К
Дан массив целых чисел nums и целое число k. Необходимо найти количество пар элементов массива, разница которых больше или равна k.
Парой считаются два элемента nums[i] и nums[j], где abs(nums[i] - nums[j]) >= k (i может быть равен j).
Сортируем массив. Фиксируем указатель left = 0. Для каждого right двигаем left вправо, пока nums[right] - nums[left] >= k — left движется к первому невалидному индексу. Все элементы с индексами 0..left-1 имеют разность >= k с nums[right], поэтому count += left. Поскольку массив отсортирован и left не откатывается назад — два указателя работают корректно.
import java.util.*;
public class Solution {
    public Integer countPairs(List<Integer> nums, Integer k) {
         Collections.sort(nums);
        int n = nums.size();
        if (k <= 0) {
            return n * (n + 1) / 2;
        }
        int left = 0;
        int count = 0;      
        for (int right = 0; right < n; right++) {
            while (left < right && nums.get(right) - nums.get(left) >= k) {
                left++;
            }
            count += left;
        }
        return count;
    }
}
Время: O(n log n), Память: O(1)
Сортировка — O(n log n). Затем два указателя: right проходит массив, left только растёт — итого O(n). Итого O(n log n). Память O(1).
Рост акций компании
Дан массив цен акций по дням. Найти максимальную прибыль от одной покупки и одной продажи (купить раньше, продать позже).
Один проход: отслеживаем минимальную цену. Для каждого дня — потенциальная прибыль = price - minPrice. Обновляем максимум. O(n) время, O(1) память.
class Solution {
    public int maxProfit(int[] prices) {
        int minPrice = Integer.MAX_VALUE;
        int maxProfit = 0;
        for (int price : prices) {
            if (price < minPrice) {
                minPrice = price;
            } else {
                maxProfit = Math.max(maxProfit, price - minPrice);
            }
        }
        return maxProfit;
    }
}
Время: O(n), Память: O(1)
Один проход по ценам с minPrice — O(n). Только переменные — O(1) памяти.
URL-ификация строки
Заменить все пробелы в строке на "%20". Дана строка с дополнительным местом в конце и реальная длина.
Два прохода: считаем пробелы, затем заполняем с конца. Каждый пробел заменяется на '%20' (3 символа вместо 1). O(n) время, O(1) дополнительная память.
class Solution {
    public String urlify(char[] str, int trueLength) {
        // Считаем количество пробелов
        int spaceCount = 0;
        for (int read = 0; read < trueLength; read++) {
            if (str[read] == ' ') spaceCount++;
        }
        // Новая длина после замены (' ' → '%20')
        int newLen = trueLength + spaceCount * 2;
        int write = newLen - 1;  // указатель записи (с конца)
        // Идём с конца строки (read указывает на последний реальный символ)
        for (int read = trueLength - 1; read >= 0; read--) {
            if (str[read] == ' ') {
                str[write--] = '0';
                str[write--] = '2';
                str[write--] = '%';
            } else {
                str[write--] = str[read];
            }
        }
        return new String(str, 0, newLen);
    }
}
Время: O(n), Память: O(1)
Два прохода: подсчёт пробелов + заполнение с конца — O(n). In-place в char[] — O(1) доп. памяти.
LC 11. Container With Most Water
Дан массив heights. Найти два столбца, которые вместе с осью X образуют контейнер с наибольшим количеством воды.
Вода = min(height[l], height[r]) × (r - l)
Жадный подход с двумя указателями. Сдвигаем меньший столбец — сдвиг большего не может увеличить площадь (высота ограничена меньшим). O(n) время, O(1) память.
class Solution {
    public int maxArea(int[] height) {
        int left = 0;
        int right = height.length - 1;
        int maxWater = 0;
        while (left < right) {
            int water = Math.min(height[left], height[right]) * (right - left);
            maxWater = Math.max(maxWater, water);
            if (height[left] < height[right]) {
                left++;
            } else {
                right--;
            }
        }
        return maxWater;
    }
}
Время: O(n), Память: O(1)
Два указателя с краёв: двигаем меньшую стенку — O(n). Константная доп. память.
Поиск ближайшего значения в BST
Дан корень BST и вещественное число target (double). Найти значение узла, ближайшее к target.
Итеративный спуск по BST. На каждом шаге обновляем closest, если текущий узел ближе к target. Идём влево если target < val, иначе вправо. O(h) время, O(1) память.
class Solution {
    public int closestValue(TreeNode root, double target) {
        int closest = root.val;
        while (root != null) {
            if (Math.abs(root.val - target) < Math.abs(closest - target)) {
                closest = root.val;
            }
            root = target < root.val ? root.left : root.right;
        }
        return closest;
    }
}
Время: O(h), Память: O(1)
Спуск по BST от корня к листу — O(h). Один указатель и closest — O(1) памяти.
LC 938. Range Sum of BST
Дан корень бинарного дерева поиска и два целых числа low и high. Вернуть сумму значений всех узлов, значение которых входит в диапазон [low, high] включительно.
Ключ — использовать свойство BST. Если root.val < low, весь левый subtree тоже меньше low и его можно пропустить. Если root.val > high, весь правый subtree больше high и тоже отсекается. Иначе текущий узел входит в ответ, рекурсивно считаем обе стороны.
class Solution {
    public int rangeSumBST(TreeNode root, int low, int high) {
        if (root == null) return 0;
        //Сам узел не попадает в диапазон (он слишком маленький)
        //Всё левое поддерево тоже меньше root.val, значит оно тоже меньше low → его можно полностью пропустить
        //Правое поддерево может содержать узлы в диапазоне, потому что там значения больше
        if (root.val < low) return rangeSumBST(root.right, low, high);
        if (root.val > high) return rangeSumBST(root.left, low, high);
        return root.val
            + rangeSumBST(root.left, low, high)
            + rangeSumBST(root.right, low, high);
    }
}
Время: O(n) в худшем случае, Память: O(h)
В худшем случае посещаем все узлы — O(n). Глубина рекурсии равна высоте дерева h, поэтому стек вызовов — O(h). За счёт отсечения веток на сбалансированном BST обычно быстрее полного обхода.
LC 124. Binary Tree Maximum Path Sum
Путь в бинарном дереве — последовательность узлов, где каждые два соседних соединены ребром, и каждый узел встречается не более одного раза. Путь не обязан проходить через корень.
Вернуть максимальную сумму пути в дереве.
Главная идея здесь в том, что у задачи есть два разных ответа, и их нельзя смешивать. Для глобального максимума путь через текущий узел может забрать обе ветки сразу: левую и правую. А вот родителю мы можем вернуть только одну ветку, потому что путь вверх не может раздваиваться. Поэтому код отдельно обновляет maxSum через node.val + left + right и отдельно возвращает вклад одной лучшей стороны; отрицательные ветки просто невыгодно брать вообще.
class Solution {
    private int maxSum = Integer.MIN_VALUE;
    public int maxPathSum(TreeNode root) {
        dfs(root);
        return maxSum;
    }
    private int dfs(TreeNode node) {
        if (node == null) return 0;
        // Если сумма от ребёнка отрицательная — лучше не брать его вообще
        int left = Math.max(0, dfs(node.left));
        int right = Math.max(0, dfs(node.right));
        maxSum = Math.max(maxSum, node.val + left + right);
        //для родителя мы можем дать ТОЛЬКО ОДНУ сторону
        return node.val + Math.max(left, right);
    }
}
Время: O(n) (каждый узел один раз), Память: O(h) (стек DFS; h — высота дерева)
Каждый узел посещается один раз — O(n). Дополнительная память определяется глубиной рекурсии h (высота дерева), поэтому O(h).
LC 103 · Binary Tree Zigzag Level Order Traversal
Дан корень бинарного дерева. Вернуть зигзагообразный обход по уровням: чётные уровни слева направо, нечётные — справа налево.
BFS по уровням. Чередуем направление: чётные уровни — addLast, нечётные — addFirst (LinkedList как deque). O(n) время и память.
class Solution {
    public List<List<Integer>> zigzagLevelOrder(TreeNode root) {
        List<List<Integer>> result = new ArrayList<>();
        if (root == null) return result;
        Deque<TreeNode> queue = new ArrayDeque<>();
        queue.add(root);
        boolean leftToRight = true;  // флаг направления: true = слева направо, false = справа налево
        while (!queue.isEmpty()) {
            int size = queue.size();           // количество узлов на текущем уровне
            LinkedList<Integer> level = new LinkedList<>();  // список для текущего уровня
            // Обрабатываем все узлы текущего уровня
            for (int i = 0; i < size; i++) {
                TreeNode node = queue.poll();   // извлекаем узел из очереди
                // Добавляем значение в зависимости от направления
                if (leftToRight) {
                    level.addLast(node.val);    // при прямом порядке — в конец
                } else {
                    level.addFirst(node.val);   // при обратном — в начало
                }
                // Добавляем детей в очередь для следующего уровня (всегда слева направо)
                if (node.left != null) queue.add(node.left);
                if (node.right != null) queue.add(node.right);
            }
            result.add(level);          // сохраняем уровень
            leftToRight = !leftToRight; // меняем направление для следующего уровня
        }
        return result;
    }
}
Время: O(n), Память: O(n)
BFS по уровням: каждый узел один раз — O(n). Очередь и списки — O(n) памяти.
LC 100 Same Tree
Даны корни двух бинарных деревьев p и q. Проверить, являются ли они одинаковыми (совпадают структура и все значения узлов).
Рекурсия: базовые случаи — оба null (true), один null (false), значения разные (false). Иначе рекурсивно проверяем левые и правые поддеревья. O(n) время, O(h) стек.
class Solution {
    public boolean isSameTree(TreeNode p, TreeNode q) {
        // оба узла пусты — поддеревья одинаковы
        if (p == null && q == null) return true;
        // один пуст, другой нет — структура различается
        if (p == null || q == null) return false;
        // значения узлов различаются
        if (p.val != q.val) return false;
        // рекурсивно проверяем левые и правые поддеревья
        return isSameTree(p.left, q.left)
            && isSameTree(p.right, q.right);
    }
}
Время: O(n), Память: O(h)
Рекурсивное сравнение: каждый узел один раз — O(n). Стек рекурсии — O(h) памяти.
LC 110 Balanced Binary Tree
Дан корень бинарного дерева. Проверить, является ли оно сбалансированным по высоте. Дерево сбалансировано, если для каждого узла разница высот левого и правого поддеревьев не превышает 1.
Возвращаем -1 как сигнал несбалансированности. Для каждого узла проверяем |leftH - rightH| <= 1. Рано прерываемся при -1. O(n) время, O(h) стек.
class Solution {
    public boolean isBalanced(TreeNode root) {
        return height(root) != -1;
    }
    private int height(TreeNode node) {
        if (node == null) return 0;
        int left = height(node.left);
        if (left == -1) return -1;
        int right = height(node.right);
        if (right == -1) return -1;
        if (Math.abs(left - right) > 1) return -1;
        return Math.max(left, right) + 1;
    }
}
Время: O(n), Память: O(h)
Для каждого узла считаем высоту с ранним выходом — O(n). Стек рекурсии — O(h) памяти.
Path Sum II
Дан корень бинарного дерева и целое число targetSum. Найти все пути от корня до листа, где сумма значений узлов равна targetSum. Путь — последовательность узлов от корня до листа.
DFS + backtracking. На каждом узле: добавляем в путь, вычитаем из remaining. На листе: если remaining == 0 — нашли путь. При возврате удаляем последний элемент. O(n) обход, O(n²) для копирования путей.
class Solution {
    private List<List<Integer>> result = new ArrayList<>();
    public List<List<Integer>> pathSum(TreeNode root, int targetSum) {
        dfs(root, targetSum, new ArrayList<>());
        return result;
    }
    private void dfs(TreeNode node, int remaining, List<Integer> path) {
        if (node == null) return;
        path.add(node.val);
        remaining -= node.val;
        if (node.left == null && node.right == null && remaining == 0) {
            result.add(new ArrayList<>(path));
        } else {
            dfs(node.left, remaining, path);
            dfs(node.right, remaining, path);
        }
        path.remove(path.size() - 1);
    }
}
Время: O(n), Память: O(h)
DFS с backtracking обходит дерево за O(n). Путь длины h и стек — O(h) памяти.
Lowest Common Ancestor III
Дан корень бинарного дерева и два узла p и q. Найти их наименьшего общего предка (LCA). Узлы p и q могут отсутствовать в дереве — в этом случае вернуть null.
LCA — самый глубокий узел, являющийся предком обоих p и q (узел считается предком самого себя).
DFS обходит всё дерево (не останавливается рано). Флаги foundP/foundQ подтверждают, что оба узла найдены. Если один в левом поддереве, другой в правом — текущий узел = LCA. O(n).
class Solution {
    private boolean foundP = false;
    private boolean foundQ = false;
    public TreeNode lowestCommonAncestor(TreeNode root, TreeNode p, TreeNode q) {
        TreeNode lca = dfs(root, p, q);
        /**
         * важно:
         * если хотя бы один узел отсутствует в дереве,
         * LCA не существует → возвращаем null
         */
        return (foundP && foundQ) ? lca : null;
    }
    private TreeNode dfs(TreeNode node, TreeNode p, TreeNode q) {
        if (node == null) return null;
        /**
         * рекурсивно ищем в левом и правом поддеревьях
         */
        TreeNode left = dfs(node.left, p, q);
        TreeNode right = dfs(node.right, p, q);
        if (node == p) {
            foundP = true;
            return node;
        }
        if (node == q) {
            foundQ = true;
            return node;
        }
        /**
         * если p и q нашлись в разных поддеревьях
         * (один слева, другой справа),
         * текущий узел — их LCA
         */
        if (left != null && right != null)
            return node;
        /**
         * иначе:
         * - либо нашли что-то только слева
         * - либо только справа
         * - либо вообще ничего
         *
         * просто пробрасываем найденный узел вверх
         */
        return left != null ? left : right;
    }
}
Время: O(n), Память: O(h)
Один DFS: каждый узел константное число раз — O(n). Стек рекурсии — O(h) памяти.
652 Find Duplicate Subtrees
Дан корень бинарного дерева. Найти все повторяющиеся поддеревья. Два поддерева дублируются, если имеют одинаковую структуру и значения узлов. Вернуть корневые узлы всех дубликатов (по одному на каждую группу).
ВАЖНО: здесь нужно сравнить структуру + значения + порядок левого/правого ребёнка
Сравнивать поддеревья напрямую было бы слишком дорого, поэтому код сначала придумывает для каждого поддерева канонический идентификатор. В постпорядке это делается естественно: сначала уже известны id левого и правого детей, потом по ним и значению текущего узла строится ключ текущего поддерева. Если два поддерева дают один и тот же ключ, им присваивается один и тот же id. Когда такой id встречается второй раз, текущий узел уже можно добавлять в ответ как представителя группы дубликатов.
class Solution {
    private Map<String, Integer> ids = new HashMap<>();
    private Map<Integer, Integer> count = new HashMap<>();
    private List<TreeNode> result = new ArrayList<>();
    private int serial = 1;
    public List<TreeNode> findDuplicateSubtrees(TreeNode root) {
        dfs(root);
        return result;
    }
    private int dfs(TreeNode node) {
        if (node == null) return 0;
        int left = dfs(node.left);
        int right = dfs(node.right);
        // Ключ из значения и ID поддеревьев
        String key = node.val + " " + left + " " + right;
        int id = ids.computeIfAbsent(key, k -> serial++);
        int cnt = count.merge(id, 1, Integer::sum);
        if (cnt == 2) {
            result.add(node);
        }
        return id;
    }
}
Время: O(n) при хэшировании ключа O(1) в среднем (n узлов), Память: O(n) (карты ids/count + список ответа; до n различных сигнатур)
DFS обходит каждый из n узлов ровно один раз. Для узла ключ — короткая строка из val и двух уже вычисленных id детей, обновление карт — амортизированно O(1). Храним до O(n) различных сигнатур поддеревьев и счётчики — O(n) памяти. (Строгая оценка с учётом длины строки ключа — O(n log n) по времени.)
Поиск k-ого наибольшего в BST
Дан корень BST и число k. Найти k-й наибольший элемент в дереве (1-indexed).
Обратный inorder-обход (right → node → left) выдаёт элементы BST в убывающем порядке. Останавливаемся на k-м элементе. O(H + k) время.
class Solution {
    private int count = 0;
    private int result = 0;
    public int kthLargest(TreeNode root, int k) {
        reverseInorder(root, k);
        return result;
    }
    private void reverseInorder(TreeNode node, int k) {
        if (node == null || count >= k) return;
        reverseInorder(node.right, k);
        count++;
        if (count == k) {
            result = node.val;
            return;
        }
        reverseInorder(node.left, k);
    }
}
Время: O(h + k), Память: O(h)
Обратный inorder: h шагов вниз + k шагов — O(h+k). Стек рекурсии — O(h) памяти.
Проверка всех листьев
Дан корень бинарного дерева. Проверить, что все листья находятся на одной глубине.
DFS с отслеживанием глубины. Первый лист задаёт эталонную глубину. Все остальные листья должны совпадать. O(n) время, O(h) стек.
class Solution {
    private int leafDepth = -1;
    public boolean checkLeaves(TreeNode root) {
        return dfs(root, 0);
    }
    private boolean dfs(TreeNode node, int depth) {
        if (node == null) return true;
        if (node.left == null && node.right == null) {
            if (leafDepth == -1) {
                leafDepth = depth;
            }
            return depth == leafDepth;
        }
        return dfs(node.left, depth + 1)
            && dfs(node.right, depth + 1);
    }
}
Время: O(n), Память: O(h)
DFS посещает каждый узел и сравнивает глубину листьев — O(n). Стек рекурсии — O(h) памяти.
Сумма от корня до листа
Дан корень бинарного дерева, где каждый узел содержит цифру от 0 до 9. Каждый путь от корня до листа формирует число (корень — старший разряд). Найти сумму всех чисел, образованных путями от корня до листьев.
DFS: на каждом узле формируем число currentNum * 10 + val. На листе возвращаем число. Суммируем все пути. O(n) время, O(h) стек.
class Solution {
    public int sumNumbers(TreeNode root) {
        return dfs(root, 0);
    }
    private int dfs(TreeNode node, int currentNum) {
        if (node == null) return 0;
        currentNum = currentNum * 10 + node.val;
        if (node.left == null
            && node.right == null) {
            return currentNum;
        }
        return dfs(node.left, currentNum)
            + dfs(node.right, currentNum);
    }
}
Время: O(n), Память: O(h)
DFS передаёт число по пути, суммирует на листьях — O(n). Стек — O(h) памяти.
Эквивалентные поддеревья по множеству букв
Дано бинарное дерево, в каждой вершине которого записана одна буква A–Z.
Две вершины считаются эквивалентными, если поддеревья этих вершин содержат одинаковое множество букв (без учёта частот).
Найти любую пару эквивалентных вершин.
Каждое поддерево кодируется 26-битной маской (OR масок детей + бит своей буквы). Маска = «набор уникальных букв». Храним первый встреченный узел с каждой маской в HashMap. При повторе — возвращаем пару. Ранний выход через Holder обрывает рекурсию.
import java.util.HashMap;
import java.util.Map;
class TNode {
    char value; // 'A'..'Z'
    TNode left;
    TNode right;
}
public class Solution {
    public record Pair<T, U>(T first, U second) {}
    private Pair<TNode, TNode> result;
    public Pair<TNode, TNode> findEquivalentSubtrees(TNode root) {
        result = null;
        Map<Integer, TNode> seen = new HashMap<>();
        dfs(root, seen);
        return result;
    }
    private int dfs(TNode node, Map<Integer, TNode> seen) {
        if (node == null || result != null) {
            return 0;
        }
        int leftMask = dfs(node.left, seen);
        if (result != null) {
            return 0;
        }
        int rightMask = dfs(node.right, seen);
        if (result != null) {
            return 0;
        }
        int mask = leftMask | rightMask | (1 << (node.value - 'A'));
        TNode prev = seen.get(mask);
        if (prev != null) {
            result = new Pair<>(prev, node);
        } else {
            seen.put(mask, node);
        }
        return mask;
    }
}
Время: O(n), Память: O(n)
Один DFS по всем n узлам. HashMap хранит не более n масок. Битовые операции O(1) (маска 26 бит — int).
Эквивалентные поддеревья максимального суммарного размера
Дан корень бинарного дерева, в каждой вершине которого записана одна буква A–Z.
Найти две вершины, поддеревья которых содержат одинаковое множество букв (без учёта частот) и при этом сумма количества вершин в этих двух поддеревьях максимальна. Вернуть корни найденных поддеревьев.
Каждое поддерево кодируется 26-битной маской. DFS считает маску и размер снизу вверх. В HashMap храним для каждой маски узел с наибольшим поддеревом — это гарантирует максимальную сумму при совпадении. При каждом совпадении масок обновляем ответ, если сумма больше. Итог: один проход O(n), без повторного обхода.
import java.util.*;
class Solution {
    private Result answer = new Result(null, null);
    private int bestSum = 0;
    private final Map<Integer, Info> bestByMask = new HashMap<>();
    public Result findEquivalentSubtrees(TreeNode root) {
        dfs(root);
        return answer;
    }
    private Info dfs(TreeNode node) {
        if (node == null) {
            return new Info(0, 0, null);
        }
        Info left = dfs(node.left);
        Info right = dfs(node.right);
        int currentBit = 1 << (node.value - 'A');
        int mask = left.mask() | right.mask() | currentBit;
        int size = left.size() + right.size() + 1;
        Info current = new Info(mask, size, node);
        Info previousBest = bestByMask.get(mask);
        if (previousBest != null) {
            int candidateSum = previousBest.size() + current.size();
            if (candidateSum > bestSum) {
                bestSum = candidateSum;
                answer = new Result(previousBest.node(), current.node());
            }
            if (current.size() > previousBest.size()) {
                bestByMask.put(mask, current);
            }
        } else {
            bestByMask.put(mask, current);
        }
        return current;
    }
    record Info(int mask, int size, TreeNode node) {}
    record Result(TreeNode first, TreeNode second) {}
}
Время: O(n), Память: O(n)
Один DFS — O(n). HashMap хранит не более n масок. Битовые операции O(1).
Count Pairs With Absolute Diff K
Дан массив nums и число k. Посчитать количество пар (i, j), где i < j и |nums[i] - nums[j]| == k.
Я решаю задачу за один проход с использованием hash map, где храню частоты уже просмотренных элементов.
Идея в том, что для каждого текущего числа num я хочу найти все предыдущие элементы, которые образуют с ним пару с разницей k.
Условие:
|a - b| = k
эквивалентно:
a = b + k  или  a = b - k
Поэтому для текущего num я проверяю:
сколько раз уже встречалось num - k
сколько раз уже встречалось num + k
и добавляю эти количества к ответу. После этого добавляю текущий num в map, чтобы он мог участвовать в парах для следующих элементов.
class Solution {
    public int countKDifference(int[] nums, int k) {
        Map<Integer, Integer> freq = new HashMap<>();
        int count = 0;
        for (int num : nums) {
            count += freq.getOrDefault(num - k, 0);
            count += freq.getOrDefault(num + k, 0);
            freq.put(num, freq.getOrDefault(num, 0) + 1);
        }
        return count;
    }
}
Время: O(n), Память: O(n)
Один проход с HashMap: добавляем частоты num−k и num+k — O(n). Карта — O(n) памяти.
LC 350 Intersection of Two Arrays II
Найти пересечение двух массивов с учётом кратности (каждый элемент в результате появляется столько раз, сколько он встречается в обоих).
HashMap хранит частоты nums1. Для каждого элемента nums2: если freq > 0 — добавляем в результат и уменьшаем freq. O(n+m) время, O(min(n,m)) память.
class Solution {
    public int[] intersect(int[] nums1, int[] nums2) {
        Map<Integer, Integer> freq = new HashMap<>();
        for (int num : nums1) {
            freq.merge(num, 1, Integer::sum);
        }
        List<Integer> result = new ArrayList<>();
        for (int num : nums2) {
            int count = freq.getOrDefault(num, 0);
            if (count > 0) {
                result.add(num);
                freq.put(num, count - 1);
            }
        }
        return result.stream()
            .mapToInt(Integer::intValue).toArray();
    }
}
Время: O(n + m), Память: O(min(n, m))
Частоты первого массива + проход по второму — O(n+m). Карта — O(min(n,m)) памяти.
LC 387. Первый уникальный символ в строке
Дана строка s. Найти индекс первого неповторяющегося символа. Если такого символа нет — вернуть -1.
Если для каждого символа отдельно проверять, уникален он или нет, получится лишний повторный обход. Код делит задачу на две простые части: сначала собирает частоты всех символов, а потом возвращается к строке в исходном порядке. Как только встречается символ с частотой 1, это и есть первый уникальный индекс.
class Solution {
    public int firstUniqChar(String s) {
        int[] freq = new int[26];
        for (char c : s.toCharArray()) {
            freq[c - 'a']++;
        }
        for (int i = 0; i < s.length(); i++) {
            if (freq[s.charAt(i) - 'a'] == 1) {
                return i;
            }
        }
        return -1;
    }
}
Время: O(n), Память: O(1) доп. (cnt[26] для строчных латинских букв — |Σ|=26, константа алфавита)
Два линейных прохода по строке — O(n). Массив частот на 26 букв латиницы — O(1).
Group Anagrams для mixed-алфавита
Дан массив строк. Нужно сгруппировать анаграммы, но строки могут содержать:
- русские буквы,
- английские буквы,
- спецсимволы.
Считать анаграммами строки с одинаковым мультимножеством символов.
Подход с массивом на 26 букв здесь уже не подходит, потому что алфавит не фиксирован. Поэтому код сначала строит универсальную карту частот по code point, а потом превращает её в стабильный канонический ключ. Дальше задача снова становится обычной группировкой по HashMap: одинаковый ключ означает одну и ту же анаграммную группу.
class Solution {
    public List<List<String>> groupAnagramsMixed(String[] strs) {
        Map<String, List<String>> groups = new HashMap<>();
        for (String s : strs) {
            String key = buildKey(s);
            groups.computeIfAbsent(key, k -> new ArrayList<>()).add(s);
        }
        return new ArrayList<>(groups.values());
    }
    private String buildKey(String s) {
        // key по кодпоинтам: работает для русского/английского и спецсимволов
        Map<Integer, Integer> freq = new TreeMap<>();
        s.codePoints().forEach(cp -> freq.merge(cp, 1, Integer::sum));
        StringBuilder key = new StringBuilder();
        for (Map.Entry<Integer, Integer> e : freq.entrySet()) {
            key.append(e.getKey()).append(':').append(e.getValue()).append('|');
        }
        return key.toString();
    }
}
Время: O(T log K), Память: O(T); T — суммарное число кодпоинтов по всем строкам, K — число различных символов в одном слове (лог из TreeMap на символ)
На каждое слово: обход кодпоинтов O(длина), обновления в TreeMap O(log K) на уникальный символ в этом слове. Суммарно O(T log K). Память: ключи и группы суммарно O(T).
Первый уникальный символ в строке
Дана строка. Нужно вернуть первый символ строки, который встречается в ней ровно один раз.
Если такого символа нет — вернуть что-то другое.
Ход мысли здесь такой же, как и в версии с индексом: сначала нужно узнать частоты, а уже потом искать первый подходящий символ в исходном порядке. Поэтому код делает два прохода. Первый собирает счётчик, второй возвращает первый символ, у которого частота равна единице.
class Solution {
    public char firstUniqueChar(String s) {
        Map<Character, Integer> freq = new HashMap<>();
        for (char c : s.toCharArray()) {
            freq.merge(c, 1, Integer::sum);
        }
        for (char c : s.toCharArray()) { //идем по исходной строке
            if (freq.get(c) == 1) {
                return c;
            }
        }
        return '\0';
    }
}
Время: O(n), Память: O(k) (k — число различных символов в строке, k ≤ n; HashMap символ → частота)
Два линейных прохода по строке — O(n). В карте не больше k записей.
Группировка чисел по перестановке цифр (нули незначимы)
Дан массив неотрицательных целых чисел. Нужно сгруппировать числа, которые можно получить друг из друга перестановкой цифр.
Важно:
- ведущие нули не считаются значимыми (числа вида 011 не существует),
- значит, дополнительные нули можно «увести» в начало и они исчезнут.
Если группировать числа просто по всем цифрам, нули начнут искусственно разделять эквивалентные случаи, хотя по условию они незначимы. Значит, в ключ нужно включать только то, что реально влияет на класс: количества цифр от 1 до 9. Именно поэтому код игнорирует нули при построении канонического представления. После этого числа попадают в одну группу ровно тогда, когда различаются только перестановкой и расстановкой нулей.
class Solution {
    public List<List<Integer>> digitPermutationGroups(int[] arr) {
        Map<String, List<Integer>> groups = new LinkedHashMap<>();
        for (int num : arr) {
            String key = buildKeyIgnoringZeros(num);
            groups.computeIfAbsent(key, k -> new ArrayList<>()).add(num);
        }
        return new ArrayList<>(groups.values());
    }
    private String buildKeyIgnoringZeros(int num) {
        if (num == 0) {
            return "";
        }
        int[] cnt = new int[10];
        while (num > 0) {
            int d = num % 10;
            if (d != 0) cnt[d]++;
            num /= 10;
        }
        StringBuilder key = new StringBuilder();
        for (int d = 1; d <= 9; d++) {
            if (cnt[d] > 0) {
                key.append(d).append('#').append(cnt[d]).append('|');
            }
        }
        return key.toString();
    }
}
Время: O(n·d), Память: O(n) (группы в HashMap); d — число десятичных цифр в числе; cnt[10] при сборке ключа — константа размера алфавита цифр
n — количество чисел, d — количество цифр в числе. Для каждого числа делаем один проход по его цифрам и формируем короткий ключ. HashMap хранит все группы — O(n) суммарно.
Minimum Size Subarray Sum
Дан массив положительных чисел nums и число target. Найти минимальную длину непрерывного подмассива с суммой >= target. Если такого нет — вернуть 0.
Скользящее окно: расширяем правый край, добавляя к сумме. Когда sum >= target — сжимаем слева, обновляя минимальную длину. Каждый элемент добавляется и удаляется один раз → O(n).
class Solution {
    public int minSubArrayLen(int target, int[] nums) {
        int left = 0;
        int sum = 0;
        int minLen = Integer.MAX_VALUE;
        for (int right = 0; right < nums.length; right++) {
            sum += nums[right];
            while (sum >= target) {
                minLen = Math.min(minLen, right - left + 1);
                sum -= nums[left];
                left++;
            }
        }
        return minLen == Integer.MAX_VALUE ? 0 : minLen;
    }
}
Время: O(n), Память: O(1)
Скользящее окно: каждый индекс входит/выходит один раз — O(n). Счётчики — O(1) памяти.
Sliding Window Maximum
Дан массив nums и размер окна k. Для каждой позиции окна вернуть максимум.
Монотонный убывающий дек хранит индексы. Голова = индекс максимума окна. При добавлении нового элемента удаляем из хвоста все меньшие/равные. Удаляем из головы вышедшие за окно. Каждый элемент push/pop один раз → O(n).
class Solution {
    public int[] maxSlidingWindow(int[] nums, int k) {
        int n = nums.length;
        int[] result = new int[n - k + 1];
        Deque<Integer> deque = new ArrayDeque<>();
        for (int i = 0; i < n; i++) {
            while (!deque.isEmpty() && deque.peekFirst() < i - k + 1) {
                deque.pollFirst();
            }
            while (!deque.isEmpty() && nums[deque.peekLast()] <= nums[i]) {
                deque.pollLast();
            }
            deque.addLast(i);
            if (i >= k - 1) {
                result[i - k + 1] = nums[deque.peekFirst()];
            }
        }
        return result;
    }
}
Время: O(n), Память: O(k)
Монотонный deque: каждый индекс добавляется/удаляется O(1) раз — O(n). Deque размера O(k) памяти.
LeetCode 161: One Edit Distance
Проверить, находятся ли две строки на расстоянии ровно одной правки (одного изменения) (вставка, удаление или замена одного символа).
Если длины равны — при первом расхождении проверяем, что остальные совпадают (замена). Если разница 1 — пропускаем символ в длинной строке (вставка/удаление). O(n).
public boolean isOneEditDistance(String s, String t) {
    int sLen = s.length(), tLen = t.length();
    if (Math.abs(sLen - tLen) > 1) return false;
    if (sLen > tLen) return isOneEditDistance(t, s);
    for (int i = 0; i < sLen; i++) {//идем по короткой строке
        if (s.charAt(i) != t.charAt(i)) {
            if (sLen == tLen) {
                for (int j = i + 1; j < sLen; j++)
                    if (s.charAt(j) != t.charAt(j)) {
                        return false;
                    }
            } else {
                // Пропускаем символ в длинной строке (t)
                for (int j = i; j < sLen; j++)
                    if (s.charAt(j) != t.charAt(j + 1)) {
                        return false;
                    }
            }
        }
    }
    return true; 
}
Время: O(n), Память: O(1)
Один линейный проход с ранним выходом — O(n). Несколько индексов — O(1) памяти.
Longest Palindrome Two Letter Words
Дан массив слов из двух букв. Найти длину самого длинного палиндрома, который можно составить из этих слов.
Для пар (ab, ba): берём min(count) пар, каждая даёт +4. Для палиндромов (aa): пары дают +4, один нечётный можно в центр (+2). O(n) время.
class Solution {
    public int longestPalindrome(String[] words) {
        // Храним частоту каждого слова
        Map<String, Integer> freq = new HashMap<>();
        int length = 0;           // текущая длина палиндрома
        boolean centerUsed = false; // использовали ли центральный элемент (для палиндромов типа "aa")
        // Считаем частоту каждого слова
        for (String w : words) {
            freq.merge(w, 1, Integer::sum);
        }
        // Проходим по всем уникальным словам
        for (String w : freq.keySet()) {
            // Создаём перевёрнутую версию слова (например, "ab" -> "ba")
            String rev = "" + w.charAt(1) + w.charAt(0);
            // СЛУЧАЙ 1: слово является палиндромом (например, "aa", "bb", "zz")
            if (w.equals(rev)) {
                int cnt = freq.get(w);  // сколько раз встречается это слово
                // Каждая пара "aa" + "aa" даёт 4 символа (aa в начало и aa в конец)
                // Пример: "aa" + ... + "aa" → ...aa...aa...
                length += (cnt / 2) * 4;
                // Если есть нечётное количество палиндромов и центр ещё не использован
                // Один палиндром можно поставить в центр (он не требует пары)
                // Пример: "aa" + "bb" + "aa" → центральная "bb" непарная
                if (cnt % 2 == 1 && !centerUsed) {
                    length += 2;           // добавляем 2 символа в центр
                    centerUsed = true;    // центр занят, больше добавлять нельзя
                }
            }
            // СЛУЧАЙ 2: слово НЕ палиндром и мы ещё не обработали его пару
            // Используем w.compareTo(rev) < 0, чтобы обработать каждую пару только один раз
            // Например: для "ab" и "ba" обработаем только когда w = "ab" (т.к. "ab" < "ba")
            else if (w.compareTo(rev) < 0 && freq.containsKey(rev)) {
                // Берём минимальное количество пар, которые могут встретиться
                // Пример: "ab" встречается 3 раза, "ba" встречается 2 раза
                // Мы можем сделать только 2 полные пары (2 × "ab" + 2 × "ba")
                int pairs = Math.min(freq.get(w), freq.get(rev));
                // Каждая полная пара даёт 4 символа
                // Пример: "ab" + ... + "ba" → ...ab...ba...
                length += pairs * 4;
            }
            // Если w.compareTo(rev) > 0 — пропускаем, т.к. пара будет обработана
            // когда мы дойдём до rev (который меньше w)
        }
        return length;
    }
}
Время: O(n), Память: O(n)
Подсчёт частот пар в HashMap — O(n). Карта и ключи — O(n) памяти.
LC 243 · Shortest Word Distance
Дан массив строк words и два разных слова word1 и word2. Найти минимальную абсолютную разность индексов между любым вхождением word1 и любым вхождением word2.
Один проход: обновляем позицию при нахождении word1 или word2. Если обе позиции известны — считаем расстояние. O(n) время, O(1) память.
class Solution {
    public int shortestDistance(String[] words, String word1, String word2) {
        int pos1 = -1, pos2 = -1;
        int minDist = Integer.MAX_VALUE;
        for (int i = 0; i < words.length; i++) {
            if (words[i].equals(word1)) {
                pos1 = i;
            } else if (words[i].equals(word2)) {
                pos2 = i;
            }
            if (pos1 != -1 && pos2 != -1) {
                minDist = Math.min(minDist, Math.abs(pos1 - pos2));
            }
        }
        return minDist;
    }
}
Время: O(n), Память: O(1)
Один проход с обновлением позиций двух слов — O(n). Переменные — O(1) памяти.
LC 821 Shortest Distance to a Character
Дана строка s и символ c (гарантированно есть в s). Для каждой позиции найти расстояние до ближайшего вхождения c.
Два прохода: слева→ записываем расстояние от последнего c. Справа→ обновляем минимумом с расстоянием от ближайшего c справа. O(n) время, O(1) доп. память.
class Solution {
    public int[] shortestToChar(String s, char c) {
        int n = s.length();
        int[] result = new int[n];
        int prev = -n;
        for (int i = 0; i < n; i++) {
            if (s.charAt(i) == c) {
                 prev = i;
            }
            result[i] = i - prev;
        }
        prev = 2 * n;
        for (int i = n - 1; i >= 0; i--) {
            if (s.charAt(i) == c) {
                 prev = i;
            }
            result[i] = Math.min(result[i], prev - i); //проверяем какое расстояние меньше: справа или слева
        }
        return result;
    }
}
Время: O(n), Память: O(n)
Два прохода слева/справа для расстояния до c — O(n). Массив ответа — O(n) памяти.
Product of Array Except Self
Дан массив nums. Вернуть массив result, где result[i] = произведение всех элементов кроме nums[i].
Без деления, за O(n).
Для каждого i нужно произведение всего, кроме nums[i]. Делим задачу: prefix[i] = product(0..i-1), suffix[i] = product(i+1..n-1). Первый проход строит prefix в result[], второй — досчитывает suffix на ходу переменной suffix и умножает. Деление не нужно.
class Solution {
    public int[] productExceptSelf(int[] nums) {
        int n = nums.length;
        int[] result = new int[n];
        result[0] = 1;
        for (int i = 1; i < n; i++) {
            result[i] = result[i - 1] * nums[i - 1]; // prefix
        }
        int suffix = 1;
        for (int i = n - 1; i >= 0; i--) {
            result[i] *= suffix; // домножаем suffix справа
            suffix *= nums[i];
        }
        return result;
    }
}
Время: O(n), Память: O(1)
Два прохода по массиву — O(n). result не считается доп. памятью (это ответ). Только переменная suffix — O(1).
LeetCode 42 Trapping Rain Water
Дан массив высот столбиков. Посчитать сколько воды задерживается после дождя.
Уровень воды определяется меньшим из двух ограничивающих столбиков слева и справа.
Вода над позицией i ограничена min(maxLeft, maxRight) − height[i]. 
Два указателя: если левая стенка меньше — правая уже достаточно высока, считаем воду слева и двигаем left. 
Иначе — считаем справа и двигаем right.
class Solution {
    public int trap(int[] height) {
        // Два указателя: левый и правый, движутся навстречу друг другу
        int left = 0, right = height.length - 1;
        // Максимальная высота стенки слева и справа от текущей позиции
        int maxLeft = 0, maxRight = 0;
        // Общее количество забранной воды
        int water = 0;
        // Пока указатели не встретятся
        while (left < right) {
            // Если левая стенка ниже или равна правой
            // (значит, уровень воды слева ограничен левой стенкой)
            if (height[left] <= height[right]) {
                // Если текущая левая стенка выше всех предыдущих слева
                if (height[left] >= maxLeft) {
                    // Обновляем максимум - вода здесь не задержится
                    maxLeft = height[left];
                } else {
                    // Иначе над текущим столбиком может стоять вода
                    // Количество воды = высота левой "дамбы" минус высота текущего столбца
                    water += maxLeft - height[left];
                }
                // Сдвигаем левый указатель вправо
                left++;
            } else {
                // Симметричный случай: правая стенка ниже левой
                // Значит, уровень воды справа ограничен правой стенкой
                // Если текущая правая стенка выше всех предыдущих справа
                if (height[right] >= maxRight) {
                    // Обновляем максимум
                    maxRight = height[right];
                } else {
                    // Иначе над текущим столбиком есть вода
                    water += maxRight - height[right];
                }
                // Сдвигаем правый указатель влево
                right--;
            }
        }
        // Возвращаем общее количество воды
        return water;
    }
}
Время: O(n), Память: O(1)
Один проход двумя указателями — O(n). Только 4 переменные — O(1) памяти.
Longest Mountain in Array
Найти длину самой длинной горы в массиве. Гора: строго возрастает, затем строго убывает, длина ≥ 3.
up[i] считает сколько шагов строго вверх завершается в i, down[i] — сколько шагов строго вниз начинается в i. Вершина горы — позиция, где оба > 0. Длина горы = up[i] + down[i] + 1 (сама вершина).
class Solution {
    public int longestMountain(int[] arr) {
        int n = arr.length;
        // up[i] - количество шагов строго вверх, заканчивающихся в позиции i
        // (длина восходящего участка, завершающегося в i)
        int[] up = new int[n];
        // down[i] - количество шагов строго вниз, начинающихся в позиции i
        // (длина нисходящего участка, начинающегося в i)
        int[] down = new int[n];
        // Заполняем up[]: проходим слева направо
        for (int i = 1; i < n; i++) {
            // Если текущий элемент больше предыдущего (подъем)
            if (arr[i] > arr[i - 1]) {
                up[i] = up[i - 1] + 1;  // продолжаем предыдущий восходящий участок
            }
            // Иначе up[i] остается 0 (нет подъема, заканчивающегося в i)
        }
        // Заполняем down[]: проходим справа налево
        for (int i = n - 2; i >= 0; i--) {
            // Если текущий элемент больше следующего (спуск впереди)
            if (arr[i] > arr[i + 1]) {
                down[i] = down[i + 1] + 1;  // продолжаем предыдущий нисходящий участок
            }
            // Иначе down[i] остается 0 (нет спуска, начинающегося в i)
        }
        int result = 0;  // максимальная длина горы
        // Проверяем каждую позицию как потенциальную вершину горы
        for (int i = 0; i < n; i++) {
            // Вершина горы: должен быть подъем слева (up[i] > 0) 
            // и спуск справа (down[i] > 0)
            if (up[i] > 0 && down[i] > 0) {
                // Длина горы = длина подъема + длина спуска + 1 (вершина)
                result = Math.max(result, up[i] + down[i] + 1);
            }
        }
        return result;  // возвращаем максимальную длину горы
    }
}
Время: O(n), Память: O(n)
Три прохода по массиву — O(n). Два вспомогательных массива up[] и down[] — O(n) памяти.
LC 125 Valid Palindrome
Дана строка s. Определить, является ли она палиндромом, если учитывать только буквенно-цифровые символы (без учёта регистра).
Два указателя с концов, пропускаем не-алфавитные символы. Сравниваем без регистра. O(n) время, O(1) память.
class Solution {
    public boolean isPalindrome(String s) {
        int left = 0, right = s.length() - 1;
        while (left < right) {
            while (left < right && !Character.isLetterOrDigit(s.charAt(left))) left++;
            while (left < right && !Character.isLetterOrDigit(s.charAt(right))) right--;
            if (Character.toLowerCase(s.charAt(left)) != Character.toLowerCase(s.charAt(right))) {
                return false;
            }
            left++;
            right--;
        }
        return true;
    }
}
Время: O(n), Память: O(1)
Один проход двумя указателями — O(n). Никаких дополнительных структур — O(1).
LC 167 · Two Sum II - Input Array Is Sorted
Дан отсортированный массив numbers (1-индексация) и target. Найти два числа, дающих в сумме target. Вернуть их индексы.
Отсортированность даёт монотонность суммы: меньше — двигаем левый, больше — правый. Гарантированно находим за O(n).
class Solution {
    public int[] twoSum(int[] numbers, int target) {
        int left = 0, right = numbers.length - 1;
        while (left < right) {
            int sum = numbers[left] + numbers[right];
            if (sum == target) return new int[]{left + 1, right + 1};
            else if (sum < target) left++;
            else right--;
        }
        return new int[]{-1, -1};
    }
}
Время: O(n), Память: O(1)
Один проход — O(n). Только два указателя — O(1).
LC 344 Reverse String
Дан массив символов s. Развернуть его in-place (без дополнительной памяти).
Классика двух указателей с концов: меняем местами элементы, идём к центру. O(n) время, O(1) память.
class Solution {
    public void reverseString(char[] s) {
        int left = 0, right = s.length - 1;
        while (left < right) {
            char tmp = s[left];
            s[left] = s[right];
            s[right] = tmp;
            left++;
            right--;
        }
    }
}
Время: O(n), Память: O(1)
n/2 свапов — O(n). In-place — O(1).
LeetCode 26. Remove Duplicates from Sorted Array
Дан отсортированный массив nums. Удалить дубликаты in-place, вернуть количество уникальных элементов k.
slow отстаёт от fast: slow фиксирует позицию последнего уникального. Новый уникальный элемент — записываем в slow+1. O(n) время, O(1) память.
class Solution {
    public int removeDuplicates(int[] nums) {
        // writePointer — куда записываем следующий уникальный элемент
        // readPointer — откуда читаем следующий элемент
        int writePointer = 0;
        for (int readPointer = 1; readPointer < nums.length; readPointer++) {
            // Если нашли новый уникальный элемент
            if (nums[readPointer] != nums[writePointer]) {
                writePointer++;                          // двигаем указатель записи
                nums[writePointer] = nums[readPointer];  // записываем уникальный элемент
            }
            // Если дубликат — просто пропускаем (readPointer увеличивается в цикле)
        }
        // Количество уникальных элементов = writePointer + 1
        return writePointer + 1;
    }
}
Время: O(n), Память: O(1)
Один проход — O(n). In-place — O(1).
LC 27 Remove Element
Дан массив nums и число val. Удалить in-place все вхождения val, вернуть количество оставшихся элементов.
slow — курсор записи. Копируем в начало только нужные элементы (не равные val). O(n) время, O(1) память.
class Solution {
    public int removeElement(int[] nums, int val) {
        int slow = 0;
        for (int fast = 0; fast < nums.length; fast++) {
            if (nums[fast] != val) {
                nums[slow] = nums[fast];
                slow++;
            }
        }
        return slow;
    }
}
Время: O(n), Память: O(1)
Один проход — O(n). In-place — O(1).
LC 283 · Move Zeroes
Дан массив nums. Переместить все нули в конец, сохранив порядок остальных элементов. In-place.
Идея здесь та же, что и в другой версии Move Zeroes: не нужно тасовать нули по одному. Код держит slow как позицию, куда должен встать следующий ненулевой элемент, и одним проходом сдвигает все полезные значения влево. После этого всё, что осталось справа от slow, уже должно быть нулями, и хвост просто дозаполняется.
class Solution {
    public void moveZeroes(int[] nums) {
        int slow = 0;
        for (int fast = 0; fast < nums.length; fast++) {
            if (nums[fast] != 0) {
                nums[slow] = nums[fast];
                slow++;
            }
        }
        while (slow < nums.length) {
            nums[slow++] = 0;
        }
    }
}
Время: O(n), Память: O(1) доп. (индексы slow/fast; массив in-place)
До двух проходов по массиву длины n. Только индексы — O(1) доп. памяти.
LC 392 · Is Subsequence
Даны строки s и t. Определить, является ли s подпоследовательностью t (символы s встречаются в t в том же порядке, но не обязательно подряд).
Жадно ищем символы s в t. Нашли — сдвигаем i. Прошли всю t — если i достиг конца s, то s — подпоследовательность. O(n) время, O(1) память.
class Solution {
    public boolean isSubsequence(String s, String t) {
        int i = 0, j = 0;
        while (i < s.length() && j < t.length()) {
            if (s.charAt(i) == t.charAt(j)) i++;
            j++;
        }
        return i == s.length();
    }
}
Время: O(n), Память: O(1)
Один проход по t — O(n). Только два указателя — O(1).
LC 680 · Valid Palindrome II
Дана строка s. Можно удалить не более одного символа. Определить, можно ли сделать строку палиндромом.
При первом несовпадении пробуем пропустить один символ (левый или правый) и проверяем остаток. Не более одного пропуска — O(n) время.
class Solution {
    public boolean validPalindrome(String s) {
        int left = 0, right = s.length() - 1;
        while (left < right) {
            if (s.charAt(left) != s.charAt(right))
                return isPalin(s, left+1, right) || isPalin(s, left, right-1);
            left++;
            right--;
        }
        return true;
    }
    private boolean isPalin(String s, int l, int r) {
        while (l < r) {
            if (s.charAt(l) != s.charAt(r)) return false;
            l++; r--;
        }
        return true;
    }
}
Время: O(n), Память: O(1)
Основной проход + одна дополнительная проверка — O(n). Только указатели — O(1).
Container With Most Water
Дан массив height. Найти два столбца, которые вместе с осью X образуют контейнер с максимальным количеством воды.
Объём ограничен меньшей стенкой. Двигая меньшую стенку — единственный шанс найти большее. Двигать большую бессмысленно: ширина уменьшится, а высота не вырастет. O(n) время.
class Solution {
    public int maxArea(int[] height) {
        int left = 0, right = height.length - 1;
        int max = 0;
        while (left < right) {
            int area = Math.min(height[left], height[right]) * (right - left);
            max = Math.max(max, area);
            if (height[left] < height[right]) left++;
            else right--;
        }
        return max;
    }
}
Время: O(n), Память: O(1)
Один проход — O(n). Только три переменные — O(1).
3Sum
Дан массив nums. Найти все уникальные тройки, дающие сумму 0.
Сортируем, фиксируем i. Для остатка — классические two pointers: sum < 0 → left++, sum > 0 → right--. Пропуск дубликатов гарантирует уникальность. O(n²).
class Solution {
    public List<List<Integer>> threeSum(int[] nums) {
        Arrays.sort(nums);
        List<List<Integer>> res = new ArrayList<>();
        for (int i = 0; i < nums.length - 2; i++) {
            if (i > 0 && nums[i] == nums[i-1]) continue;
            int left = i + 1, right = nums.length - 1;
            while (left < right) {
                int sum = nums[i] + nums[left] + nums[right];
                if (sum == 0) {
                    res.add(Arrays.asList(nums[i], nums[left], nums[right]));
                    while (left < right && nums[left] == nums[left+1]) left++;
                    while (left < right && nums[right] == nums[right-1]) right--;
                    left++; right--;
                } else if (sum < 0) left++;
                else right--;
            }
        }
        return res;
    }
}
Время: O(n²), Память: O(1)
Сортировка O(n log n) + двойной проход O(n²). Хранение результата не считается — O(1) доп. памяти.
LeetCode 16. 3Sum Closest
Дан массив nums и target. Найти тройку с суммой, ближайшей к target. Вернуть эту сумму.
Для этой задачи не существует алгоритма лучше, чем O(n²).
Для каждого фиксированного i:
left и right проходят вместе по всем элементам от i+1 до n-1
class Solution {
    public int threeSumClosest(int[] nums, int target) {
        // Сортируем массив - это позволит использовать два указателя
        // и двигать их в зависимости от суммы
        Arrays.sort(nums);
        // Инициализируем closest первыми тремя числами
        // (нужно с чем-то сравнивать)
        int closest = nums[0] + nums[1] + nums[2];
        // ВНЕШНИЙ ЦИКЛ: фиксируем первый элемент тройки
        // Идем до nums.length - 2, так как нужно место для left и right
        for (int i = 0; i < nums.length - 2; i++) {
            // ДВА УКАЗАТЕЛЯ:
            // left - следующий после i элемент
            // right - последний элемент массива
            int left = i + 1;
            int right = nums.length - 1;
            // ВНУТРЕННИЙ ЦИКЛ:
            // left и right движутся навстречу друг другу
            // В СУММЕ за все итерации внешнего цикла они проходят O(n) шагов
            while (left < right) {
                // Текущая сумма трех элементов
                int sum = nums[i] + nums[left] + nums[right];
                // Если текущая сумма ближе к target, чем предыдущая - обновляем closest
                if (Math.abs(sum - target) < Math.abs(closest - target)) {
                    closest = sum;
                }
                // Двигаем указатели в зависимости от того, куда нужно изменить сумму
                if (sum < target) {
                    // Сумма слишком маленькая → нужно её увеличить
                    // Для этого двигаем left вправо (к бОльшим числам)
                    left++;
                } else if (sum > target) {
                    // Сумма слишком большая → нужно её уменьшить
                    // Для этого двигаем right влево (к меньшим числам)
                    right--;
                } else {
                    // Точно попали в target! Разница = 0
                    // Лучше быть не может → сразу возвращаем
                    return sum;
                }
            }
        }
        // Возвращаем closest - сумму, которая оказалась ближе всех к target
        return closest;
    }
}
Время: O(n²), Память: O(1)
Сортировка O(n log n) + two pointers O(n²). O(1) доп. памяти.
4Sum
Дан массив nums и target. Найти все уникальные четвёрки с суммой target.
Расширение 3Sum: фиксируем два элемента, на остатке — two pointers. Пропуск дубликатов на каждом уровне. O(n³).
class Solution {
    public List<List<Integer>> fourSum(int[] nums, int target) {
        Arrays.sort(nums);
        List<List<Integer>> res = new ArrayList<>();
        for (int i = 0; i < nums.length - 3; i++) {
            if (i > 0 && nums[i] == nums[i-1]) continue;
            for (int j = i + 1; j < nums.length - 2; j++) {
                if (j > i + 1 && nums[j] == nums[j-1]) continue;
                int left = j + 1, right = nums.length - 1;
                while (left < right) {
                    long sum = (long)nums[i] + nums[j] + nums[left] + nums[right];
                    if (sum == target) {
                        res.add(Arrays.asList(nums[i],nums[j],nums[left],nums[right]));
                        while (left < right && nums[left] == nums[left+1]) left++;
                        while (left < right && nums[right] == nums[right-1]) right--;
                        left++; right--;
                    } else if (sum < target) left++;
                    else right--;
                }
            }
        }
        return res;
    }
}
Время: O(n³), Память: O(1)
Три вложенных прохода — O(n³). O(1) доп. памяти.
Sort Colors
Дан массив с числами 0 (красный), 1 (белый), 2 (синий). Отсортировать in-place без sort().
Dutch National Flag: low..mid-1 — нули, mid..high — единицы (необработанные), high+1..n-1 — двойки. Один проход. O(n) время, O(1) память.
class Solution {
    public void sortColors(int[] nums) {
        int low = 0, mid = 0, high = nums.length - 1;
        while (mid <= high) {
            if (nums[mid] == 0) {
                int tmp = nums[low]; nums[low] = nums[mid]; nums[mid] = tmp;
                low++; mid++;
            } else if (nums[mid] == 2) {
                int tmp = nums[mid]; nums[mid] = nums[high]; nums[high] = tmp;
                high--;
            } else {
                mid++;
            }
        }
    }
}
Время: O(n), Память: O(1)
Один проход — O(n). In-place три указателя — O(1).
LeetCode 142. Linked List Cycle II
Дан связный список. Найти узел, где начинается цикл. Если цикла нет — вернуть null.
Обычный список (без цикла):
[3] → [2] → [0] → [-4] → null (Конец! Никуда не ведёт)
Алгоритм Флойда: fast и slow встречаются внутри цикла. После встречи — slow в head, двигаем оба по 1 шагу: встретятся ровно в начале цикла (математически доказуемо). O(n), O(1).
public class Solution {
    public ListNode detectCycle(ListNode head) {
        ListNode slow = head, fast = head;
        while (fast != null && fast.next != null) {
            slow = slow.next;
            fast = fast.next.next;
            if (slow == fast) {
                slow = head;
                while (slow != fast) {
                    slow = slow.next;
                    fast = fast.next;
                }
                return slow;
            }
        }
        return null;
    }
}
Время: O(n), Память: O(1)
Два прохода — O(n). Только два указателя — O(1).
Find the Duplicate Number
Дан массив из n+1 чисел, каждое от 1 до n. Ровно одно число дублируется. Найти его без изменения массива, O(1) памяти.
Массив как связный список: индекс i ведёт в nums[i]. Дубликат создаёт цикл. Алгоритм Флойда находит вход в цикл = дублирующее число. O(n), O(1).
class Solution {
    public int findDuplicate(int[] nums) {
        int slow = nums[0], fast = nums[0];
        do {
            slow = nums[slow];
            fast = nums[nums[fast]];
        } while (slow != fast);
        slow = nums[0];
        while (slow != fast) {
            slow = nums[slow];
            fast = nums[fast];
        }
        return slow;
    }
}
Время: O(n), Память: O(1)
Два прохода по списку — O(n). Только два указателя — O(1).
Longest Repeating Character Replacement
Дана строка s и число k. Можно заменить не более k символов. Найти длину самой длинной подстроки, состоящей из одного символа.
Заменять нужно все символы кроме самого частого. Если замен больше k — сужаем окно. maxCount не уменьшаем (нас интересует максимальное окно). O(n) время, O(1) память.
class Solution {
    public int characterReplacement(String s, int k) {
        int[] count = new int[26];
        int left = 0, maxCount = 0, result = 0;
        for (int right = 0; right < s.length(); right++) {
            maxCount = Math.max(maxCount, ++count[s.charAt(right) - 'A']);
            while ((right - left + 1) - maxCount > k) {
                count[s.charAt(left) - 'A']--;
                left++;
            }
            result = Math.max(result, right - left + 1);
        }
        return result;
    }
}
Время: O(n), Память: O(1)
Один проход — O(n). Массив из 26 символов — O(1).
Subarray Product Less Than K
Дан массив nums и число k. Найти количество непрерывных подмассивов, произведение элементов которых строго меньше k.
Для каждого right — максимальное окно с product < k. Новых подмассивов, заканчивающихся на right: right - left + 1. O(n) время, O(1) память.
class Solution {
    public int numSubarrayProductLessThanK(int[] nums, int k) {
        if (k <= 1) return 0;
        int left = 0, product = 1, count = 0;
        for (int right = 0; right < nums.length; right++) {
            product *= nums[right];
            while (product >= k) {
                product /= nums[left];
                left++;
            }
            count += right - left + 1;
        }
        return count;
    }
}
Время: O(n), Память: O(1)
Один проход — O(n). Только три переменные — O(1).
Boats to Save People
Дан массив people (веса) и limit (грузоподъёмность лодки). В лодке не более 2 человек, суммарный вес ≤ limit. Найти минимальное количество лодок.
Самый тяжёлый всегда занимает лодку. Если с ним помещается самый лёгкий — берём обоих. Сортировка + two pointers: O(n log n) время, O(1) память.
class Solution {
    public int numRescueBoats(int[] people, int limit) {
        Arrays.sort(people);
        int left = 0, right = people.length - 1, boats = 0;
        while (left <= right) {
            if (people[left] + people[right] <= limit) left++;
            right--;
            boats++;
        }
        return boats;
    }
}
Время: O(n log n), Память: O(1)
Сортировка O(n log n), один проход O(n). O(1) доп. памяти.
Interval List Intersections
Даны два списка интервалов firstList и secondList (каждый отсортирован). Найти все пересечения.
Два указателя: пересечение двух отрезков — max начал до min концов. Интервал с меньшим концом больше не пересечётся с текущим — двигаем его. O(n+m) время.
class Solution {
    public int[][] intervalIntersection(int[][] first, int[][] second) {
        List<int[]> res = new ArrayList<>();
        int i = 0, j = 0;
        while (i < first.length && j < second.length) {
            int lo = Math.max(first[i][0], second[j][0]);
            int hi = Math.min(first[i][1], second[j][1]);
            if (lo <= hi) res.add(new int[]{lo, hi});
            if (first[i][1] < second[j][1]) i++;
            else j++;
        }
        return res.toArray(new int[0][]);
    }
}
Время: O(n+m), Память: O(1)
Один проход по обоим спискам — O(n+m). Только два указателя — O(1) доп. памяти.
Max Consecutive Ones III
Дан бинарный массив nums и число k. Можно перевернуть не более k нулей. Найти длину самой длинной подпоследовательности из единиц.
Если смотреть на задачу правильно, мы ищем самое длинное окно, внутри которого можно "потратить" не больше k удалений нулей. Поэтому код держит количество нулей внутри текущего окна и расширяет right, пока ограничение не нарушено. Когда нулей стало слишком много, left двигается вправо, пока окно снова не станет допустимым. Максимальная длина такого окна и есть ответ.
class Solution {
    public int longestOnes(int[] nums, int k) {
        int left = 0, zeros = 0, result = 0;
        for (int right = 0; right < nums.length; right++) {
            if (nums[right] == 0) zeros++;
            while (zeros > k) {
                if (nums[left] == 0) zeros--;
                left++;
            }
            result = Math.max(result, right - left + 1);
        }
        return result;
    }
}
Время: O(n) (right и left только вперёд по массиву), Память: O(1) доп. (left, right, zeros, result — константное число целых)
Каждый индекс добавляется в окно и может вытолкнуться слева не более одного раза — O(n).
LC 1768 · Merge Strings Alternately
Даны две строки word1 и word2. Составить новую строку, поочерёдно беря символы: сначала из word1, затем из word2. Если одна строка длиннее — дописать остаток.
Два указателя на две строки. Поочерёдный выбор символов, затем дописываем хвост. O(n+m) время и память.
class Solution {
    public String mergeAlternately(String word1, String word2) {
        StringBuilder sb = new StringBuilder();
        int p1 = 0, p2 = 0;
        while (p1 < word1.length() && p2 < word2.length()) {
            sb.append(word1.charAt(p1++));
            sb.append(word2.charAt(p2++));
        }
        // Добавляем остатки (если есть)
        sb.append(word1.substring(p1));
        sb.append(word2.substring(p2));
        return sb.toString();
    }
}
Время: O(n+m), Память: O(n+m)
Один проход по обеим строкам — O(n+m). StringBuilder — O(n+m) памяти.
Односторонняя разница
Даны два массива nums1 и nums2, отсортированные по не убыванию.
Необходимо вернуть все элементы из nums1, которые не встречаются в nums2, так же в порядке по не убыванию.
Главная идея — использовать два указателя на отсортированных массивах. Если nums1[p1] меньше nums2[p2], текущее значение из nums1 уже точно не встретится дальше в nums2, значит его надо добавить в ответ. Если nums1[p1] больше, двигаем p2 и догоняем. Если значения равны, это число нужно удалить из ответа полностью, поэтому просто пропускаем элемент в nums1. За счёт того, что p2 остаётся на том же значении, все дубли этого числа в nums1 тоже будут пропущены.
import java.util.*;
public class Solution {
    public List<Integer> findDifference(List<Integer> nums1, List<Integer> nums2) {
        List<Integer> result = new ArrayList<>();
        int p1 = 0;
        int p2 = 0;
        while (p1 < nums1.size()) {
            // если nums2 закончился — все оставшиеся из nums1 идут в результат
            if (p2 >= nums2.size()) {
                result.add(nums1.get(p1));
                p1++;
                continue;
            }
            // если текущий элемент nums1 меньше — он есть только в nums1
            if (nums1.get(p1) < nums2.get(p2)) {
                result.add(nums1.get(p1));
                p1++;
            } else if (nums1.get(p1) > nums2.get(p2)) {
                p2++;
            } else {
                // при равных элементах пропускаем в nums1
                // для верной обработки дублей в nums1
                p1++;
            }
        }
        return result;
    }
}
Время: O(n+m), Память: O(k)
Оба указателя движутся только вперёд: p1 не более n шагов, p2 не более m шагов, значит суммарно O(n+m). Дополнительная память нужна только под ответ из k элементов.
К ближайших чисел (якорь idx)
Дан массив nums, отсортированный в неубывающем порядке, индекс idx и число k. Нужно найти k ближайших к значению nums[idx] чисел в массиве и вернуть в любом порядке. При равных расстояниях предпочтение отдаётся меньшим числам.
Массив отсортирован, опорное значение ref = nums[idx]. Разносим указатели влево и вправо как у слияния: всегда добавляем конец с меньшим расстоянием до ref; при равенстве — левый (меньший элемент). O(k) время и память.
import java.util.*;
class Solution {
    public List<Integer> findNearestNumbers(List<Integer> nums, int idx, int k) {
        if (k == 0) {
            return new ArrayList<>();
        }
        List<Integer> result = new ArrayList<>();
        int ref = nums.get(idx);
        result.add(ref);
        int left = idx - 1;
        int right = idx + 1;
        for (int i = 0; i < k - 1; i++) {
            boolean canTakeLeft = left >= 0;
            boolean canTakeRight = right < nums.size();
            if (!canTakeRight) {
                result.add(nums.get(left--));
                continue;
            }
            if (!canTakeLeft) {
                result.add(nums.get(right++));
                continue;
            }
            // Оба доступны — выбираем ближайший к ref
            int leftDist = Math.abs(ref - nums.get(left));
            int rightDist = Math.abs(ref - nums.get(right));
            if (leftDist <= rightDist) {
                result.add(nums.get(left--));  
            } else {
                result.add(nums.get(right++)); 
            }
        }
        return result;
    }
}
Время: O(k), Память: O(k)
Ровно k−1 итераций расширения от центра, на шаг константная работа — O(k). Новый список длины k — O(k) памяти.
LC 234. Палиндром в связном списке
Дан head односвязного списка. Вернуть true, если список является палиндромом, иначе false.
Ищем середину двумя указателями. Разворачиваем вторую половину списка in-place. После этого сравниваем первую половину с развёрнутой второй узел к узлу. Если все значения совпали — это палиндром.
class Solution {
    public boolean isPalindrome(ListNode head) {
        // 1. Находим середину (slow/fast)
        ListNode slow = head, fast = head;
        while (fast != null && fast.next != null) {
            slow = slow.next;
            fast = fast.next.next;
        }
        // 2. Разворачиваем вторую половину
        ListNode prev = null, curr = slow;
        while (curr != null) {
            ListNode next = curr.next;
            curr.next = prev;
            prev = curr;
            curr = next;
        }
        // 3. Сравниваем две половины
        ListNode left = head, right = prev;
        while (right != null) {
            if (left.val != right.val) return false;
            left = left.next;
            right = right.next;
        }
        return true;
    }
}
Время: O(n), Память: O(1)
Проходим список для поиска середины, затем для разворота второй половины и сравнения — суммарно O(n). Дополнительные указатели занимают O(1).
LC 557. Reverse Words in a String III
Дана строка s. Перевернуть символы в каждом слове, сохраняя порядок слов и пробелы между ними.
Сканируем строку слева направо. Как только встретили пробел или конец массива, разворачиваем текущий диапазон символов слова in-place в char[]. Затем продолжаем со следующего слова.
class Solution {
    public String reverseWords(String s) {
        char[] arr = s.toCharArray();
        int start = 0;
        for (int i = 0; i <= arr.length; i++) {
            if (i == arr.length || arr[i] == ' ') {
                reverse(arr, start, i - 1);
                start = i + 1;
            }
        }
        return new String(arr);
    }
    private void reverse(char[] arr, int l, int r) {
        while (l < r) {
            char tmp = arr[l];
            arr[l++] = arr[r];
            arr[r--] = tmp;
        }
    }
}
Время: O(n), Память: O(n)
Один проход для поиска границ слов и локальные развороты суммарно покрывают каждый символ константное число раз — O(n). char[] копия строки занимает O(n) памяти.
LC 281. Zigzag Iterator
Даны два списка целых чисел v1 и v2. Нужно итерироваться по ним поочерёдно: один элемент из первого, один из второго и так далее.
Если один список закончился, продолжаем брать оставшиеся элементы из другого.
Здесь важно не просто склеить два списка, а честно чередовать их, причём один источник может закончиться раньше другого. Поэтому код хранит оба итератора и флаг очереди, но берёт элемент из "текущего" только если у того ещё есть данные; иначе сразу переключается на другой источник. То есть реальная идея не в boolean как таковом, а в round-robin по непустым итераторам. Именно поэтому follow-up для k списков естественно обобщается в очередь итераторов.
public class ZigzagIterator {
    private Iterator<Integer> i1, i2;
    private boolean turn; // true = первый список
    public ZigzagIterator(List<Integer> v1, List<Integer> v2) {
        i1 = v1.iterator();
        i2 = v2.iterator();
        turn = i1.hasNext();
    }
    public int next() {
        if ((turn && i1.hasNext()) || !i2.hasNext()) {
            turn = i2.hasNext(); // следующий ход — второму, если есть
            return i1.next();
        } else {
            turn = i1.hasNext();
            return i2.next();
        }
    }
    public boolean hasNext() {
        return i1.hasNext() || i2.hasNext();
    }
}
Время: O(1) на вызов next/hasNext (константное число операций с итераторами), Память: O(1) доп. (две ссылки на Iterator + boolean turn; сами списки v1/v2 — вход)
Каждый вызов next/hasNext делает константное число проверок и переходов по итераторам. Храним только два итератора и флаг turn — O(1) памяти.
LC 163 · Missing Ranges
Дан отсортированный массив уникальных чисел и границы [lower, upper]. Найти все пропущенные диапазоны.
Ожидаемое значение expect; при разрыве добавляем диапазон через formatRange (как в Summary Ranges). O(n).
class Solution {
    public List<String> findMissingRanges(int[] nums, int lower, int upper) {
        List<String> result = new ArrayList<>();
        long expect = lower;
        int idx = 0;
        while (idx < nums.length) {
            int currentNum = nums[idx];
            if (currentNum > expect) {
                String missingRange = formatRange(expect, (long) currentNum - 1);
                result.add(missingRange);
            }
            expect = (long) currentNum + 1;
            idx++;
        }
        if (expect <= upper) {
            result.add(formatRange(expect, upper));
        }
        return result;
    }
    private String formatRange(long from, long to) {
        if (from == to) {
            return String.valueOf(from);
        }
        return from + "->" + to;
    }
}
Время: O(n), Память: O(1)
Один проход по nums с expect — O(n). Список строк — O(1) доп. памяти кроме вывода.
LC 228 · Summary Ranges
Дан отсортированный массив без дубликатов. Свернуть последовательные числа в диапазоны "a->b".
Группируем последовательные числа (nums[i+1] == nums[i] + 1). Одиночные — как число, диапазоны — как "start->end". O(n) время, O(1) доп. память.
class Solution {
    public List<String> summaryRanges(int[] nums) {
        List<String> list = new ArrayList<>();
        for (int i = 0; i < nums.length; i++) {
            int start = nums[i];
            while (i + 1 < nums.length && nums[i + 1] == nums[i] + 1) {
                i++;
            }
            if (start == nums[i]) {
                list.add(String.valueOf(start));
            } else {
                list.add(start + "->" + nums[i]);
            }
        }
        return list;
    }
}
Время: O(n), Память: O(1)
Указатель с расширением непрерывных отрезков — O(n). Список строк — O(1) доп. памяти.
Interval List Intersections
Даны два списка непересекающихся отсортированных интервалов. Найти их пересечения.
Два указателя по спискам. Пересечение = [max(starts), min(ends)], если lo <= hi. Двигаем указатель, чей интервал заканчивается раньше. O(n+m) время.
class Solution {
    public int[][] intervalIntersection(
            int[][] firstList, int[][] secondList) {
        List<int[]> result = new ArrayList<>();
        int i = 0, j = 0;
        while (i < firstList.length
            && j < secondList.length) {
            int lo = Math.max(firstList[i][0],
                secondList[j][0]);
            int hi = Math.min(firstList[i][1],
                secondList[j][1]);
            if (lo <= hi) {
                result.add(new int[]{lo, hi});
            }
            if (firstList[i][1] < secondList[j][1]) {
                i++;
            } else {
                j++;
            }
        }
        return result.toArray(new int[result.size()][]);
    }
}
Время: O(n + m), Память: O(1)
Два указателя по спискам интервалов — O(n+m). Указатели — O(1) памяти.
986. Interval List Intersections
Даны два отсортированных массива непересекающихся отрезков segments1 и segments2, где каждый отрезок — пара [начало, конец]. Нужно вернуть массив всех попарных пересечений отрезков из segments1 с отрезками из segments2.
Инвариант как у слияния двух отсортированных списков интервалов: текущая пара — единственная кандидатура на ненулевое пересечение на этом шаге. После обработки отбрасываем отрезок, который раньше заканчивается — он уже не пересечётся с «хвостом» другого.
import java.util.*;
class Solution {
    public int[][] intersectSegments(int[][] segments1, int[][] segments2) {
        List<int[]> out = new ArrayList<>();
        int i = 0, j = 0;
        while (i < segments1.length && j < segments2.length) {
            int lo = Math.max(segments1[i][0], segments2[j][0]);
            int hi = Math.min(segments1[i][1], segments2[j][1]);
            if (lo <= hi) {
                out.add(new int[]{lo, hi});
            }
            if (segments1[i][1] < segments2[j][1]) {
                i++;
            } else {
                j++;
            }
        }
        return out.toArray(new int[0][]);
    }
}
Время: O(n + m), Память: O(1) доп. кроме ответа
Каждый указатель двигается только вперёд — суммарно O(n+m). Константное число переменных — O(1) доп. памяти.
Prefix Common Array
Даны две перестановки A и B длины n (числа от 1 до n). Найти массив C, где C[i] = количество общих чисел в префиксах A[0..i] и B[0..i].
Перестановки содержат числа 1..n по одному разу. freq[v] считает суммарные вхождения. Когда freq == 2 — число встречено в обоих массивах. O(n) время и память.
class Solution {
    public int[] findThePrefixCommonArray(
            int[] A, int[] B) {
        int n = A.length;
        int[] freq = new int[n + 1];
        int[] C = new int[n];
        int common = 0;
        for (int i = 0; i < n; i++) {
            freq[A[i]]++;
            if (freq[A[i]] == 2) common++;
            freq[B[i]]++;
            if (freq[B[i]] == 2) common++;
            C[i] = common;
        }
        return C;
    }
}
Время: O(n), Память: O(n)
Один цикл с обновлением freq и счётчика общих — O(n). Массивы freq и C — O(n) памяти.
LC 674 · Longest Continuous Increasing Subsequence
Найти длину самой длинной непрерывно возрастающей подпоследовательности (подмассива).
Один проход: если nums[i] > nums[i-1] — увеличиваем текущую длину. Иначе сбрасываем в 1. Отслеживаем максимум. O(n) время, O(1) память.
class Solution {
    public int findLengthOfLCIS(int[] nums) {
        if (nums.length == 0) return 0;
        int maxLen = 1, curLen = 1;
        for (int i = 1; i < nums.length; i++) {
            if (nums[i] > nums[i - 1]) {
                curLen++;
                maxLen = Math.max(maxLen, curLen);
            } else {
                curLen = 1;
            }
        }
        return maxLen;
    }
}
Время: O(n), Память: O(1)
Один проход с curLen — O(n). Несколько переменных — O(1) памяти.
LC 643 · Maximum Average Subarray I 
Дан массив nums и число k. Найти максимальное среднее подмассива длиной ровно k.
Фиксированное окно длины k. Инициализируем сумму первых k элементов, затем сдвигаем окно за один проход: прибавляем правый, вычитаем левый. Отслеживаем максимальную сумму, делим на k.
class Solution {
    public double findMaxAverage(int[] nums, int k) {
        int sum = 0;
        for (int i = 0; i < k; i++) {
            sum += nums[i];
        }
        int maxSum = sum;
        for (int right = k; right < nums.length; right++) {
            sum += nums[right] - nums[right - k];
            maxSum = Math.max(maxSum, sum);
        }
        return (double) maxSum / k;
    }
}
Время: O(n), Память: O(1)
Один проход по массиву с фиксированным окном — O(n). Только несколько переменных — O(1) памяти.
Subarrays of Size K and Average > Threshold
Дан массив nums, число k и порог threshold. Вернуть количество подмассивов длиной k, среднее значение которых строго больше threshold.
Вместо сравнения среднего (sum/k > threshold) умножаем порог: sum > threshold*k — избегаем деления. Фиксированное окно длины k сдвигается за один проход.
class Solution {
    public int numOfSubarrays(int[] arr, int k, int threshold) {
        int sum = 0;
        int count = 0;
        int target = threshold * k;
        for (int i = 0; i < k; i++) {
            sum += arr[i];
        }
        if (sum > target) count++;
        for (int right = k; right < arr.length; right++) {
            sum += arr[right] - arr[right - k];
            if (sum > target) count++;
        }
        return count;
    }
}
Время: O(n), Память: O(1)
Один проход с фиксированным окном — O(n). Только несколько переменных — O(1) памяти.
Longest Substring Without Repeating Characters
Найти длину самой длинной подстроки без повторяющихся символов.
Скользящее окно: расширяем right. При повторном символе наивно сжимали бы left по одному, но зная последнюю позицию символа можно прыгнуть сразу: left = lastIndex[c] + 1. Прыгаем только вперёд — если lastIndex[c] < left, символ уже вне окна, прыгать не нужно.
Массив int[128] вместо HashMap — O(1) память. Каждый символ обрабатывается один раз — O(n).
class Solution {
    public int lengthOfLongestSubstring(String s) {
        int[] lastIndex = new int[128];  // ASCII 0-127
        Arrays.fill(lastIndex, -1);
        int left = 0;
        int result = 0;
        for (int right = 0; right < s.length(); right++) {
            char c = s.charAt(right);
            if (lastIndex[c] >= left) {
                left = lastIndex[c] + 1;
            }
            lastIndex[c] = right;
            result = Math.max(result, right - left + 1);
        }
        return result;
    }
}
Время: O(n), Память: O(min(n, m))
Каждый символ добавляется и удаляется из set не более одного раза — O(n). Set хранит символы текущего окна — O(min(n,m)), где m — размер алфавита.
Подстроки без повторов — число пар (i, j)
Дана строка s. Требуется найти количество пар индексов (i, j) таких, что 0 ≤ i ≤ j < s.length() и все символы в подстроке s[i..j] (включительно) различны (в подстроке нет повторов).
Максимально расширяем окно вправо без повторов. Для фиксированного l любая подстрока s[l..k] при l ≤ k ≤ r уже уникальна по построению — ровно (r−l+1) штук. Сдвигаем l и повторяем.
public long countSubstrings(String s) {
    int[] lastSeenPlusOne = new int[128];
    long count = 0;
    int left = 0;
    for (int right = 0; right < s.length(); right++) {
        char c = s.charAt(right);
        left = Math.max(left, lastSeenPlusOne[c]);
        lastSeenPlusOne[c] = right + 1;
        count += right - left + 1;
    }
    return count;
}
Время: O(n), Память: O(min(n, |Σ|))
Каждый индекс входит в окно и выходит из него не более одного раза — O(n) по времени. В множестве не больше уникальных символов алфавита — O(min(n, |Σ|)) памяти.
LC 340 · Longest Substring with At Most K Distinct Characters
Дана строка s и целое число k. Нужно найти длину самой длинной подстроки, которая содержит не более k различных символов.
В лоб можно проверять все подстроки, но код опирается на более сильный инвариант: текущее окно всегда содержит не более k различных символов. Пока это условие выполняется, right можно спокойно расширять. Как только разных символов стало слишком много, нужно сдвигать left и уменьшать частоты, пока окно снова не станет валидным. После каждого такого восстановления инварианта длина окна становится кандидатом в ответ.
class Solution {
    public int lengthOfLongestSubstringKDistinct(String s, int k) {
        if (k == 0 || s.isEmpty()) return 0;
        Map<Character, Integer> freq = new HashMap<>();
        int left = 0;
        int maxLen = 0;
        for (int right = 0; right < s.length(); right++) {
            char c = s.charAt(right);
            freq.put(c, freq.getOrDefault(c, 0) + 1);
            while (freq.size() > k) {
                char leftChar = s.charAt(left);
                int count = freq.get(leftChar) - 1;
                if (count == 0) {
                    freq.remove(leftChar);
                } else {
                    freq.put(leftChar, count);
                }
                left++;
            }
            maxLen = Math.max(maxLen, right - left + 1);
        }
        return maxLen;
    }
}
Время: O(n) (|s| = n; каждый символ входит/выходит из окна ≤1 раз), Память: O(min(n, |Σ|)) (|Σ| — размер алфавита в условии; в map не больше числа различных символов в текущем окне)
Каждый символ добавляется в окно и удаляется из него не более одного раза, поэтому оба указателя проходят строку линейно — O(n). HashMap хранит частоты символов текущего окна, максимум по числу разных символов в строке — O(min(n, |Σ|)).
Умножение строк
Даны два неотрицательных числа в виде строк. Вернуть их произведение в виде строки. Нельзя использовать BigInteger или встроенное умножение строк.
Умножение столбиком: произведение цифр num1[i] и num2[j] попадает в позиции [i+j] и [i+j+1]. Обработка переноса встроена. Ведущие нули удаляются. O(m×n) время.
class Solution {
    public String multiply(String num1, String num2) {
        int m = num1.length(), n = num2.length();
        int[] result = new int[m + n];
        for (int i = m - 1; i >= 0; i--) {
            for (int j = n - 1; j >= 0; j--) {
                int mul = (num1.charAt(i) - '0')
                    * (num2.charAt(j) - '0');
                int p1 = i + j;
                int p2 = i + j + 1;
                int sum = mul + result[p2];
                result[p2] = sum % 10;
                result[p1] += sum / 10;
            }
        }
        StringBuilder sb = new StringBuilder();
        for (int digit : result) {
            if (!(sb.length() == 0 && digit == 0)) {
                sb.append(digit);
            }
        }
        return sb.length() == 0 ? "0" : sb.toString();
    }
}
Время: O(m·n), Память: O(m+n)
Вложенные циклы по парам цифр — O(m·n). Массив result[m+n] — O(m+n) памяти.
LC 415. Сложение строк
Даны два неотрицательных целых числа num1 и num2 в виде строк. Вернуть их сумму в виде строки.
Нельзя использовать BigInteger и нельзя напрямую конвертировать строки в числа.
Складываем числа как в столбик: берём текущие цифры справа, прибавляем carry, пишем младший разряд в результат и обновляем carry. Так как цифры добавляются с конца, в финале разворачиваем StringBuilder.
class Solution {
    public String addStrings(String num1, String num2) {
        StringBuilder sb = new StringBuilder();
        int i = num1.length() - 1;
        int j = num2.length() - 1;
        int carry = 0;
        while (i >= 0 || j >= 0 || carry > 0) {
            int a = i >= 0 ? num1.charAt(i--) - '0' : 0;
            int b = j >= 0 ? num2.charAt(j--) - '0' : 0;
            int sum = a + b + carry;
            sb.append(sum % 10);
            carry = sum / 10;
        }
        return sb.reverse().toString();
    }
}
Время: O(max(n, m)), Память: O(max(n, m))
Один проход по цифрам справа налево с переносом — O(max(n,m)). StringBuilder хранит итоговую строку длины до max(n,m)+1 — O(max(n,m)).
Неточный поиск
Дана строка text и строка pattern. Найти первую позицию в text, где pattern встречается с не более чем k несовпадениями (остальные символы совпадают точно).
Для каждой стартовой позиции i в text сравниваем посимвольно с pattern. Считаем несовпадения; если k превышен — прерываем внутренний цикл. При mismatches ≤ k — возвращаем i. O(n·m) время, O(1) память.
class Solution {
    public int fuzzySearch(String text, String pattern, int k) {
        int n = text.length(), m = pattern.length();
        for (int i = 0; i <= n - m; i++) {
            int mismatches = 0;
            for (int j = 0; j < m; j++) {
                if (text.charAt(i + j) != pattern.charAt(j)) {
                    mismatches++;
                    if (mismatches > k) break;
                }
            }
            if (mismatches <= k) return i;
        }
        return -1;
    }
}
Время: O(n·m), Память: O(1)
Внешний цикл O(n), внутренний O(m) с ранним выходом — O(n·m) в худшем случае. Только счётчики — O(1) памяти.
Лучшее место на парковке
Дан массив spots, где spots[i] = 1 — занятое место, spots[i] = 0 — свободное. Нужно выбрать свободное место так, чтобы расстояние до ближайшей занятой машины было максимальным. Вернуть это максимальное расстояние.
Важно: гарантировано есть хотя бы одно свободное и хотя бы одно занятое место.
Свободные места разбиты на непрерывные блоки нулей. У левого или правого края оптимально встать в край блока — расстояние до ближайшей единицы равно длине блока. Между двумя единицами оптимум в середине: минимальное из двух расстояний максимизируется как ⌈длина/2⌉, что для целых длин совпадает с (r−l+2)/2.
class Solution {
    public int maxDistToClosest(int[] spots) {
        int n = spots.length;
        int maxDist = 0;
        int lastOccupied = -1;
        for (int i = 0; i < n; i++) {
            if (spots[i] == 1) {
                if (lastOccupied == -1) {
                    // Левый край: расстояние от начала до первой занятой позиции
                    maxDist = i;
                } else {
                    // Между двумя занятыми: оптимально сесть посередине
                    maxDist = Math.max(maxDist, (i - lastOccupied) / 2);
                }
                lastOccupied = i;
            }
        }
        // Правый край: расстояние от последней занятой позиции до конца
        maxDist = Math.max(maxDist, n - 1 - lastOccupied);
        return maxDist;
    }
}
Время: O(n), Память: O(1)
Каждый индекс посещается константное число раз при движении l и r — O(n). Только несколько целых — O(1) доп. памяти.
Сложение графиков
Даны два массива nums1 и nums2, каждый — список пар [время, значение]. Нужно сложить два графика в один и вернуть итоговый график в виде списка пар [время, сумма значений], отсортированный по времени.
Между точками значение графика не меняется (ступень): на каждом времени из объединения множеств меток берётся последняя заданная величина для каждого графика.
Каждый график — кусочно-постоянная функция времени. После сортировки сливаем уникальные моменты времени как при merge двух отсортированных последовательностей: при совпадении времени с очередной точкой графика обновляем его текущее значение и сдвигаем указатель; в ответ всегда пишем сумму двух текущих значений.
import java.util.*;
public class Solution {
    public List<List<Integer>> merge(List<List<Integer>> nums1, List<List<Integer>> nums2) {
        nums1.sort(Comparator.comparingInt(p -> p.get(0)));
        nums2.sort(Comparator.comparingInt(p -> p.get(0)));
        int p1 = 0;
        int p2 = 0;
        int currentValue1 = 0;
        int currentValue2 = 0;
        List<List<Integer>> result = new ArrayList<>();
        while (p1 < nums1.size() || p2 < nums2.size()) {
            boolean has1 = p1 < nums1.size();
            boolean has2 = p2 < nums2.size();
            int time1 = has1 ? nums1.get(p1).get(0) : Integer.MAX_VALUE;
            int time2 = has2 ? nums2.get(p2).get(0) : Integer.MAX_VALUE;
            int time = Math.min(time1, time2);
            if (time1 == time) {
                currentValue1 = nums1.get(p1).get(1);
                p1++;
            }
            if (time2 == time) {
                currentValue2 = nums2.get(p2).get(1);
                p2++;
            }
            result.add(List.of(time, currentValue1 + currentValue2));
        }
        return result;
    }
}
Время: O(n log n + m log m), Память: O(n + m)
Сортировка — O(n log n) и O(m log m). Слияние указателями — O(n+m). Список результата — O(n+m) пар.
UserStatistics: окно k мс и ≥ limit запросов
Есть последовательность запросов пользователей, каждый запрос — это пара (time, userId), запросы всегда приходят в отсортированном по времени порядке.
Нужно быстро отвечать: сколько за последние k мс было пользователей, которые сделали >= limit запросов.
k > 0, limit > 0.
Скользящее окно по времени: дек хранит события в порядке прихода. Карта — сколько запросов у каждого userId внутри окна. hotCount увеличивается, когда частота пользователя впервые становится >= limit, и уменьшается, когда после удаления устаревших событий она падает ниже limit.
import java.util.*;
class UserStatistics {
    private final long k;
    private final long limit;
    private final Deque<Event> queue = new ArrayDeque<>();
    private final Map<Long, Integer> freq = new HashMap<>();
    private int hotCount = 0;
    // Вспомогательный класс для читаемости
    private static class Event {
        long time;
        long userId;
        Event(long time, long userId) {
            this.time = time;
            this.userId = userId;
        }
    }
    public UserStatistics(int k, int limit) {
        this.k = k;
        this.limit = limit;
    }
    public void event(long now, long userId) {
        //не очень обязательно, но если не вызывать expire, то старые события будут копиться в памяти.
        expire(now);
        // 2. Добавляем новое событие
        queue.addLast(new Event(now, userId));
        int count = freq.merge(userId, 1, Integer::sum);
        // 3. Обновляем счётчик "горячих" пользователей
        if (count == limit) {
            hotCount++;
        }
    }
    public int robotCount(long now) {
        expire(now);
        return hotCount;
    }
    private void expire(long now) {
        long windowStart = now - k;
        while (!queue.isEmpty() && queue.peekFirst().time < windowStart) {
            Event oldest = queue.pollFirst(); //уже не входит в окно
            long userId = oldest.userId
            int count = freq.get(userId); //сколько у этого юзера было запросов в окне
            if (count == limit) {
                hotCount--; 
            }
            if (count == 1) {
                freq.remove(userId); //этот юзер теперь не в окне, чтоб не накапливались юзеры бесконечно
            } else {
                freq.put(userId, count - 1);
            }
        }
    }
}
Время: амортизированно O(1) на вызов, Память: O(число событий в окне)
Каждое событие один раз попадает в дек и один раз из него удаляется — за всю последовательность линейно по числу событий.
LC 849 · Maximize Distance to Closest Person
Места в кинотеатре расположены в один ряд. Новый зритель хочет сесть так, чтобы расстояние до ближайшего занятого места было максимальным.
Дан массив seats:
- seats[i] = 1 — место занято
- seats[i] = 0 — место свободно
Гарантируется, что есть хотя бы одно свободное место и хотя бы одно занятое.
Нужно вернуть максимальную возможную дистанцию до ближайшего зрителя.
Группируем подряд идущие нули в блоки. Крайние блоки дают полный размер как расстояние до ближайшей единицы. Внутренний блок между единицами даёт максимум в середине: ceil(len/2).
class Solution {
    public int maxDistToClosest(int[] seats) {
        int n = seats.length;
        int maxDist = 0;
        int lastOccupied = -1;
        for (int i = 0; i < n; i++) {
            if (seats[i] == 1) {
                if (lastOccupied == -1) {
                    // Левый край: расстояние до первой занятой позиции
                    maxDist = i;
                } else {
                    // Между двумя занятыми: расстояние до ближайшего = (i - lastOccupied) / 2
                    maxDist = Math.max(maxDist, (i - lastOccupied) / 2);
                }
                lastOccupied = i;
            }
        }
        // Правый край: расстояние от последней занятой позиции до конца
        maxDist = Math.max(maxDist, n - 1 - lastOccupied);
        return maxDist;
    }
}
Время: O(n), Память: O(1)
Каждый индекс посещается константное число раз при одном проходе по массиву.
RLE с валидацией строки A-Z
Дана строка, содержащая только заглавные буквы английского алфавита (A-Z).
Нужно реализовать RLE-сжатие:
- если символ встречается один раз — оставляем символ как есть;
- если символ повторяется несколько раз — после символа пишем количество повторений.
«Невалидная строка» - это строка, содержащая любые символы, кроме латинских букв верхнего регистра (A-Z).
Сначала валидируем вход: строка считается невалидной, если содержит любой символ вне диапазона A-Z (а также если null/пустая). После этого группируем одинаковые соседние символы в блоки. Для каждого блока добавляем символ и, если длина блока больше 1, его размер.
public String rle(String s) {
    if (s == null || s.isEmpty()) {
        throw new IllegalArgumentException();
    }
    StringBuilder out = new StringBuilder();
    int i = 0;
    while (i < s.length()) {
        char c = s.charAt(i);
        validateUppercaseLetter(c);
        int count = 1;
        i++;
        while (i < s.length() && s.charAt(i) == c) {
            validateUppercaseLetter(s.charAt(i));
            count++;
            i++;
        }
        out.append(c);
        if (count > 1) {
            out.append(count);
        }
    }
    return out.toString();
}
private void validateUppercaseLetter(char c) {
    if (c < 'A' || c > 'Z') {
        throw new IllegalArgumentException();
    }
}
Время: O(n), Память: O(n)
Один линейный проход по строке. Дополнительная память — под результирующую строку.
Поиск подстроки с точностью до перестановки
Дан текст T и строка S. Нужно найти такую подстроку S' в T, что S' является перестановкой S (анаграммой), и вернуть начальный индекс этой подстроки.
Если подходящей подстроки нет — вернуть -1.
Проверка:
T = "a", S = "a" => 0
T = "abcd", S = "aaa" => -1
T = "abcd", S = "cbd" => 1
T = "abcdda", S = "bcdd" => 1
Подстрока длины |S| может быть перестановкой S только при совпадении частот. Вместо полного сравнения массивов на каждом шаге храним freq и счётчик need: при добавлении правого символа и удалении левого обновляем его за O(1). Поэтому после инициализации по S получаем один линейный проход по T: O(|T| + |S|), память O(1).
 public int findPermutation(String t, String s) {
      if (s.length() > t.length()) return -1;
      int[] freq = new int[26];
      for (char c : s.toCharArray()) freq[c - 'a']++;
      int len = s.length();
      int need = s.length();
      int left = 0;
      for (int right = 0; right < t.length(); right++) {
          // правый символ входит в окно
          int inIdx = t.charAt(right) - 'a';
          if (freq[inIdx] > 0) need--;
          freq[inIdx]--;
          // окно переросло len — левый выходит
          if (right - left + 1 > len) {
              int outIdx = t.charAt(left) - 'a';
              if (freq[outIdx] >= 0) need++;
              freq[outIdx]++;
              left++;
          }
          if (need == 0) return left;
      }
      return -1;
  }
Время: O(|T| + |S|), Память: O(1)
Сначала заполняем freq по строке S за O(|S|), затем один раз проводим окно по T. Каждый символ T не более одного раза входит в окно и одного раза выходит, поэтому суммарно O(|T|). Массив freq[26] фиксированного размера — O(1) памяти.
Кратчайшая подстрока, покрывающая алфавит
Даны 2 строки text и alphabet.
Нужно вернуть самую короткую подстроку text, в которой каждый символ из alphabet встречается хотя бы один раз. В подстроке могут быть и другие символы.
Если подходящих подстрок несколько одинаковой длины, вернуть ту, что начинается левее. Если подходящей подстроки нет — вернуть пустую строку.
Поддерживаем окно [left..right]. Когда окно покрывает все символы alphabet, сжимаем его слева до потери покрытия и обновляем лучший ответ. Первое найденное окно минимальной длины автоматически будет левейшим.
public String searchPangram(String text, String alphabet) {
    int[] count = new int[128];
    for (char c : alphabet.toCharArray()) {
        count[c]++;
    }
    int required = alphabet.length();
    int left = 0;
    int minLen = Integer.MAX_VALUE;
    int bestLeft = -1;
    for (int right = 0; right < text.length(); right++) {
        char c = text.charAt(right);
        // Уменьшаем счетчик для нужных символов
        if (count[c] > 0) {
            required--;
        }
        count[c]--;
        // Когда все символы найдены
        while (required == 0) {
            // Обновляем ответ
            if (right - left + 1 < minLen) {
                minLen = right - left + 1;
                bestLeft = left;
            }
            // Удаляем левый символ
            char leftChar = text.charAt(left);
            count[leftChar]++;
            if (count[leftChar] > 0) {
                required++;
            }
            left++;
        }
    }
    return bestLeft == -1 ? "" : text.substring(bestLeft, bestLeft + minLen);
}
Время: O(n), Память: O(A), где A = размер алфавита (здесь 256)
Каждый указатель (left/right) проходит строку не более одного раза. Доп. память — массивы частот/флагов фиксированного размера.
Ближайший митинг
Даны два списка свободных временных окон двух сотрудников: nums1 и nums2, где каждое окно представлено как [start, end] — начало и конец свободного периода в минутах. Гарантируется, что start < end. Длина окна равна end - start. Оба списка отсортированы по возрастанию начала интервала, интервалы внутри одного списка не пересекаются.
Нужно найти первое по времени окно длины duration, когда оба сотрудника свободны. Вернуть его как [start, end]. Если такого окна нет — вернуть пустой массив.
Пересечение двух текущих свободных отрезков — [max(start1,start2), min(end1,end2)]. Если его длина ≥ duration, самое раннее подходящее окно начинается в max(start1,start2). Если нет — отбрасываем интервал с меньшим end: он уже не даст более раннего пересечения с текущим партнёром.
import java.util.*;
class Solution {
    public List<Integer> findMeetingTime(List<List<Integer>> nums1, List<List<Integer>> nums2, int duration) {
        int i = 0, j = 0;
        while (i < nums1.size() && j < nums2.size()) {
            int start1 = nums1.get(i).get(0), end1 = nums1.get(i).get(1);
            int start2 = nums2.get(j).get(0), end2 = nums2.get(j).get(1);
            int intersection = Math.min(end1, end2) - Math.max(start1, start2);
            if (intersection >= duration) {
                int start = Math.max(start1, start2);
                return Arrays.asList(start, start + duration);
            }
            if (end1 < end2) {
                i++;
            } else {
                j++;
            }
        }
        return new ArrayList<>();
    }
}
Время: O(n + m), Память: O(1)
Каждый указатель проходит свой список не более одного раза — O(n+m). Храним только индексы и константное число целых — O(1).
Минимальная абсолютная разность между двумя массивами
Даны два НЕ ОТСОРТИРОВАННЫХ массива a и b. Нужно найти минимальное значение abs(a[i] - b[j]) по всем парам индексов i, j.
Без сортировки хочется сравнивать почти всё со всем, но это лишняя работа. После сортировки в каждый момент мы сравниваем текущие a[i] и b[j] и понимаем: улучшить ответ можно только двигая указатель у меньшего числа, потому что второй указатель и так стоит на ближайшем кандидате справа. Именно поэтому два указателя здесь проходят массивы линейно после сортировки.
class Solution {
    public int minAbsDiff(int[] a, int[] b) {
        Arrays.sort(a);
        Arrays.sort(b);
        int i = 0, j = 0;
        int ans = Integer.MAX_VALUE;
        while (i < a.length && j < b.length) {
            ans = Math.min(ans, Math.abs(a[i] - b[j]));
            if (a[i] < b[j]) i++;
            else j++;
        }
        return ans;
    }
}
Время: O(n log n + m log m) доминирует сортировка a и b + O(n+m) два указателя, Память: O(1) доп. к входу (только индексы i, j и ans; сортировка in-place — стек O(log n)+O(log m), обычно не выделяют отдельно)
Доминирует сортировка двух массивов. После сортировки два указателя проходят массивы линейно за O(n+m).
LC 362. Design Hit Counter
Реализовать счётчик запросов за последние 5 минут (300 секунд).
Методы:
- hit(timestamp): зафиксировать запрос в секунду timestamp
- getHits(timestamp): вернуть число запросов за последние 300 секунд (включая текущую секунду)
По коду видно, что нас интересует не вся история запросов, а только скользящее окно последних 300 секунд. Для такой модели очередь подходит идеально: новые hit добавляются в хвост, а при getHits из головы выкидывается всё, что уже вышло из окна. После этой очистки размер очереди и есть правильный ответ. В follow-up мысль остаётся той же, просто события агрегируются по секундам в фиксированном кольце из 300 бакетов.
public class HitCounter {
    private Queue<Integer> queue;
    public HitCounter() {
        queue = new LinkedList<>();
    }
    public void hit(int timestamp) {
        queue.offer(timestamp);
    }
    public int getHits(int timestamp) {
        while (!queue.isEmpty() && timestamp - queue.peek() >= 300) {
            queue.poll();
        }
        return queue.size();
    }
}
Время: O(1) hit (одна операция с очередью), O(q) getHits в худшем (q — число устаревших записей в очереди; суммарно по жизни объекта O(число hit)), Память: O(q_max) (очередь хранит метки в окне 300 с)
В базовой реализации очередь хранит все запросы из окна. hit — одна вставка O(1). getHits может удалить много устаревших элементов, в худшем O(n).
LC 933 · Number of Recent Calls
Есть класс RecentCounter, который считает количество недавних запросов.
Реализуй:
- RecentCounter() — инициализирует счётчик;
- int ping(int t) — добавляет запрос в момент времени t (в миллисекундах) и возвращает число запросов за последние 3000 мс, то есть в диапазоне [t - 3000, t].
Гарантируется, что значения t в вызовах ping строго возрастают.
Инвариант одинаковый: структура хранит только времена из актуального окна [t-3000, t]. После добавления нового t удаляем (или сдвигаем head) все значения < t-3000, затем размер структуры и есть ответ.
class RecentCounter {
    // По условию LeetCode максимум 10^4 вызовов ping
    private final int[] records = new int[10000];
    private int start;
    private int end;
    public RecentCounter() {
        start = 0;
        end = 0;
    }
    public int ping(int t) {
        while (start < end && t - records[start] > 3000) {
            start++;
        }
        records[end++] = t;
        return end - start;
    }
}
Решение 1 (массив+2 указателя): Время: O(1) амортизированно на ping, Память: O(n) (records хранит все добавленные времена; start/end — константное число индексов). Решение 2 (очередь): Время: O(1) амортизированно на ping, Память: O(w) (в очереди только окно [t-3000,t], где w — число запросов в окне)
В обоих подходах каждый timestamp добавляется один раз и удаляется (или пропускается указателем) не более одного раза, поэтому суммарно линейно по числу вызовов, а на один ping — амортизированно O(1).
LC 332 · Reconstruct Itinerary
Дан список авиабилетов [from, to]. Нужно восстановить маршрут, начиная с "JFK", который использует все билеты ровно один раз.
Если маршрутов несколько, вернуть лексикографически минимальный.
Если идти обычным DFS и жадно выбирать соседей без структуры, легко либо потерять лексикографический минимум, либо неправильно использовать все билеты. На самом деле задача про путь Эйлера: каждое ребро нужно пройти ровно один раз. Поэтому код хранит исходящие рейсы в min-heap, чтобы всегда брать минимальный по лексикографическому порядку, а сам маршрут собирает на выходе из DFS. Это и есть логика алгоритма Хиерхольцера.
class Solution {
    public List<String> findItinerary(List<List<String>> tickets) {
        Map<String, PriorityQueue<String>> graph = new HashMap<>();
        // строим граф: аэропорт → min-heap куда можно полететь
        // min-heap гарантирует лексикографический порядок
        for (List<String> t : tickets) {
            graph.computeIfAbsent(t.get(0), k -> new PriorityQueue<>()).offer(t.get(1));
        }
        LinkedList<String> route = new LinkedList<>();
        dfs("JFK", graph, route);
        return route;
    }
    private void dfs(String airport, Map<String, PriorityQueue<String>> graph, LinkedList<String> route) {
        PriorityQueue<String> next = graph.get(airport);
        // пока есть неиспользованные билеты из этого аэропорта
        while (next != null && !next.isEmpty()) {
            dfs(next.poll(), graph, route); // poll() удаляет билет — повторно не пройдём
        }
        // все исходящие рёбра обойдены — этот аэропорт занимает своё место в маршруте
        // addFirst потому что маршрут собирается с конца
        route.addFirst(airport);
    }
}
Время: O(E log D) (E — число билетов; из каждого аэропорта poll из min-heap степени исхода D; суммарно E извлечений из куч), Память: O(E) (граф из очередей + маршрут)
Каждое ребро удаляется из min-heap ровно один раз, операции с кучей дают логарифм. Храним граф и маршрут из O(E) элементов.
LC 199 · Binary Tree Right Side View
Вернуть значения узлов, которые видны при взгляде на бинарное дерево справа.
Смысл решения в том, чтобы на каждой глубине первым увидеть именно тот узел, который виден справа. Поэтому код идёт не по уровням, а DFS сначала в правого ребёнка, потом в левого. Если глубина встречается впервые, значит мы дошли до самого правого узла на этом уровне и его значение нужно записать в ответ.
public class Solution {
    public List<Integer> rightSideView(TreeNode root) {
        List<Integer> result = new ArrayList<>();
        rightView(root, result, 0);
        return result;
    }
    public void rightView(TreeNode curr, List<Integer> result, int currDepth) {
        if (curr == null) return;
        // Первый узел на глубине (при обходе правый → левый) — виден справа
        if (currDepth == result.size()) {
            result.add(curr.val);
        }
        // Сначала идём в правого ребёнка, потом в левого
        rightView(curr.right, result, currDepth + 1);
        rightView(curr.left, result, currDepth + 1);
    }
}
Время: O(n) (каждый узел один раз), Память: O(w) (очередь BFS; w — максимальная ширина уровня, т.е. макс. число узлов в очереди одновременно)
Каждый узел посещается один раз. В очереди максимум узлы одного уровня, где w — максимальная ширина дерева.
LC 380 · Insert Delete GetRandom O(1)
Реализовать структуру RandomizedSet с операциями:
- insert(val)
- remove(val)
- getRandom()
Все в среднем за O(1).
Трюк remove: вместо удаления из середины массива переносим последний элемент на место удаляемого и обновляем индекс в карте.
class RandomizedSet {
    private final List<Integer> values = new ArrayList<>();
    private final Map<Integer, Integer> index = new HashMap<>();
    private final Random random = new Random();
    public boolean insert(int val) {
        if (index.containsKey(val)) return false;
        index.put(val, values.size());
        values.add(val);
        return true;
    }
    public boolean remove(int val) {
        Integer i = index.get(val);
        if (i == null) return false;
        int lastIdx = values.size() - 1;
        int lastVal = values.get(lastIdx);
        values.set(i, lastVal);
        index.put(lastVal, i);
        values.remove(lastIdx);
        index.remove(val);
        return true;
    }
    public int getRandom() {
        return values.get(random.nextInt(values.size()));
    }
}
Время: O(1) амортизированно на операцию, Память: O(n)
HashMap даёт O(1) доступ к позиции, динамический массив — O(1) доступ по индексу и O(1) удаление с конца.
LC 88 · Merge Sorted Array
Даны два отсортированных массива nums1 и nums2, где в nums1 есть место под все элементы. Нужно слить их в nums1 по возрастанию.
Если сливать массивы с начала, мы быстро начнём перезаписывать значения в nums1, которые ещё сами не успели сравнить. Поэтому код идёт с конца, где в nums1 уже есть свободное место под результат. На каждом шаге выбирается больший из двух хвостов и ставится в конец. Так ничего полезного не затирается, и дополнительный массив не нужен.
class Solution {
    public void merge(int[] nums1, int m, int[] nums2, int n) {
        int i = m - 1, j = n - 1;
        int k = m + n - 1;
        while (j >= 0) {
            if (i >= 0 && nums1[i] > nums2[j]) {
                nums1[k--] = nums1[i--];
            } else {
                nums1[k--] = nums2[j--];
            }
        }
    }
}
Время: O(m+n) (каждый из m+n слотов nums1 обрабатывается один раз), Память: O(1) доп. (три индекса i, j, k; слияние в nums1 на месте)
Каждый элемент переносится ровно один раз. Используются только три индекса.
LC 236 · Lowest Common Ancestor of a Binary Tree
Дан корень бинарного дерева и два узла p и q. Найти их наименьшего общего предка (LCA).
ГАРАНТИРУЕТСЯ, что оба узла p и q СУЩЕСТВУЮТ в дереве.
Здесь удобно мыслить не глобально, а по возвращаемому значению рекурсии. Если поддерево содержит p или q, оно возвращает найденный узел наверх. Если непустой ответ пришёл и слева, и справа, значит именно в текущем узле пути к p и q сходятся впервые, и он и есть LCA. Если ответ пришёл только с одной стороны, его просто нужно протащить выше.
class Solution {
    public TreeNode lowestCommonAncestor(TreeNode root, TreeNode p, TreeNode q) {
        if (root == null || root == p || root == q) return root;
        TreeNode left = lowestCommonAncestor(root.left, p, q);
        TreeNode right = lowestCommonAncestor(root.right, p, q);
        if (left != null && right != null) {
            return root;
        }
        return left != null ? left : right;
    }
}
Время: O(n) (каждый узел один раз), Память: O(h) (стек рекурсии DFS; h — высота дерева)
Каждый узел посещаем один раз. Память — глубина стека рекурсии h.
LC 4 · Median of Two Sorted Arrays
Найти медиану двух отсортированных массивов за O(log(min(m,n))).
Правильный разрез удовлетворяет условиям l1<=r2 и l2<=r1. После этого медиана берётся из границ разрезов.
class Solution {
    public double findMedianSortedArrays(int[] nums1, int[] nums2) {
        if (nums1.length > nums2.length) return findMedianSortedArrays(nums2, nums1);
        int m = nums1.length, n = nums2.length;
        int leftSize = (m + n + 1) / 2;
        int lo = 0, hi = m;
        while (lo <= hi) {
            int cut1 = lo + (hi - lo) / 2;
            int cut2 = leftSize - cut1;
            int l1 = cut1 == 0 ? Integer.MIN_VALUE : nums1[cut1 - 1];
            int r1 = cut1 == m ? Integer.MAX_VALUE : nums1[cut1];
            int l2 = cut2 == 0 ? Integer.MIN_VALUE : nums2[cut2 - 1];
            int r2 = cut2 == n ? Integer.MAX_VALUE : nums2[cut2];
            if (l1 <= r2 && l2 <= r1) {
                if (((m + n) & 1) == 1) return Math.max(l1, l2);
                return (Math.max(l1, l2) + Math.min(r1, r2)) / 2.0;
            } else if (l1 > r2) {
                hi = cut1 - 1;
            } else {
                lo = cut1 + 1;
            }
        }
        return 0.0;
    }
}
Время: O(log(min(m,n))), Память: O(1)
Ищем позицию разреза бинарным поиском только по меньшему массиву.
LC 567 · Permutation in String
Даны строки s1 и s2. Проверить, содержит ли s2 подстроку, являющуюся перестановкой s1.
Перебирать все перестановки строки s1 бессмысленно. Гораздо важнее заметить, что любая перестановка имеет ту же длину и те же частоты букв. Значит, вместо генерации перестановок нужно просто пройти по s2 окном фиксированного размера и для каждого окна сравнить частоты символов с эталоном из s1. Именно эту идею и реализует код.
class Solution {
    public boolean checkInclusion(String s1, String s2) {
        if (s1.length() > s2.length()) return false;
        int[] freq = new int[26];
        for (char c : s1.toCharArray()) {
            freq[c - 'a']++;
        }
        int needCount = s1.length();
        int k = s1.length();
        int left = 0;
        for (int right = 0; right < s2.length(); right++) {
            int in = s2.charAt(right) - 'a';
            // Если этот символ ещё был нужен — закрыли одну потребность
            if (freq[in] > 0) {
                needCount--;
            }
            freq[in]--;
            // Если окно стало длиннее k — убираем левый символ
            if (right - left + 1 > k) {
                int out = s2.charAt(left) - 'a';
                // Если после удаления символ снова становится нужным
                if (freq[out] >= 0) {
                    needCount++;
                }
                freq[out]++;
                left++;
            }
            if (needCount == 0) {
                return true;
            }
        }
        return false;
    }
}
Время: O(|s2|) (окно скользит по s2), Память: O(1) доп. (два массива need[26] и win[26] — |Σ|=26 строчных латинских букв; сравнение массивов O(26)=O(1))
На каждом сдвиге окна обновляем две позиции массива частот длины 26.
LC 2 · Add Two Numbers
Даны два неотрицательных числа в виде связных списков (цифры в обратном порядке). Вернуть сумму в том же формате.
Побитовое (по разрядам) сложение с переносом, как в столбик.
class Solution {
    public ListNode addTwoNumbers(ListNode l1, ListNode l2) {
        ListNode dummy = new ListNode(0);
        ListNode cur = dummy;
        int carry = 0;
        while (l1 != null || l2 != null || carry != 0) {
            int a = l1 != null ? l1.val : 0;
            int b = l2 != null ? l2.val : 0;
            int sum = a + b + carry;
            carry = sum / 10;
            cur.next = new ListNode(sum % 10);
            cur = cur.next;
            if (l1 != null) l1 = l1.next;
            if (l2 != null) l2 = l2.next;
        }
        return dummy.next;
    }
}
Время: O(max(m,n)), Память: O(max(m,n))

LC 155 · Min Stack
Реализовать стек с операциями push, pop, top и getMin за O(1).
Обычный стек умеет быстро работать только с вершиной, но не с минимумом по всему содержимому. Значит, минимум нужно сопровождать отдельно. Код держит второй стек mins, куда кладутся только те значения, которые в момент вставки стали новым минимумом. Тогда текущий минимум всегда лежит на вершине mins, а при pop его нужно убирать синхронно только если удаляется именно этот минимум.
class MinStack {
    private final Deque<Integer> stack = new ArrayDeque<>(); // основной стек
    private final Deque<Integer> mins = new ArrayDeque<>();  // стек минимумов
    public void push(int val) {
        stack.push(val);
        // добавляем в mins только если val — новый минимум
        if (mins.isEmpty() || val <= mins.peek()) {
            mins.push(val);
        }
    }
    public void pop() {
        int x = stack.pop();
        // если удаляемый элемент был минимумом — убираем его и из mins
        if (x == mins.peek()) {
            mins.pop();
        }
    }
    public int top() {
        return stack.peek(); // верхушка основного стека
    }
    public int getMin() {
        return mins.peek(); // текущий минимум всегда на вершине mins
    }
}
Время: O(1) на операцию (push/pop/peek с deque), Память: O(n) (основной стек до n значений; mins — не больше числа «новых минимумов», в худшем O(n) дубликатов на вершине mins)
Все операции — push/pop/peek на deque. Дополнительный стек минимумов хранит кандидатов минимумов.
LC 1436 · Destination City
Дан список путей [from, to]. Нужно найти город, из которого нет исходящих путей.
Если представить билеты как ориентированные рёбра, конечный город выделяется сразу: в него можно приехать, но из него нельзя уехать дальше. Поэтому код сначала собирает все города-источники from в set, а потом ищет такой город назначения to, которого в этом множестве нет. Это и будет точка окончания маршрута.
class Solution {
    public String destCity(List<List<String>> paths) {
        Set<String> from = new HashSet<>();
        for (List<String> p : paths) {
            from.add(p.get(0));
        }
        for (List<String> p : paths) {
            if (!from.contains(p.get(1))) {
                return p.get(1);
            }
        }
        return "";
    }
}
Время: O(p) (p — число путей; два прохода по списку), Память: O(p) (HashSet городов-источников from; до p строк)
Один проход для источников и один для проверки назначения.
LC 1446 · Consecutive Characters
Сила строки — это максимальная длина непустой подстроки, состоящей только из одного уникального символа.
Дана строка s. Нужно вернуть силу этой строки.
Поддерживаем длину текущего блока одинаковых символов (curr). Если следующий символ такой же — увеличиваем curr, иначе фиксируем максимум (count) и начинаем новый блок с 1. Ответ — наибольшая длина блока.
class Solution {
    public int maxPower(String s) {
        int len = s.length();
        int count = 0;
        int curr = 1;
        for (int i = 0; i < len; i++) {
            if (i < len - 1 && s.charAt(i) == s.charAt(i + 1)) {
                curr++;
            } else {
                if (curr > count) {
                    count = curr;
                }
                curr = 1;
            }
        }
        return count;
    }
}
Время: O(n) (один проход по s длины n), Память: O(1) доп. (два счётчика count/curr и индекс i — константное число переменных)
Каждый символ строки обрабатывается один раз. Дополнительные структуры не используются.
LC 771 · Jewels and Stones
Даны строки jewels и stones.
- Символы в jewels — это типы камней, которые считаются драгоценностями.
- Символы в stones — камни, которые у тебя есть.
Нужно вернуть, сколько символов из stones являются драгоценностями (то есть входят в jewels).
Сравнение чувствительно к регистру: 'a' и 'A' — разные символы.
Идея прямая: считаем только те символы stones, которые присутствуют в jewels. Так как indexOf учитывает точный символ, регистр автоматически обрабатывается корректно.
class Solution {
    public int numJewelsInStones(String jewels, String stones) {
        int num = 0;
        for (int i = 0; i < stones.length(); i++) {
            if (jewels.indexOf(stones.charAt(i)) != -1) {
                num++;
            }
        }
        return num;
    }
}
Время: O(n·m) в худшем (n = |stones|, m = |jewels|; на каждом из n символов stones вызывается jewels.indexOf с линейным поиском по m), Память: O(1) доп. (только счётчик и индекс цикла)
Внешний цикл идёт по stones. Проверка indexOf для строки jewels — линейная по её длине. Дополнительных структур данных нет.
LC 35 · Search Insert Position
Дан отсортированный массив nums из различных целых чисел и число target.
Нужно:
- вернуть индекс target, если он есть в массиве;
- иначе вернуть индекс, куда его нужно вставить, чтобы порядок сортировки сохранился.
Требование: алгоритм должен работать за O(log n).
Если target найден, сразу возвращаем mid. Если не найден, цикл завершается при left > right, и left становится первой позицией, где элемент не меньше target — это и есть корректный индекс вставки.
class Solution {
    public int searchInsert(int[] nums, int target) {
        int left = 0;
        int right = nums.length - 1;
        while (left <= right) {
            int mid = left + (right - left) / 2;
            if (nums[mid] == target) {
                return mid;
            } else if (nums[mid] > target) {
                right = mid - 1;
            } else {
                left = mid + 1;
            }
        }
        return left;
    }
}
Время: O(log n) (каждая итерация отбрасывает половину диапазона), Память: O(1) доп. (три индекса left/right/mid)
Бинарный поиск на диапазоне индексов [0..n-1] делает логарифмическое число шагов. Дополнительных структур данных нет.
LC 69 · Sqrt(x)
Дано неотрицательное целое число x. Нужно вернуть целую часть квадратного корня из x (округление вниз).
Возвращаемое число тоже должно быть неотрицательным.
Важно: нельзя использовать встроенные функции/операторы возведения в степень (например, pow, x ** 0.5 и т.п.).
Ищем целый корень как «правую границу истинности»: предикат mid*mid <= x истинен до некоторой точки, потом ложен. Последний истинный mid и есть floor(sqrt(x)).
class Solution {
    public int mySqrt(int x) {
        if (x < 2) return x;
        int left = 1;
        int right = x / 2;
        int ans = 0;
        while (left <= right) {
            int mid = left + (right - left) / 2;
            if (mid <= x / mid) {
                ans = mid;        // mid^2 <= x, пробуем больше
                left = mid + 1;
            } else {
                right = mid - 1;  // mid^2 > x, идём влево
            }
        }
        return ans;
    }
}
Время: O(log x) (бинпоиск по диапазону [1..x/2]), Память: O(1) доп. (константное число переменных left/right/mid/ans)
На каждом шаге бинарный поиск отбрасывает половину диапазона. Используем только несколько целочисленных переменных.
LC 374 · Guess Number Higher or Lower
Играем в угадайку:
- Загадано число от 1 до n.
- Нужно угадать это число.
- После каждого неверного ответа API подсказывает, загаданное число больше или меньше.
Доступен API:
- guess(num)  -1, если num больше загаданного;
- guess(num)  1, если num меньше загаданного;
- guess(num)  0==, если num равен загаданному.
Нужно вернуть загаданное число.
На каждом шаге берём середину диапазона. Если guess(mid) говорит, что mid больше загаданного, сужаемся влево; если меньше — вправо. При res == 0 нашли точный ответ.
public class Solution extends GuessGame {
    public int guessNumber(int n) {
        int left = 1;
        int right = n;
        while (left <= right) {
            int mid = left + (right - left) / 2;
            int res = guess(mid);
            if (res == 0) {
                return mid;
            } else if (res < 0) {
                right = mid - 1; // mid слишком большой
            } else {
                left = mid + 1;  // mid слишком маленький
            }
        }
        return -1; // для корректного ввода не достигается
    }
}
Время: O(log n) (каждый шаг отбрасывает половину диапазона поиска), Память: O(1) доп. (индексы left/right/mid и переменная res)
Бинарный поиск делает логарифмическое число запросов к API guess. Дополнительные структуры данных не нужны.
LC 153 · Find Minimum in Rotated Sorted Array
Дан отсортированный массив без дубликатов, сдвинутый в неизвестной точке. Найти минимальный элемент за O(log n).
После поворота массив состоит из двух отсортированных кусков, а минимум сидит в месте перелома между ними. Сравнение nums[mid] с nums[right] как раз показывает, в каком куске мы находимся. Если mid больше right, значит минимум правее. Иначе перелом находится слева, включая mid. Поэтому здесь снова работает бинарное отбрасывание половины массива.
class Solution {
    public int findMin(int[] nums) {
        int left = 0;
        int right = nums.length - 1;
        while (left < right) {
            int mid = left + (right - left) / 2; //НЕТУ RETURN
            // Если mid > right → мы в левой части, минимум справа
            if (nums[mid] > nums[right]) {
                left = mid + 1;
            } 
            // Если mid <= right → мы в правой части, минимум слева (включая mid)
            else {
                right = mid;
            }
        }
        //ВАЖНО, что возвращаем левую границу
        return nums[left];
    }
}
Время: O(log n) (бинпоиск по индексам 0..n−1), Память: O(1) доп. (три индекса left, right, mid)
Каждая итерация отбрасывает половину диапазона.
LC 24 · Swap Nodes in Pairs
Поменять местами соседние узлы связного списка попарно: 1->2->3->4 превращается в 2->1->4->3.
Мы не меняем значения, только ссылки между узлами.
class Solution {
    public ListNode swapPairs(ListNode head) {
        ListNode dummy = new ListNode(0);
        dummy.next = head;
        ListNode prev = dummy;
        while (prev.next != null && prev.next.next != null) {
            ListNode a = prev.next;
            ListNode b = a.next;
            a.next = b.next;
            b.next = a;
            prev.next = b;
            prev = a;
        }
        return dummy.next;
    }
}
Время: O(n), Память: O(1)
Каждый узел участвует в перестановке не более одного раза.
LC 470 · Implement Rand10() Using Rand7()
Используя API rand7() (равновероятно 1..7), реализовать rand10() (равновероятно 1..10).
Чтобы сохранить равномерность, нельзя брать модуль от 49 напрямую. Нужно отбрасывать «лишний хвост» 41..49.
class Solution extends SolBase {
    public int rand10() {
        while (true) {
            int num = (rand7() - 1) * 7 + rand7(); // 1..49
            if (num <= 40) return 1 + (num - 1) % 10;
        }
    }
}
Время: O(1) в среднем, Память: O(1)
Отбраковка срабатывает редко: принимаем 40 из 49 исходов.
LC 71 · Simplify Path
Дан Unix-путь. Нормализовать его: убрать ".", обработать "..", лишние "/".
Стек отражает текущий абсолютный путь: ".." удаляет последний сегмент, "." игнорируется.
class Solution {
    public String simplifyPath(String path) {
        Deque<String> stack = new ArrayDeque<>();
        for (String part : path.split("/")) {
            if (part.isEmpty() || part.equals(".")) {
                continue;
            }
            if (part.equals("..")) {
                if (!stack.isEmpty()) {
                    stack.pollLast();
                }
            } else {
                stack.offerLast(part);
            }
        }
        return "/" + String.join("/", stack);
    }
}
Время: O(n), Память: O(n)
Один проход по сегментам пути, стек хранит валидные директории.
LC 21 · Merge Two Sorted Lists
Слить два отсортированных связных списка в один отсортированный.
Здесь основная мысль очень прямолинейная: пока у обоих списков есть голова, в ответ безопасно брать меньшую из двух, потому что она точно должна стоять раньше в отсортированном результате. После этого двигаем тот список, из которого взяли узел, и продолжаем. dummy нужен только для удобной сборки результата без отдельного случая для первого элемента.
class Solution {
    public ListNode mergeTwoLists(ListNode list1, ListNode list2) {
        ListNode dummy = new ListNode(0); // фиктивная голова — чтобы не писать отдельный случай для первого узла
        ListNode cur = dummy;             // cur — хвост уже собранного списка
        while (list1 != null && list2 != null) {
            if (list1.val <= list2.val) {
                cur.next = list1;        // берём узел из list1
                list1 = list1.next;      // двигаем list1 вперёд
            } else {
                cur.next = list2;        // берём узел из list2
                list2 = list2.next;      // двигаем list2 вперёд
            }
            cur = cur.next;              // двигаем хвост
        }
        // один из списков закончился — добавляем остаток другого
        cur.next = (list1 != null) ? list1 : list2;
        return dummy.next; // голова реального списка
    }
}
Время: O(m+n) (каждый узел list1/list2 рассматривается один раз), Память: O(1) доп. (указатели dummy, cur и два указателя на головы; новые узлы не создаём — только переставляем ссылки входных списков)
Каждый узел из обоих списков проходит через указатель cur один раз.
MIN и MAX зарплаты по отделу (не уволенные)
В базе данных две таблицы: units (подразделения) и employees (сотрудники).
Таблица **units**:
- id — первичный ключ
- name — название подразделения
Таблица **employees**:
- id — первичный ключ
- unit_id — внешний ключ на units.id
- salary — зарплата
- fired — флаг «уволен» (истина/ложь или 0/1 в зависимости от СУБД)
Нужно написать SQL-запрос, который вернёт **минимальную и максимальную зарплату по каждому отделу** среди не уволенных сотрудников. Результат: колонки (unit_id, min_salary, max_salary).
Подразделения, в которых нет ни одного активного сотрудника, в результат обычно не попадают (только строки из employees после фильтра).
Сначала отбрасываем уволенных (fired), затем для каждого unit_id считаем MIN и MAX по salary. Если fired хранится как 0/1, замените условие на fired = 0 или NOT fired.
SELECT
    unit_id,
    MIN(salary) AS min_salary,
    MAX(salary) AS max_salary
FROM employees
WHERE fired = FALSE   -- в MySQL часто: fired = 0
GROUP BY unit_id;
Время: O(n log n) при сортировке/хэш-группировке по плану СУБД, Память: O(u)
Один проход по employees с группировкой по unit_id — типично O(n) работы плюс стоимость группировки. Доп. память — по числу отделов u в группах.
